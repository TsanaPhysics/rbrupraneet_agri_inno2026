## (Synthesis of Decision Support Systems & AI for Agriculture)

## 🎯 วัตถุประสงค์การเรียนรู้ (Learning Objectives)
เมื่อศึกษาบทนี้จบแล้ว ผู้เรียนจะสามารถ:
1. **วิเคราะห์ (Analyze)** ส่วนประกอบเชิงสถาปัตยกรรมและกระบวนการทำงานของระบบสนับสนุนการตัดสินใจ (Decision Support System - DSS) ในการแปลงข้อมูลภูมิอากาศและเซ็นเซอร์เป็นกลยุทธ์การจัดการแปลง
2. **สังเคราะห์ (Synthesize)** และออกแบบระบบปฏิทินทางชีววิทยาพืช (Agro-Phenological Calendar) บูรณาการร่วมกับเงื่อนไขอุตุนิยมวิทยาเพื่อการควบคุมแบบแม่นยำสูง
3. **ออกแบบ (Design)** และทดสอบอัลกอริทึมระบบแจ้งเตือนภัยล่วงหน้า (Early Warning System - EWS) โดยมีลอจิกการคัดกรองสัญญาณเตือนที่ลดข้อผิดพลาดแบบเตือนหลอก (False Positives) และไม่เตือนภัยจริง (False Negatives)
4. **ประเมิน (Evaluate)** ความเป็นจริยธรรม ความโปร่งใส และความปลอดภัยในการใช้ปัญญาประดิษฐ์ตัดสินใจแทนมนุษย์ตามหลักการปัญญาประดิษฐ์อธิบายได้ (Explainable AI - XAI)

## 5.1 ทฤษฎีระบบสนับสนุนการตัดสินใจในบริบทเกษตรดิจิทัล
ระบบสนับสนุนการตัดสินใจ (Decision Support System - DSS) คือสถาปัตยกรรมซอฟต์แวร์ประยุกต์เชิงวิทยาศาสตร์ที่ออกแบบมาเพื่อช่วยเหลือนักจัดการฟาร์มหรือเกษตรกรในการวิเคราะห์ข้อมูลที่ซับซ้อนและเปราะบาง เพื่อใช้ประกอบการตัดสินใจจัดการทรัพยากรภายในฟาร์มได้อย่างถูกต้องสูงสุด การเกิดขึ้นของ DSS มีวัตถุประสงค์หลักในการเปลี่ยนผ่านกระบวนทัศน์จาก **"การเกษตรตามสัญชาตญาณ" (Intuition-based Agriculture)** ไปสู่ **"การเกษตรตามหลักฐานวิทยาศาสตร์" (Evidence-based Agriculture)**

### โครงสร้างหลัก 3 องค์ประกอบของระบบ DSS เกษตรกรรมแม่นยำ:

```
  [ 1. ฐานข้อมูล (Data Engine) ] ──► [ 2. โมเดลการตัดสินใจ (Model Engine) ] ──► [ 3. ส่วนอินเตอร์เฟส (User Interface) ]
    - ข้อมูลสดจากเซ็นเซอร์             - ตรรกะฟัซซี / อัลกอริทึมกฎ             - แดชบอร์ดข้อมูลเชิงลึก
    - ข้อมูลพยากรณ์อากาศสด            - แบบจำลองสรีรวิทยาพืช                 - แถบแจ้งเตือน / LINE API
```

1. **เครื่องยนต์จัดการข้อมูล (Data Engine)**: ทำหน้าที่เก็บรวบรวม จัดการ และล้างข้อมูลอนุกรมเวลาภูมิอากาศ ทั้งจากเซ็นเซอร์ตรวจวัดอากาศจุลภาคภาคสนาม และข้อมูลพยากรณ์อากาศสดจากระบบคลาวด์ภายนอก
2. **เครื่องยนต์จำลองและตัดสินใจ (Model Engine)**: สมองส่วนประมวลผลที่บรรจุสมการทางคณิตศาสตร์ แบบจำลองจำลองพืช (Crop Models) และกฎปัญญาประดิษฐ์เชิงสัญลักษณ์ (Symbolic AI) ที่สกัดจากผู้เชี่ยวชาญด้านพืชสวนและสรีรวิทยาพืช
3. **ส่วนติดต่อผู้ใช้งาน (User Interface - UI)**: ช่องทางการสื่อสารที่แปลงรหัสผลลัพธ์ทางคณิตศาสตร์และชีววิทยาให้กลายเป็นคำเตือนหรือแผนภูมิการปฏิบัติการที่เข้าใจง่ายสำหรับเกษตรกร เช่น แดชบอร์ดหน้าจอสัมผัส หรือข้อความแจ้งเตือนอัจฉริยะทางสมาร์ทโฟน

## 5.2 การบูรณาการข้อมูลสภาพอากาศเข้ากับปฏิทินสรีรวิทยาพืช
การรับรู้เพียงข้อมูลสภาพภูมิอากาศปัจจุบันโดยไม่พิจารณาสภาพร่างกายของพืช (Plant Physiological Stage) เป็นแนวทางที่ไม่สามารถนำไปสู่เกษตรแม่นยำขั้นสูงได้ เนื่องจากพืชแต่ละสายพันธุ์มีความต้องการทรัพยากรน้ำ ธาตุอาหาร และอุณหภูมิที่เปลี่ยนไปโดยสิ้นเชิงในแต่ละช่วงวัฏจักรชีวิตพืช (Phenological Stages)

ตัวอย่างเงื่อนไขการควบคุมน้ำและธาตุอาหารตามปฏิทินสรีรวิทยาพืชเศรษฐกิจ:
* **ทุเรียนหมอนทอง**:
  * *ระยะไหลดอก/ตาดอกเริ่มพัฒนา*: พืชต้องการความแห้งแล้งอย่างต่อเนื่อง ดินต้องมีความชื้นต่ำสะสม ค่าดัชนีผลต่างความดันไอในอากาศ (Vapor Pressure Deficit - VPD) ควรอยู่ในระดับสูง หากประเมินผิดพลาดไปรดน้ำปริมาณสูงในช่วงนี้ พืชจะยกเลิกการออกดอกและผลิใบอ่อนแทน ทำให้สูญเสียโอกาสผลผลิตรอบปี
  * *ระยะติดผลอ่อน/ผลขยายขนาด*: ต้องการความชื้นดินที่สม่ำเสมอและค่อนข้างชื้นคงที่ที่ประมาณ 20-25% VWC เพื่อนำพาแคลเซียมและธาตุอาหารไปสร้างผนังเซลล์ผล หากปล่อยให้ดินแห้งสลับเปียก ผลทุเรียนจะแตกและสลัดผลอ่อนร่วงหล่นทั้งหมด
* **มังคุด**:
  * *ระยะสร้างแป้งและน้ำตาลรอบผลสุก*: ต้องการสภาพภูมิอากาศจุลภาคที่แห้งตัวลงเพื่อสะสมความหวานและสกัดเปลือกให้แข็งตึง หากเกิดฝนตกหนักสะสมในดิน ระบบรากจะดูดซึมน้ำปริมาณมหาศาลเฉียบพลัน แรงดันเต่งจะทำท่อน้ำยางภายในผลปริแตกจนเกิดยางไหลเปื้อนเนื้อเป็นโรคเนื้อแก้ว
* **ยางพารา**:
  * *ระยะผลัดใบประจำปี (Defoliation)*: พืชจะลดการผลิตน้ำยางอย่างเป็นธรรมชาติเพื่อถนอมน้ำและสารอาหาร การกรีดยางในช่วงนี้จะสร้างความเสียหายต่อท่อน้ำยางถาวร ระบบ DSS ต้องสั่งปิดกำหนดการกรีดยางโดยอ้างอิงข้อมูลดัชนีพื้นที่ใบพืช (Leaf Area Index - LAI) จากข้อมูลดาวเทียมหรือเซ็นเซอร์ตรวจวัดแสง

## 5.3 การออกแบบอัลกอริทึมระบบแจ้งเตือนภัยล่วงหน้า
ระบบแจ้งเตือนภัยล่วงหน้า (Early Warning System - EWS) มีหน้าที่ปกป้องผลผลิตล่วงหน้าก่อนเกิดความเสียหายทางกายภาพ ในทางวิศวกรรมปัญญาประดิษฐ์เชิงกฎ การสร้าง EWS ต้องรักษาสมดุลความแม่นยำเพื่อป้องกันความล้มเหลวเชิงการตัดสินใจ 2 รูปแบบหลัก:

```
  ┌────────────────────────────────────────────────────────────────────────┐
  │                            เกณฑ์ความผิดพลาดของ EWS                       │
  ├────────────────────────────────────────┬───────────────────────────────┤
  │       เตือนภัยทั้งที่ไม่มีภัยจริง          │       เกิดภัยจริงแต่ระบบเงียบเฉย       │
  │     (False Positives - เตือนหลอก)      │  (False Negatives - ภัยมาไม่เตือน)     │
  │  - เกษตรกรเกิดความเคยชินและเฉยชา          │  - เกิดความเสียหายต่อพืชผลทันที         │
  │  - ปิดกั้นความเชื่อมั่นในนวัตกรรมระยะยาว   │  - ไม่สามารถเตรียมการป้องกันล่วงหน้าได้   │
  └────────────────────────────────────────┴───────────────────────────────┘
```

การลดระดับข้อผิดพลาดทั้งสองรูปแบบทำได้โดยการใช้เทคนิค **"เกณฑ์หน้าต่างเวลาแบบมีเงื่อนไขร่วม" (Co-incident Window Thresholding)** เช่น แทนที่จะสั่งเตือนภัยลมพายุจากค่าความเร็วลมกระโชกเพียงจุดเดียว ให้กำหนดลอจิกเป็น:
> *ความเร็วลมเฉลี่ย 10 นาทีเกิน 45 กม./ชม. ร่วมกับความกดอากาศดิ่งต่ำลงมากกว่า 5 hPa ภายใน 1 ชั่วโมง และพืชอยู่ในช่วงกิ่งแขนงเริ่มหนักจากน้ำหนักผลทุเรียน*

## 5.4 จริยธรรมและการตรวจสอบความถูกต้องของระบบปัญญาประดิษฐ์
เมื่อระบบปัญญาประดิษฐ์อัจฉริยะได้รับการเชื่อมต่อโดยตรงเข้ากับระบบการทำงานกายภาพผ่านอินเทอร์เน็ตของสรรพสิ่ง (AIoT) เช่น การประมวลผล Logic Layer แล้วส่งสัญญาณไปสั่งเปิดปั๊มพ่นปุ๋ยเคมี หรือเปิดวาล์วปล่อยน้ำปริมาณมากโดยอัตโนมัติ คำถามเชิงจริยธรรมและความปลอดภัย (Agri-AI Ethics & Safety) จึงเกิดขึ้น: 
*หากระบบแนะนำปริมาณการปล่อยสารเคมีอารักขาพืชเกินความจำเป็นจนทำให้ดินปนเปื้อนพิษ หรือสั่งจ่ายน้ำปริมาณมหาศาลจนทำให้รากทุเรียนเน่าเสียหายตายยกแปลง ใครต้องเป็นผู้รับผิดชอบเชิงกฎหมายและเศรษฐศาสตร์?*

นวัตกร RBRU-Praneet จึงต้องออกแบบระบบบนพื้นฐาน 2 ประการหลัก:
1. **การควบคุมดูแลโดยมีมนุษย์ในวงจรตัดสินใจ (Human-in-the-loop)**: ออกแบบระบบสั่งการเชิงระดับขั้น โดยคำสั่งที่มีผลกระทบสูงมาก (High-risk Decisions เช่น การฉีดสารพ่นกำจัดโรคพืชทางเคมี หรือการเปลี่ยนเกณฑ์แผนจ่ายน้ำใหญ่) ระบบ DSS จะทำได้เพียงวิเคราะห์ แนะนำ และรอคอยให้เกษตรกรรูดยืนยันผ่านแอพพลิเคชั่นมือถือเท่านั้น
2. **ปัญญาประดิษฐ์อธิบายได้ (Explainable AI - XAI)**: อัลกอริทึมการคำนวณและประเมินผลของ AI จะต้องไม่เป็นแบบกล่องดำ (Black Box) ที่แสดงเพียงผลลัพธ์โดยไร้ที่มาที่ไป แต่ต้องมีท่อส่งประวัติการประมวลผล (Audit Trail) ที่พร้อมอธิบายแก่ผู้ใช้งานว่า "เพราะเหตุใดจึงแนะนำเช่นนี้" เช่น:
   > *"แนะนำให้สเปรย์น้ำพ่นหมอก เนื่องจากตรวจพบค่าอุณหภูมิอากาศปัจจุบันสูงเกิน 37.5°C สะสมนานกว่า 30 นาที ร่วมกับค่าความดันไอของน้ำในทรงพุ่มเบี่ยงเบนผิดปกติ"*

## 5.5 โค้ดคอมพิวเตอร์และแบบฝึกหัดทบทวน 3 ระดับ

เพื่อให้สอดคล้องกับยุทธศาสตร์การพัฒนาเกษตรกรอัจฉริยะ (RBRU-Praneet) โค้ดคอมพิวเตอร์และแบบฝึกหัดท้ายบทนี้ได้รับการพัฒนาและแบ่งโครงสร้างออกเป็น 3 ระดับชั้นเรียนอย่างเป็นระบบ:

### 🟩 ระดับที่ 1: มัธยมศึกษาตอนต้น (Junior High Level) - เน้นความเข้าใจพื้นฐาน
*ระบบตรรกะแจ้งเตือนฝนชะช่อดอกทุเรียนจำลอง* โค้ดนี้ใช้สำหรับการศึกษาการทำงานของโครงสร้างควบคุม `if-elif-else` ในระดับตรรกะซ้อนเบื้องต้น

#### 💻 โค้ดภาษา Python สำหรับมัธยมต้น:
```python
# โปรแกรมวิเคราะห์และเตือนภัยฝนชะดอกทุเรียนอย่างง่าย
# จัดทำโดย: นวัตกรน้อย RBRU-Praneet (ระดับมัธยมต้น)

# 1. ป้อนค่าสถานะจำลองของแปลงเพาะปลูก
phenology_stage = "ระยะไหลดอก" # ระยะสรีรวิทยาพืช
rain_probability = 85.0       # โอกาสฝนตกจากคลาวด์ (%)
air_temp = 32.5               # อุณหภูมิอากาศ (°C)

print("--- ระบบสนับสนุนการตัดสินใจ RBRU-Praneet (มัธยมต้น) ---")
print("ระยะการเติบโตปัจจุบัน:", phenology_stage)
print("โอกาสเกิดฝนตกจริง:", rain_probability, "%")

# 2. วิเคราะห์การรดน้ำและการป้องกันช่อดอก
if phenology_stage == "ระยะไหลดอก":
    if rain_probability > 75.0:
        print("วิเคราะห์ผล: [คำเตือนวิกฤต!] พืชอยู่ระยะไหลดอกและเสี่ยงฝนชะดอกอ่อน ให้ขึงตาข่ายกันฝนด่วน!")
        irrigation_command = "ปิดวาล์วน้ำทันที"
    else:
        print("วิเคราะห์ผล: [สภาพอากาศปกติ] ปล่อยให้หน้าดินแห้งตามสูตรสะสมความแห้งชักนำดอก")
        irrigation_command = "งดจ่ายน้ำชั่วคราว"
        
elif phenology_stage == "ระยะติดผลอ่อน":
    print("วิเคราะห์ผล: [ระยะต้องการน้ำสม่ำเสมอ] ห้ามปล่อยให้ดินแห้งเด็ดขาด")
    irrigation_command = "เปิดน้ำหยดปริมาณปานกลางรอบเช้า"
    
else:
    print("วิเคราะห์ผล: ระยะพืชอื่น ๆ ทำงานตามตารางปกติ")
    irrigation_command = "ทำงานอัตโนมัติตามรอบเวลา"

print("คำสั่งสั่งการไปยังอุปกรณ์ภาคสนาม:", irrigation_command)
print("=====================================================")
```

#### ❓ แบบฝึกหัดทบทวนระดับมัธยมต้น:
1. จงอธิบายการประมวลผลโค้ดข้างต้นในกรณีที่ผู้ใช้เปลี่ยนค่าตัวแปร `phenology_stage` เป็น `"ระยะติดผลอ่อน"` และอุณหภูมิอากาศกระโดดไปสูงเกิน 39 องศาเซลเซียส
2. ดัดแปลงโปรแกรมโดยเพิ่มคำสั่งเงื่อนไขการตรวจสอบกรณีพืชเป้าหมายคือ **"มังคุด"** ในช่วงเก็บเกี่ยว หากมีฝนตกชุก ให้เตือนภัยสภาวะเนื้อแก้วยางไหลอ่อน ๆ

### 🟨 ระดับที่ 2: มัธยมศึกษาตอนปลาย (Senior High Level) - เน้นเครื่องยนต์กฎและการบูรณาการปฏิทินพืช
*ระบบวิเคราะห์ตรรกะที่ปรึกษาการเกษตรด้วยเครื่องยนต์กฎผู้เชี่ยวชาญแบบพจนานุกรม* โค้ดนี้สอนแนวคิดการแยกตรรกะเกณฑ์ความปลอดภัยออกจากตัวแปรสภาวะสิ่งแวดล้อม และประยุกต์ใช้ฟังก์ชันในการคำนวณคำสั่งปฏิบัติการ

#### 💻 โค้ดภาษา Python สำหรับมัธยมปลาย:
```python
# ระบบปัญญาประดิษฐ์เชิงตรรกะแจ้งเตือนการรดน้ำทุเรียน
# จัดทำโดย: นวัตกรสายวิทย์ RBRU-Praneet (ระดับมัธยมปลาย)

# 1. กำหนดโครงสร้างข้อมูลเกณฑ์การให้น้ำทุเรียนแต่ละระยะสรีรวิทยา (Dynamic Threshold Dictionary)
DURIAN_GROWTH_RULES = {
    "ตาดอกพัฒนา": {
        "min_moisture": 10.0,
        "max_moisture": 16.0,
        "action_desc": "จ่ายน้ำปริมาณน้อยพิเศษเพื่อกระตุ้นก้านดอก ไม่รดน้ำขัง"
    },
    "ผลอ่อนขยายขนาด": {
        "min_moisture": 20.0,
        "max_moisture": 28.0,
        "action_desc": "รดน้ำชุ่มปานกลางสม่ำเสมอ รักษาระดับน้ำดินป้องผลร่วงหล่น"
    },
    "เก็บเกี่ยวผลผลิต": {
        "min_moisture": 12.0,
        "max_moisture": 18.0,
        "action_desc": "ปล่อยงดน้ำชั่วคราวก่อนตัดเพื่อสะสมความหวานและสกัดเนื้อแห้ง"
    }
}

def execute_dss_engine(stage, current_soil_moisture):
    """
    ฟังก์ชันวิเคราะห์ระบบสนับสนุนการตัดสินใจอิงปฏิทินสรีรวิทยา
    """
    # ตรวจสอบว่ามีข้อมูลกฎรองรับระยะการเติบโตนี้หรือไม่
    if stage not in DURIAN_GROWTH_RULES:
        return "แจ้งเตือน: [ไม่พบข้อมูลกฎ] รันโปรแกรมรดน้ำแบบดั้งเดิม 30 นาที", "UNSUPPORTED_STAGE"
        
    rule = DURIAN_GROWTH_RULES[stage]
    low_limit = rule["min_moisture"]
    high_limit = rule["max_moisture"]
    recommended_action = rule["action_desc"]
    
    print(f"--- กระบวนการวิเคราะห์เกณฑ์สรีรวิทยา: {stage} ---")
    print(f"ขีดจำกัดความชื้นดินเหมาะสม: {low_limit}% - {high_limit}% | ค่าดินปัจจุบัน: {current_soil_moisture}%")
    
    # คำนวณความเบี่ยงเบนทางกายภาพ
    if current_soil_moisture < low_limit:
        alert_msg = f"แจ้งเตือน: [ดินแห้งเกินไป] ตรวจพบวิกฤตรากขาดน้ำ! คำสั่งสั่งการ: {recommended_action}"
        action_code = "VALVE_OPEN"
    elif current_soil_moisture > high_limit:
        alert_msg = f"แจ้งเตือน: [น้ำขังเกินเกณฑ์] เสี่ยงเกิดโรครากเน่าโคนเน่า! คำสั่งสั่งการ: สั่งปิดน้ำทันที"
        action_code = "VALVE_CLOSE"
    else:
        alert_msg = f"วิเคราะห์ผล: [ระดับความชื้นดินสมดุลดี] ดำเนินการเฝ้าระวังต่อไป"
        action_code = "SYSTEM_IDLE"
        
    return alert_msg, action_code

# --- ทดลองวิจัยรันระบบจำลองสถานการณ์สวนทุเรียนหมอนทอง ---
test_stage = "ผลอ่อนขยายขนาด"
test_moisture = 15.5 # ความชื้นดินต่ำกว่าช่วงเหมาะสมของผลอ่อนขยายขนาด (20.0 - 28.0)

alert_output, cmd_output = execute_dss_engine(test_stage, test_moisture)
print("\nผลลัพธ์แจ้งเตือนทางหน้าจอระบบ:")
print(alert_output)
print("รหัสคำสั่งอุปกรณ์ IoT:", cmd_output)
```

#### ❓ แบบฝึกหัดทบทวนระดับมัธยมปลาย:
1. หากแปลงปลูกของท่านเปลี่ยนชนิดพืชเป็น **"ข้าวอัจฉริยะ"** จงระบุแนวทางการดัดแปลงพจนานุกรม `DURIAN_GROWTH_RULES` ให้ครอบคลุมระยะสรีรวิทยาของข้าว เช่น "ระยะข้าวน้ำนม" และ "ระยะสะสมแป้ง"
2. จงอธิบายผลเสียของการรวบตรรกะเขียนค่าระดับความชื้นเหมาะสมลงไปในชุดคำสั่ง `if` ตรงๆ โดยไม่มีการดึงค่าผ่านตัวแปลพจนานุกรมในแง่ของสถาปัตยกรรมซอฟต์แวร์และการนำโค้ดกลับมาใช้ซ้ำ

### 🟦 ระดับที่ 3: อุดมศึกษา (Higher Education Level) - เน้นระบบปัญญาประดิษฐ์อธิบายได้และความปลอดภัยเชิงสถิติขั้นสูง
*เครื่องยนต์ปัญญาประดิษฐ์สนับสนุนการตัดสินใจระดับอุตสาหกรรมพร้อมการป้องกันข้อมูลและการให้เหตุผลเบื้องหลังการตัดสินใจ (XAI)* โค้ดนี้มีโครงสร้างคลาสที่แข็งแกร่ง มีระบบตรวจสอบพฤติกรรม ตรวจสอบและให้เหตุผลการคำนวณอย่างเป็นระบบอธิบายแก่ผู้ควบคุมแปลง

#### 💻 โค้ดภาษา Python สำหรับอุดมศึกษา:
```python
# ระบบที่ปรึกษาอัจฉริยะวิเคราะห์ความปลอดภัยและการให้น้ำอัจฉริยะระดับอุตสาหกรรม (Industrial-Grade Agri-XAI Advisor)
# พัฒนาขึ้นภายใต้ยุทธศาสตร์ RBRU-Praneet (ระดับอุดมศึกษา)

import json
import time

class ExplainableAgriAdvisor:
    def __init__(self, crop_id, target_crop):
        self.crop_id = crop_id
        self.target_crop = target_crop
        self.decision_log = []
        
    def infer_irrigation_strategy(self, sensor_payload, phenology_stage, weather_forecast):
        """
        เครื่องยนต์ประมวลผลตรรกะแบบให้เหตุผลเชิงวิเคราะห์ย้อนกลับ (Prescriptive & Explainable AI Engine)
        """
        # 1. โหลดข้อมูลสิ่งแวดล้อมภาคสนาม
        soil_moisture = sensor_payload.get("soil_moisture")
        air_temp = sensor_payload.get("air_temp")
        air_humidity = sensor_payload.get("humidity")
        
        # ปรับความต้านทานระบบตรวจสอบความสมบูรณ์ของเซ็นเซอร์
        if None in [soil_moisture, air_temp, air_humidity]:
            return self._fallback_safety_protocol("ตรวจพบช่องข้อมูลเซ็นเซอร์ชำรุดเสียหาย (Null Value)")
            
        # 2. คำนวณค่าผลต่างความดันไอในอากาศ (Vapor Pressure Deficit - VPD) เพื่อประเมินสรีรวิทยาปากใบพืช
        # สมการประมาณการอย่างง่ายของค่าแรงดันอิ่มตัวและแรงดันไอจริง
        es = 0.6108 * np_exp((17.27 * air_temp) / (air_temp + 237.3)) if 'np_exp' in globals() else 0.6108 * (2.718 ** ((17.27 * air_temp) / (air_temp + 237.3)))
        ea = es * (air_humidity / 100.0)
        vpd = es - ea # หน่วยเป็นกิโลปาสคาล (kPa)
        
        # 3. เริ่มประมวลกฎการอนุมานสนับสนุนการตัดสินใจ
        decision = "HOLD"
        explanations = []
        confidence_score = 1.0
        
        # กรณีศึกษาการให้น้ำระยะดอกบานถึงผลอ่อนทุเรียน
        if phenology_stage == "ระยะสลัดดอกผลอ่อน":
            if soil_moisture < 20.0:
                decision = "ACTIVATE_PULSE_IRRIGATION"
                explanations.append(f"ความชื้นดินปัจจุบันต่ำเกินไป ({soil_moisture}%) เสี่ยงการสลัดผลอ่อนร่วงหล่น")
                confidence_score *= 0.95
            
            # ตรวจสอบความเครียดความร้อนของพืชจาก VPD และอากาศร้อน
            if vpd > 2.0:
                decision = "ACTIVATE_MISTING_SYSTEM"
                explanations.append(f"ค่า VPD สูงสุดขีด ({vpd:.2f} kPa) พืชมีความเค้นความร้อนสูงและเริ่มปิดตาดอกปากใบพืช สั่งพ่นหมอกลดอุณหภูมิ")
                confidence_score *= 0.98
                
            if weather_forecast.get("rain_probability_3h", 0.0) > 80.0:
                # ปรับแกะตรรกะคลาดเคลื่อนชนกัน (Conflict Resolution Rule)
                decision = "ABORT_ALL_WATERING"
                explanations.append(f"ความน่าจะเป็นของฝน 3 ชั่วโมงข้างหน้าสูงเกินเกณฑ์ ({weather_forecast['rain_probability_3h']}%) งดให้น้ำป้องกันรากท่วมขัง")
                confidence_score *= 0.90
                
        else:
            decision = "DEFAULT_SCHEDULE"
            explanations.append("สรีรวิทยาพืชอยู่นอกเกณฑ์เฝ้าระวังสูงสุด ทำงานตามตารางปกติ")
            
        # 4. บันทึกประวัติและเหตุผล (Audit Trail & XAI Log)
        timestamp = time.strftime("%Y-%m-%d %H:%M:%S", time.localtime())
        log_entry = {
            "timestamp": timestamp,
            "crop_id": self.crop_id,
            "stage": phenology_stage,
            "inputs": {"soil_moisture": soil_moisture, "vpd": round(vpd, 3), "rain_prob": weather_forecast.get("rain_probability_3h")},
            "output_decision": decision,
            "explanations": explanations,
            "confidence": round(confidence_score, 3)
        }
        self.decision_log.append(log_entry)
        return log_entry
        
    def _fallback_safety_protocol(self, reason):
        """ระบบความปลอดภัยเมื่อเกิดการชำรุดเชิงระบบ"""
        return {
            "status": "EMERGENCY_FALLBACK",
            "decision": "HUMAN_OVERRIDE_REQUIRED",
            "explanations": [f"ระบบ DSS ทำการยกเลิกคำสั่งอัตโนมัติ เนื่องจาก: {reason}"],
            "confidence": 0.0
        }

# --- รันการประเมินผลระดับอุตสาหกรรมในศูนย์บริหาร RBRU-Praneet ---
advisor = ExplainableAgriAdvisor(crop_id="ORCHARD-BLOCK-D4", target_crop="ทุเรียนหมอนทอง")

# จำลองสถานการณ์เซ็นเซอร์วัดค่าและพยากรณ์อากาศที่ส่งมาจาก Sensory Layer
sensor_data = {"soil_moisture": 18.2, "air_temp": 38.5, "humidity": 45.0} # สภาพดินค่อนข้างแห้งและอากาศร้อนจัด
forecast_data = {"rain_probability_3h": 15.0}                             # โอกาสฝนตกต่ำมาก

# ประมวลผลลัพธ์
dss_result = advisor.infer_irrigation_strategy(
    sensor_payload=sensor_data,
    phenology_stage="ระยะสลัดดอกผลอ่อน",
    weather_forecast=forecast_data
)

print("=== สรุปผลการวินิจฉัยจากระบบสนับสนุนการตัดสินใจอัจฉริยะ (XAI Output) ===")
print(f"สิทธิ์ดำเนินการสั่งการ: {dss_result['output_decision']}")
print(f"ค่าสัมประสิทธิ์ความมั่นใจระบบ: {dss_result['confidence'] * 100:.1f}%")
print("\n[คำอธิบายที่ตรวจสอบได้จากปัญญาประดิษฐ์ - Explainable AI Reasons]:")
for idx, explanation in enumerate(dss_result["explanations"]):
    print(f"  {idx + 1}. {explanation}")
print("=========================================================================")
```

#### ❓ แบบฝึกหัดทบทวนระดับอุดมศึกษา:
1. จงวิเคราะห์ความสมบูรณ์เชิงสถาปัตยกรรมของท่อส่งระบบการรันคำสั่ง หากต้องนำระบบ `ExplainableAgriAdvisor` ไปติดตั้งเป็นบริการขนาดจิ๋ว (Microservice) บนคลาวด์ร่วมกับการตรวจสอบความปลอดภัยข้อมูล API Gateway และ Line Notify Messaging
2. ออกแบบวิธีการเรียนรู้และอัพเกรดเงื่อนไขการตัดสินใจโดยการผสานแนวคิดเครือข่ายความเชื่อแบบเบย์เซียน (Bayesian Belief Network) เพื่อชดเชยน้ำหนักของค่าความมั่นใจเมื่อข้อมูลพยากรณ์ฝนมีความผันผวนบ่อยครั้ง
