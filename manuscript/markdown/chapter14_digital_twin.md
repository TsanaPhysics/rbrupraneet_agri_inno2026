## 🎯 วัตถุประสงค์การเรียนรู้ (Learning Objectives)
เมื่อศึกษาบทนี้จบแล้ว ผู้เรียนจะสามารถ:
1. **วิเคราะห์ (Analyze)** ข้อดีและข้อจำกัดของสถาปัตยกรรมคลาวด์แบบกระจายตัวและเปรียบเทียบประสิทธิภาพระหว่างฐานข้อมูลอนุกรมเวลากับฐานข้อมูลเชิงสัมพันธ์ในงานประมวลผลข้อมูลเกษตรดิจิทัล
2. **ประเมิน (Evaluate)** และออกแบบประสบการณ์ของผู้ใช้สำหรับระบบแดชบอร์ดควบคุมการเกษตรอัจฉริยะ ให้มีลักษณะวิเคราะห์ข้อมูลเชิงลึกที่พร้อมนำไปปฏิบัติงานและเหมาะสมกับพฤติกรรมของเกษตรกรในสนาม
3. **อนุพัทธ์ (Derive)** และประยุกต์ใช้สมการการคายระเหยน้ำอ้างอิงตามมาตรฐาน FAO-56 Penman-Monteith เพื่อสร้างแบบจำลองสมดุลน้ำในดินระดับพลศาสตร์
4. **สังเคราะห์ (Synthesize)** สถาปัตยกรรมคู่แฝดดิจิทัลที่เชื่อมต่อข้อมูลทางกายภาพจากอุปกรณ์รับส่งสัญญาณจริงเข้าสู่โลกเสมือนแบบเวลาจริง
5. **สร้างสรรค์ (Create)** โปรแกรมคอมพิวเตอร์จำลองสภาวะแวดล้อมและพืชพรรณอัจฉริยะ 3 ระดับการเรียนรู้ (มัธยมศึกษาตอนต้น, มัธยมศึกษาตอนปลาย, และอุดมศึกษา) เพื่อวิเคราะห์สถานะและการบริหารจัดการทรัพยากรน้ำของพืชเศรษฐกิจไทย

## 14.1 สถาปัตยกรรมคลาวด์สำหรับการเกษตรแบบกระจายตัว (Distributed Cloud Architecture)

ในระบบเกษตรอัจฉริยะยุคเสรีภาพทางข้อมูล โหนดเซ็นเซอร์และสถานีตรวจวัดอากาศนับพันตัวที่กระจายตัวอยู่ในแปลงเพาะปลูกทุเรียนและมังคุดของประเทศจะสร้างชุดข้อมูลขึ้นมาอย่างมหาศาลตลอด 24 ชั่วโมง การส่งผ่าน จัดเก็บ และประมวลผลข้อมูลเหล่านี้จำเป็นต้องพึ่งพาระบบการประมวลผลแบบคลาวด์ (Cloud Computing) ที่มีสถาปัตยกรรมแบบกระจายตัว (Distributed Architecture) เพื่อรองรับขีดจำกัดความยืดหยุ่นในการขยายตัว (Scalability) และรักษาเสถียรภาพยามเกิดเหตุขัดข้องทางโครงข่ายไฟฟ้าหรือสัญญาณเซลลูลาร์ในสนาม

### 14.1.1 สถาปัตยกรรมข้อมูลแบบแบ่งชั้น (Layered Data Architecture)
เพื่อรองรับระบบคู่แฝดดิจิทัล (Digital Twin) คลาวด์จะต้องจัดชั้นข้อมูลออกเป็น 3 ระดับหลัก เพื่อจัดระเบียบข้อมูลตามความด่วนของการตอบสนองและปริมาณพื้นที่จัดเก็บ:

```
        ┌──────────────────────────────────────────────────────────┐
        │                 ชั้นแอปพลิเคชันคลาวด์                    │
        │    - การสร้างโมเดลทำนายโรค  - การจำลองคู่แฝดดิจิทัลระยะยาว   │
        └────────────────────────────┬─────────────────────────────┘
                                     │ (API Gateway)
        ┌────────────────────────────▼─────────────────────────────┐
        │                  ชั้นฐานข้อมูลและการประมวลผล             │
        │    - ฐานข้อมูลอนุกรมเวลา    - ฐานข้อมูลไม่เชิงสัมพันธ์       │
        └────────────────────────────┬─────────────────────────────┘
                                     │ (Data Ingestion Pipeline)
        ┌────────────────────────────▼─────────────────────────────┐
        │                ชั้นเครือข่ายขอบและเกตเวย์                 │
        │    - การคัดกรองสัญญาณรบกวน  - การบีบอัดข้อมูลก่อนส่งคลาวด์   │
        └──────────────────────────────────────────────────────────┘
```

1. **ชั้นนำเข้าข้อมูลเวลาจริง (Real-time Ingestion Layer):** ใช้โปรโตคอลความหน่วงต่ำ เช่น MQTT หรือ Apache Kafka ในการรับทราฟฟิกข้อมูลที่สตรีมมาจากอุปกรณ์เกตเวย์ในสนาม ข้อมูลชั้นนี้จะไม่มีการจัดเก็บระยะยาว แต่เน้นการส่งผ่านอย่างรวดเร็วไปยังระบบตอบสนองฉับพลัน (เช่น การเตือนภัยปั๊มน้ำทุเรียนขัดข้อง)
2. **ชั้นประมวลผลและจัดเก็บเชิงวิเคราะห์ (Analytics & Storage Layer):** แบ่งออกเป็นโครงสร้างข้อมูลดิบอนุกรมเวลาสำหรับการคำนวณเชิงสถิติย้อนหลัง และข้อมูลเชิงสัมพันธ์สำหรับจัดเก็บโปรไฟล์ประวัติของแปลงปลูก สภาพดิน อัตราการเจริญเติบโตของพืช และตารางการบำรุงรักษา
3. **ชั้นบริการข้อมูลและสถาปัตยกรรมไมโครเซอร์วิส (API & Microservices Layer):** ทำหน้าที่เปิดบริการข้อมูลที่ประมวลผลแล้วผ่านจุดเชื่อมต่อประสานโปรแกรม (Application Programming Interface หรือ API) เพื่อให้หน้าต่างแสดงผลของผู้ใช้งาน หรือโมเดลฟิสิกส์ในคู่แฝดดิจิทัลดึงไปสร้างภาพจำลองต่อไป

> [!NOTE]
> ข้อดีอันโดดเด่นของสถาปัตยกรรมแบบกระจายตัวคือ "ความทนทานต่อความเสียหายแบบแบ่งส่วน" (Fault Tolerance) หากฐานข้อมูลส่วนใดส่วนหนึ่งหยุดทำงาน อุปกรณ์เกตเวย์ในสนามจะยังคงเก็บข้อมูลไว้ในหน่วยความจำขอบทางกายภาพ (Edge Storage) และทำการประสานเวลาข้อมูล (Synchronization) เข้าสู่คลาวด์ทันทีที่การเชื่อมต่อกลับคืนมา ซึ่งสำคัญมากต่อแปลงปลูกทุเรียนป่าละอูหรือยางพาราบนพื้นที่ภูเขาสูงชันที่สัญญาณสื่อสารขาดหายได้ง่าย

## 14.2 ฐานข้อมูลอนุกรมเวลา (Time-series Databases) vs. ฐานข้อมูลทั่วไป

ข้อมูลจากอุปกรณ์เกษตรไอโอทีมีคุณลักษณะที่เฉพาะตัว คือ ข้อมูลทุกระเบียนที่หลั่งไหลเข้ามาจะมีเวลาประทับ (Timestamp) กำกับเสมอ เช่น แรงดันไฟฟ้า ความชื้นดิน ความชื้นสัมพัทธ์ในอากาศ และความเข้มแสง ข้อมูลลักษณะนี้เรียกว่า "ข้อมูลอนุกรมเวลา" (Time-series Data) ซึ่งมีความแตกต่างอย่างสิ้นเชิงจากข้อมูลธุรกิจเชิงพาณิชย์ทั่วไป

### 14.2.1 เปรียบเทียบสถาปัตยกรรมและโครงสร้างข้อมูล
การเลือกประเภทระบบจัดการฐานข้อมูลมีผลอย่างมีนัยสำคัญต่อความหน่วงของระบบและความสามารถในการประมวลผลข้อมูลประวัติศาสตร์ขนาดเทราไบต์ (Terabyte) เพื่อวิเคราะห์ผลผลิตมังคุดหรือปริมาณน้ำยางพาราเฉลี่ยรายปี:

| คุณลักษณะ | ฐานข้อมูลเชิงสัมพันธ์ (Relational Database - เช่น MySQL / PostgreSQL) | ฐานข้อมูลอนุกรมเวลา (Time-series Database - เช่น InfluxDB / TimescaleDB) |
| :--- | :--- | :--- |
| **โครงสร้างข้อมูลหลัก** | ตารางที่มีแถวและคอลัมน์ (Table-based) เชื่อมโยงกันด้วยคีย์หลักและคีย์นอก | เมทริกซ์เวลาประทับ (Timestamp) ร่วมกับชุดข้อมูลการวัด (Measurement) และป้ายกำกับ (Tags) |
| **การประมวลผลการบันทึก** | รองรับ ACID Transaction แบบละเอียด ทำดัชนีคีย์หลัก เหมาะกับข้อมูลที่ไม่ซ้ำซาก | ออกแบบมาเพื่อเน้นเขียนข้อมูลปริมาณมหาศาลอย่างรวดเร็ว (High-Write Throughput) |
| **ประสิทธิภาพการสืบค้นข้อมูลย้อนหลัง** | ช้าลงเป็นแบบทวีคูณเมื่อจำนวนแถวทะลุสิบล้านแถวเนื่องจากต้องทำดัชนีและเชื่อมตาราง (JOIN) | ค้นหาข้อมูลย้อนหลังตามขอบเขตเวลาได้อย่างรวดเร็วมาก (Time-range Query Optimized) |
| **ฟังก์ชันในตัว** | ฟังก์ชันทางคณิตศาสตร์ทั่วไป (SUM, AVG) ไม่มีฟังก์ชันการย่อขนาดข้อมูลตามกรอบเวลาโดยตรง | มีฟังก์ชันลดขนาดข้อมูลในตัว (Downsampling) เช่น การจับกลุ่มหาค่าเฉลี่ยรายชั่วโมงจากระดับวินาทีในคำสั่งเดียว |
| **การใช้พื้นที่จัดเก็บ** | ขนาดใหญ่ เนื่องจากมีการทำดัชนีหลายมิติและบันทึกข้อมูลโครงสร้างคงที่ | บีบอัดข้อมูลสูงมาก (เช่น อัลกอริทึม Gorilla compression) ประหยัดพื้นที่ได้มากกว่า 5-10 เท่า |

### 14.2.2 กลไกการย่อขนาดข้อมูล (Downsampling) และนโยบายการเก็บรักษาข้อมูล (Retention Policy)
ในระบบฐานข้อมูลอนุกรมเวลาประสิทธิภาพสูง เช่น InfluxDB จะมีกลไกกำหนดให้ลบข้อมูลดิบความละเอียดสูง (เช่น ข้อมูลความชื้นดินที่ส่งทุกๆ 10 วินาที) ที่เก่ากว่า 30 วันทิ้งโดยอัตโนมัติ แต่ก่อนจะลบ ระบบจะรันกระบวนการย่อขนาดข้อมูลเพื่อคำนวณหาค่าเฉลี่ย ค่าสูงสุด และค่าต่ำสุดรายชั่วโมง แล้วนำไปบันทึกในถังข้อมูลระยะยาว (Long-term Bucket) แทน

คุณสมบัตินี้ช่วยให้คู่แฝดดิจิทัลสามารถคำนวณสถิติน้ำย้อนหลังรายปีของแปลงปลูกส้มโอทับทิมสยามได้โดยไม่ทำให้เซิร์ฟเวอร์เกิดสภาวะคอขวดด้านความจุและการอ่านเขียนข้อมูล

## 14.3 การสร้างแดชบอร์ดแสดงผล (Dashboard Visualization) และ User Experience

เป้าหมายสูงสุดของการจัดการข้อมูลเกษตรดิจิทัลไม่ใช่การเก็บข้อมูลให้มากที่สุด แต่คือการแปลงข้อมูลดิบให้กลายเป็น "ข้อมูลเชิงลึกที่พร้อมนำไปปฏิบัติงาน" (Actionable Insights) ผ่านแดชบอร์ด (Dashboard) ที่ตอบสนองการรับรู้และส่งมอบประสบการณ์ของผู้ใช้ (User Experience หรือ UX) ที่เหมาะสมกับธรรมชาติของกลุ่มเกษตรกรและผู้จัดการฟาร์ม

### 14.3.1 หลักการออกแบบการจัดวางข้อมูลตามลำดับความสำคัญ (Information Hierarchy)
แดชบอร์ดที่ดีควรแสดงผลตามหลักการ "ภาพรวมระดับมหภาค สภาพแบบเจาะลึก และข้อมูลดิบยามต้องการ" (Overview first, zoom and filter, then details on-demand):

```
┌──────────────────────────────────────────────────────────────────┐
│ [ระดับที่ 1: สถานะด่วน]                                           │
│ ดัชนีความเตลิดระบบน้ำ: ปกติ  │ ปริมาณน้ำคงเหลือในอ่าง: 85%  │ สัญญาณขัดข้อง: 0  │
├──────────────────────────────────────────────────────────────────┤
│ [ระดับที่ 2: แบบจำลองคู่แฝดดิจิทัลและกราฟแนวโน้ม]                      │
│ ┌───────────────────────────────┐ ┌────────────────────────────┐ │
│ │                               │ │  กราฟแนวโน้มความชื้นดิน 24ชม.│ │
│ │       โมเดล 3 มิติเชิงกายภาพ    │ │  100% ───────────────────  │ │
│ │       แสดงระนาบความชุ่มชื้นดิน   │ │   50% ───\/\─────────────  │ │
│ │       ของต้นทุเรียนแต่ละโซน   │ │    0% ───────────────────  │ │
│ └───────────────────────────────┘ └────────────────────────────┘ │
├──────────────────────────────────────────────────────────────────┤
│ [ระดับที่ 3: ตารางข้อมูลเซ็นเซอร์แยกตามโหนดและสถานะเปิดปิดวาล์วน้ำ]       │
└──────────────────────────────────────────────────────────────────┘
```

1. **ระดับที่ 1: การรับรู้สถานการณ์ในทันที (Situational Awareness Indicators):** ใช้เกจ (Gauge) หรือสีสัญญาณไฟจราจรที่สะดุดตา เพื่อแสดงว่าระบบในภาพรวมปกติหรืออันตราย เช่น หากความชื้นดินของต้นทุเรียนลดลงต่ำกว่า $25\%$ ซึ่งเป็นจุดเหี่ยวแห้งชั่วคราว ระบบต้องแสดงสัญลักษณ์สีแดงกะพริบแจ้งเตือนทันที
2. **ระดับที่ 2: กราฟแนวโน้มพลศาสตร์ (Dynamic Trend Graphs):** หลีกเลี่ยงกราฟเส้นสับสนซ้อนทับกันหลายสิบเส้น ให้ใช้ตัวเลือกระบุแปลงย่อย และเน้นการเทียบเคียงระหว่างค่าจริงกับค่าเกณฑ์วิกฤตความชื้นในดิน
3. **ระดับที่ 3: แผงควบคุมสั่งการที่ชัดเจน (Clear Interactive Controls):** การกดวาล์วน้ำแบบสวิตช์ปิด-เปิด ดิจิทัล ต้องมีการแสดงสถานะการโต้ตอบกลับ (Visual Feedback) ว่าคำสั่งได้เดินทางไปถึงบอร์ดไมโครคอนโทรลเลอร์ปลายทางในสนามแล้วเพื่อป้องกันการกดสั่งซ้ำซ้อน

### 14.3.2 จิตวิทยาการรับรู้และบริบทการใช้งานจริง (Contextual Responsive UX)
* **การออกแบบเพื่อใช้กลางแจ้ง (Outdoor Usability):** เกษตรกรมักดูแดชบอร์ดผ่านโทรศัพท์มือถือขณะอยู่กลางแดดจัดในแปลงปลูก ดังนั้น อัตราส่วนความคมชัดของสีตัวอักษรกับพื้นหลัง (Color Contrast Ratio) ต้องสูงกว่ามาตรฐานปกติ (แนะนำที่ $7:1$ ขึ้นไปตามแนวทาง WCAG) และต้องหลีกเลี่ยงการใช้ปุ่มกดขนาดเล็กที่กดหยิบจับยากเมื่อนิ้วมือเปียกชื้นหรือเปื้อนดิน
* **การสื่อความหมายแบบเข้าใจง่าย (Simplified Metrics):** แทนที่จะแสดงค่าแรงดันไฟฟ้าของเซ็นเซอร์เป็นมิลลิโวลต์ ($mV$) หรือแสดงค่าการคายน้ำในหน่วยมิลลิเมตรต่อชั่วโมง ซึ่งสร้างความสับสน ให้แปลงผลลัพธ์เป็นคำอธิบายที่เข้าใจง่าย เช่น "ระดับความเครียดน้ำของต้นมังคุด: ปกติ/เฝ้าระวัง/อันตราย" หรือ "อัตราการใช้น้ำของทุเรียนเทียบเท่ากับน้ำ $15$ ลิตรต่อต้นในวันนี้"

## 14.4 มโนทัศน์เรื่องคู่แฝดดิจิทัล (Digital Twin Concepts) ในฟาร์มอัจฉริยะ

มโนทัศน์เรื่อง "คู่แฝดดิจิทัล" คือการสร้างตัวแทนเสมือนจริงในมิติดิจิทัลของวัตถุทางกายภาพในโลกจริง โดยในมิติการเกษตรอัจฉริยะ คู่แฝดดิจิทัลไม่ได้เป็นเพียงแค่แผนภาพกราฟิก 3 มิติที่สวยงามเท่านั้น แต่คือ "เครื่องยนต์จำลองพลศาสตร์เคมีและฟิสิกส์" (Biophysical Simulation Engine) ที่ทำงานล้อไปกับนาฬิกาจริงของธรรมชาติ

### 14.4.1 สถาปัตยกรรมระบบคู่แฝดดิจิทัลเกษตรกรรม
คู่แฝดดิจิทัลระดับอุตสาหกรรมจะเชื่อมประสานข้อมูลสามมิติเข้าด้วยกัน:

```
    ┌──────────────────────┐               ┌──────────────────────┐
    │   สภาพจริงทางกายภาพ  │               │    คู่แฝดเสมือนดิจิทัล   │
    │  (Physical Orchard)  │               │    (Digital Twin)    │
    │                      │               │                      │
    │  - ดิน อากาศ แดด ฝน   ├──────────────>│  - สมการฟิสิกส์จำลอง  │
    │  - การไหลเวียนของน้ำ  │  (ส่งข้อมูล)  │  - ทำนายล่วงหน้า 24 ชม.│
    │  - ต้นไม้คายน้ำจริงๆ  │               │  - คำนวณสมดุลน้ำพืช  │
    │                      │<──────────────┤                      │
    └──────────────────────┘  (ป้อนกลับน้ำ) └──────────────────────┘
```

1. **กายภาพจริง (Physical Entity):** ต้นพืช ดิน อากาศ ระบบส่งน้ำ และระบบระบายน้ำในสวนจริง
2. **ตัวแทนเสมือน (Virtual Twin):** แบบจำลองคณิตศาสตร์ที่เก็บโครงสร้างทางพฤกษศาสตร์ อัตราส่วนการดูดน้ำผ่านราก และสภาวะการสะสมคาร์โบไฮเดรตในใบพืช
3. **การไหลเวียนข้อมูลแบบสองทิศทาง (Bidirectional Data Flow):**
   * **ขาขึ้น (Upward Stream):** ข้อมูลเซ็นเซอร์จากฟิลด์ส่งผ่านคลาวด์เพื่อไปปรับปรุงตัวแปรภายในแบบจำลอง (Calibration on-the-fly) ให้แม่นยำล้อกับพฤติกรรมจริงของต้นไม้ ณ วันนั้น
   * **ขาลง (Downward Action):** แบบจำลองคู่แฝดจำลองอนาคตแล้วพบว่าดินจะแห้งเกินไปจนต้นทุเรียนเกิดความเครียดในอีก 3 ชั่วโมงข้างหน้า จึงส่งสัญญาณกลับมาสั่งเปิดปั๊มวาล์วน้ำในระบบกายภาพโดยอัตโนมัติ

### 14.4.2 ทฤษฎีความต้องการน้ำและการคายระเหยน้ำของพืช (Evapotranspiration Theory)

หัวใจสำคัญของการสร้างคู่แฝดดิจิทัลเพื่อจัดการน้ำพืชเศรษฐกิจ เช่น ทุเรียนหมอนทอง หรือยางพารา คือการคำนวณปริมาณการสูญเสียน้ำออกจากระบบแปลงปลูก ซึ่งประกอบด้วย 2 กระบวนการย่อย ได้แก่ การระเหยน้ำจากผิวดิน (Evaporation) และการคายน้ำผ่านปากใบพืช (Transpiration) รวมเรียกว่า **การคายระเหยน้ำ** (Evapotranspiration หรือ ET)

ในการคำนวณหาอัตราการคายระเหยน้ำอ้างอิงรายวัน ($ET_0$) มาตรฐานทางวิชาการและงานวิจัยระดับสากลยอมรับ **สมการของเพนแมน-มอนทีท** (Penman-Monteith Equation) ตามแนวทาง FAO-56 ซึ่งได้รับการพิสูจน์แล้วว่ามีความแม่นยำสูงเนื่องจากคำนึงถึงทั้งสมดุลพลังงานความร้อนและการถ่ายเทมวลอากาศโดยรอบทรงพุ่มพืช

#### สมการ FAO-56 Penman-Monteith เต็มรูปแบบ:

$$\lambda ET_0 = \frac{\Delta (R_n - G) + \rho_a c_p \frac{(e_s - e_a)}{r_a}}{\Delta + \gamma \left(1 + \frac{r_s}{r_a}\right)}$$

เพื่อความสะดวกในการประยุกต์ใช้งานด้วยโปรแกรมคอมพิวเตอร์ องค์การอาหารและเกษตรแห่งสหประชาชาติ (FAO) ได้แปลงสูตรข้างต้นให้อยู่ในรูปสมการมาตรฐานสำหรับคาบเวลา 1 วัน ดังนี้:

$$ET_0 = \frac{0.408 \Delta (R_n - G) + \gamma \frac{900}{T + 273} u_2 (e_s - e_a)}{\Delta + \gamma (1 + 0.34 u_2)}$$

โดยที่ตัวแปรและหน่วยวัดของแต่ละองค์ประกอบในสมการอธิบายได้ดังนี้:
* $ET_0$ คืออัตราการคายระเหยน้ำอ้างอิงของพืช (Reference Crop Evapotranspiration) มีหน่วยเป็นมิลลิเมตรต่อวัน ($\text{mm/day}$)
* $R_n$ คือรังสีสุทธิที่ผิวพืช (Net Radiation at the Crop Surface) มีหน่วยเป็นเมกะจูลต่อตารางเมตรต่อวัน ($\text{MJ/m}^2/\text{day}$) ซึ่งคำนวณจากความเข้มแสงอาทิตย์ที่ส่องลงมาลบด้วยรังสีสะท้อนกลับจากผิวดินและใบไม้
* $G$ คือความหนาแน่นของฟลักซ์ความร้อนในดิน (Soil Heat Flux Density) มีหน่วยเป็นเมกะจูลต่อตารางเมตรต่อวัน ($\text{MJ/m}^2/\text{day}$) โดยในรอบเวลา 24 ชั่วโมง ค่านี้มักมีค่าต่ำมากจนถือเป็นศูนย์ได้ ($G \approx 0$)
* $T$ คืออุณหภูมิอากาศเฉลี่ยรายวัน ณ ความสูง 2 เมตรจากผิวดิน มีหน่วยเป็นองศาเซลเซียส ($^\circ\text{C}$)
* $u_2$ คือความเร็วลมเฉลี่ยรายวัน ณ ความสูง 2 เมตรจากผิวดิน มีหน่วยเป็นเมตรต่อวินาที ($\text{m/s}$)
* $e_s$ คือความดันไออิ่มตัว (Saturation Vapor Pressure) มีหน่วยเป็นกิโลปาสกาล ($\text{kPa}$)
* $e_a$ คือความดันไอจริง (Actual Vapor Pressure) มีหน่วยเป็นกิโลปาสกาล ($\text{kPa}$)
* $e_s - e_a$ คือความดันไอพร่อง (Vapor Pressure Deficit หรือ VPD) ซึ่งสะท้อนถึงระดับความแห้งแล้งของอากาศ ยิ่งค่านี้สูง อัตราการคายน้ำของพืชจะยิ่งรวดเร็ว
* $\Delta$ คือความชันของเส้นโค้งความดันไอ (Slope of the Vapor Pressure Curve) ณ อุณหภูมิอากาศเฉลี่ย มีหน่วยเป็นกิโลปาสกาลต่อองศาเซลเซียส ($\text{kPa/}^\circ\text{C}$)
* $\gamma$ คือค่าคงที่ไซโครเมตริก (Psychrometric Constant) มีหน่วยเป็นกิโลปาสกาลต่อองศาเซลเซียส ($\text{kPa/}^\circ\text{C}$) ($\approx 0.066 \text{ kPa/}^\circ\text{C}$ ที่ระดับน้ำทะเล)

#### ขั้นตอนการอนุพัทธ์ค่าสัมประสิทธิ์ย่อยเพื่อประมวลผลในโค้ด:

1. **การคำนวณความดันไออิ่มตัว $e_s$ จากอุณหภูมิอากาศ ($T$):**
   $$e_s(T) = 0.6108 \exp\left(\frac{17.27 T}{T + 237.3}\right)$$

2. **การคำนวณความดันไอจริง $e_a$ จากค่าความชื้นสัมพัทธ์ ($RH$):**
   $$e_a = e_s(T) \times \frac{RH}{100}$$

3. **การคำนวณความชันของเส้นโค้งความดันไอ $\Delta$ จากอุณหภูมิ ($T$):**
   $$\Delta = \frac{4098 \times e_s(T)}{(T + 237.3)^2}$$

เมื่อคู่แฝดดิจิทัลได้ค่า $ET_0$ ซึ่งเป็นค่าสูญเสียน้ำอ้างอิงของพืชหญ้ามาตรฐานแล้ว ขั้นต่อไปคือการปรับแต่งค่าน้ำตามสปีชีส์ของพืชจริงด้วย **สัมประสิทธิ์พืช** (Crop Coefficient หรือ $K_c$) เพื่อหาค่าการคายระเหยน้ำจริงของพืชนั้นๆ ($ET_c$):

$$ET_c = K_c \times ET_0$$

* สำหรับ **ทุเรียน** ช่วงเติบโตทางใบ $K_c \approx 0.70$ และขยับขึ้นสูงถึง $K_c \approx 1.10 - 1.20$ ในช่วงพัฒนาผลอ่อนซึ่งต้องการน้ำสม่ำเสมอ
* สำหรับ **มังคุด** มีโครงสร้างปากใบที่เปิดปิดช้า อัตรา $K_c \approx 0.65 - 0.85$ ตามสภาพอากาศและช่วงการเตรียมต้นสะสมอาหาร

## 14.5 การเชื่อมต่อข้อมูล IoT เข้ากับโลกเสมือน (XR & Simulation Integration)

เมื่อสถาปัตยกรรมข้อมูลบนคลาวด์และโมเดลทางฟิสิกส์ถูกพัฒนาขึ้นจนสมบูรณ์ ขั้นต่อไปคือการสร้างส่วนประสานงานภาพในระดับสูงด้วยการบูรณาการเข้ากับเทคโนโลยีความเป็นจริงขยาย (Extended Reality หรือ XR) เช่น เทคโนโลยีความเป็นจริงเสริม (Augmented Reality หรือ AR) และความเป็นจริงเสมือน (Virtual Reality หรือ VR)

```
 [ สถานการณ์จริงในสนาม ] ──────> [ คลาวด์ & คู่แฝดดิจิทัล ] ──────> [ ส่วนแสดงผล XR ]
 1. ข้อมูลเซ็นเซอร์และกล้อง      1. คำนวณความชื้นเชิงพื้นที่       1. แว่น AR ซ้อนข้อมูลความชื้น
 2. สตรีมผ่านโปรโตคอลย่อย       2. อัปเดตแบบจำลอง 3 มิติ          2. แสดงระนาบสเปกตรัมทับต้นจริง
```

### 14.5.1 กลไกการฉายซ้อนข้อมูลแบบเวลาจริง (Real-time Spatial Data Overlay)
1. **การกำหนดพิกัดภูมิศาสตร์เชิงพื้นที่แบบละเอียด (High-Precision Spatial Tagging):** ต้นไม้และเซ็นเซอร์รับสัญญาณดินทุกจุดในสนามจะได้รับการบันทึกพิกัดภูมิศาสตร์สามมิติ (Latitude, Longitude, Altitude) ในสถาปัตยกรรมคู่แฝดดิจิทัล
2. **การจับตำแหน่งเชิงมิติสำหรับแอปพลิเคชัน AR (Spatial SLAM Integration):** เมื่อนักวิชาการเกษตรสวมใส่แว่น AR หรือใช้กล้องแท็บเล็ตเดินเข้าไปในแปลงปลูกทุเรียน อัลกอริทึม SLAM (Simultaneous Localization and Mapping) จะสแกนโครงสร้างพุ่มไม้และผิวดินรอบข้าง จากนั้นทำการซ้อนทับภาพกราฟิกสามมิติ เช่น พื้นผิวระดับความชื้นดินไล่โทนสีฟ้าถึงแดง หรือท่อน้ำจำลองเสมือนจริงที่กำลังไหลอยู่ใต้พื้นดิน
3. **การเข้าถึงระบบควบคุมแบบปฏิสัมพันธ์ (Immersive Remote Operation):** ผู้เรียนไม่เพียงแต่สามารถยืนมองดูค่าพารามิเตอร์ต่างๆ ที่ลอยอยู่เหนือยอดไม้จริงได้เท่านั้น แต่ยังสามารถสัมผัสปุ่มจำลองกลางอากาศ (AR Virtual Button) เพื่อทดลองปรับเพิ่มรอบปริมาณการจ่ายน้ำ และดูการเปลี่ยนแปลงของสถานะน้ำในแบบจำลองคู่แฝดดิจิทัลได้ทันที

การประยุกต์ใช้ XR ร่วมกับระบบจำลองเหล่านี้ทำให้นวัตกรสามารถควบคุมพื้นที่สวนขนาดใหญ่หลายร้อยไร่จากโต๊ะทำงานได้อย่างแม่นยำ พร้อมช่วยลดข้อผิดพลาดในการฝึกสอนบุคลากรใหม่ในสนามจริงได้อย่างมีนัยสำคัญ

## 14.6 การสังเคราะห์กรณีศึกษา: การสร้างสวนทุเรียนจำลอง (Digital Twin) เพื่อทดสอบการให้น้ำ

เพื่อให้ผู้เรียนเข้าใจภาพรวมของการนำสถาปัตยกรรมจัดการข้อมูลและทฤษฎีคู่แฝดดิจิทัลไปประยุกต์ใช้งานจริงในประเทศไทย กรณีศึกษานี้จะอธิบายระบบจำลองสวนทุเรียนหมอนทอง ณ จังหวัดจันทบุรี ที่ผสานการประมวลผลบนคลาวด์และโมเดลการคายน้ำเพื่อตัดสินใจให้น้ำแบบอัจฉริยะล่วงหน้า

### 14.6.1 ปัญหาของเกษตรกรในสนามจริง
ทุเรียนหมอนทองช่วงการเจริญเติบโตของผลอ่อน (Fruit Development Stage) มีพฤติกรรมไวต่อปริมาณน้ำอย่างยิ่ง หากดินแห้งเกินไปผลจะแตกและร่วงหล่นเสียหาย แต่หากเกษตรกรรดน้ำมากเกินไปเพราะความกลัว ต้นทุเรียนจะแตกใบอ่อน (Flushing) ซึ่งดึงธาตุอาหารไปเลี้ยงใบและสลัดผลอ่อนร่วงทิ้งเช่นเดียวกัน การให้น้ำจึงต้องสัมพันธ์กับระดับสภาพอากาศในวันนั้นและสภาวะดินจริง

### 14.6.2 การสังเคราะห์ระบบคู่แฝดดิจิทัลทุเรียนหมอนทอง (Durian Digital Twin Architecture)
การออกแบบโซลูชันระบบคู่แฝดดิจิทัลสำหรับกรณีศึกษานี้ ประกอบด้วยการทำงานประสานกันของ 3 องค์ประกอบหลัก:

1. **ระบบโทรมาตรสนาม (Field Telemetry System):**
   * ติดตั้งโหนดเซ็นเซอร์ตรวจวัดความชื้นดินคาปาซิทีฟ 2 ความลึก ($15 \text{ cm}$ ระดับรากฝอยหาอาหาร และ $40 \text{ cm}$ ระดับรากยึดเกาะและกักเก็บน้ำ)
   * ติดตั้งสถานีตรวจวัดภูมิอากาศย่อย (Microclimate Station) เพื่ออ่านอุณหภูมิอากาศ, ความชื้นสัมพัทธ์, ความเร็วลม และตัวเซ็นเซอร์วัดรังสีดวงอาทิตย์ (Pyranometer)
   * ข้อมูลทุกอย่างสตรีมเข้าสู่ API คลาวด์ทุกๆ 5 นาที ผ่านโปรโตคอล LoRaWAN

2. **เครื่องยนต์ประมวลผลคู่แฝดดิจิทัลบนคลาวด์ (Cloud-based Digital Twin Engine):**
   * จัดการนำเข้าข้อมูลสตรีมมิ่งลงสู่ถังข้อมูลอนุกรมเวลา InfluxDB
   * รันโปรแกรมจำลองสมดุลน้ำในดินระดับรากพืช (Soil Water Balance Simulation) ในทุกๆ 1 ชั่วโมง โดยดึงพารามิเตอร์สภาพอากาศมาป้อนเข้าสมการ FAO-56 Penman-Monteith
   * วิเคราะห์หา **ระดับความแห้งน้ำเชิงลึก** (Vapor Pressure Deficit - VPD) เพื่อตรวจสอบว่าต้นทุเรียนมีอัตราปั๊มน้ำขึ้นสู่ใบเร็วเกินความเร็วการดึงน้ำจากดินหรือไม่

3. **แดชบอร์ดสนับสนุนการตัดสินใจเชิงโต้ตอบ (Interactive DSS Dashboard):**
   * แสดงผลโครงสร้างจำลองต้นทุเรียน 3 มิติ โดยระนาบสีรอบต้นจะเปลี่ยนไปตามปริมาณน้ำในเนื้อดินชั้นรากพืช
   * มีปุ่มทดลองจำลองอนาคต 24 ชั่วโมง (Simulation Sandbox Run): เกษตรกรสามารถคลิกเลือก "หากฝนไม่ตกในวันพรุ่งนี้ และความชื้นดินเหลือ $35\%$ หากรดน้ำเพียง $20$ นาที สภาวะความเค้นน้ำของต้นทุเรียนจะเป็นอย่างไร?" คู่แฝดดิจิทัลจะนำประวัติการคายระเหยน้ำจากวันก่อนหน้าร่วมกับการทำนายสภาพอากาศกรมอุตุฯ มาทำการเร่งความเร็วการจำลองแล้วพลอตกราฟแนวโน้มผลลัพธ์ให้เกษตรกรตัดสินใจเปิดน้ำได้ล่วงหน้าได้อย่างถูกต้องแม่นยำ

## 14.7 โครงสร้างการเขียนโปรแกรมระบบจำลองคู่แฝดดิจิทัลและสมดุลน้ำ

ในส่วนนี้เป็นการนำเสนอสถาปัตยกรรมการพัฒนาโปรแกรมคอมพิวเตอร์เพื่อสร้างระบบควบคุม จำลอง และคำนวณสมดุลน้ำพืชและคู่แฝดดิจิทัล โดยแบ่งระดับความซับซ้อนของการพัฒนาโค้ดตามความเหมาะสมของผู้เรียน 3 ช่วงชั้น ได้แก่ มัธยมศึกษาตอนต้น มัธยมศึกษาตอนปลาย และอุดมศึกษา

### 14.7.1 ระดับมัธยมศึกษาตอนต้น: ระบบติดตามสมดุลน้ำในดินแบบง่าย (Simple Soil Water Balance Tracker)

**แนวคิด:** เขียนโปรแกรมเชิงลำดับขั้นตอน (Procedural Programming) เพื่อบันทึกระดับน้ำสะสมในดินรายวันของต้นทุเรียน โดยมีค่าแปรผันหลักคือ ฝนตก การระเหยน้ำของดิน และระดับน้ำจากการรด ช่วยให้เข้าใจกลไกกฎข้ออนุรักษ์สมดุลน้ำแบบไม่มีเงื่อนไขซับซ้อน

```python
# ==============================================================================
# ระดับมัธยมศึกษาตอนต้น: ระบบติดตามสมดุลน้ำในดินแบบง่าย (Simple Soil Water Balance Tracker)
# ==============================================================================

# กำหนดตัวแปรควบคุมระบบ (หน่วยเป็น มิลลิเมตร หรือ mm)
max_soil_water_capacity = 100.0  # ปริมาตรกักเก็บน้ำสูงสุดของหน้าดินแปลงทุเรียน
soil_water_stock = 60.0          # ปริมาณน้ำสะสมเริ่มต้นในดิน ณ วันปัจจุบัน

# ข้อมูลจำลองสภาวะแวดล้อมรายสัปดาห์ (7 วัน)
# ประกอบด้วยค่า (ฝนตก, น้ำที่รดเพิ่ม, การระเหยน้ำตามธรรมชาติของวันนั้น)
weekly_weather_data = [
    (0.0, 15.0, 5.2),  # วันที่ 1: รดน้ำ 15mm, แดดจัดสูญเสียน้ำ 5.2mm
    (0.0, 15.0, 6.0),  # วันที่ 2: รดน้ำ 15mm, ร้อนจัดสูญเสียน้ำ 6.0mm
    (12.5, 0.0, 2.1),  # วันที่ 3: ฝนตกหนัก 12.5mm งดรดน้ำ สูญเสียน้ำ 2.1mm
    (0.0, 15.0, 5.5),  # วันที่ 4: รดน้ำ 15mm, สูญเสียน้ำ 5.5mm
    (0.0, 15.0, 5.8),  # วันที่ 5: รดน้ำ 15mm, สูญเสียน้ำ 5.8mm
    (5.0, 0.0, 3.0),   # วันที่ 6: ฝนตกปรอยๆ 5mm งดรดน้ำ สูญเสียน้ำ 3.0mm
    (0.0, 20.0, 6.5)   # วันที่ 7: อากาศร้อนมาก รดน้ำเพิ่มพิเศษ 20mm สูญเสีย 6.5mm
]

print("=== ระบบคำนวณและติดตามสมดุลน้ำในดินทุเรียนรายวัน ===")
print(f"ปริมาณน้ำในดินเริ่มต้น: {soil_water_stock:.1f} mm\n")

for day, (precipitation, irrigation, evapotranspiration) in enumerate(weekly_weather_data, 1):
    # สมการสมดุลน้ำพื้นฐาน: น้ำสะสมใหม่ = น้ำเดิม + น้ำที่เติมเข้า - น้ำที่เสียไป
    added_water = precipitation + irrigation
    new_stock = soil_water_stock + added_water - evapotranspiration
    
    # ป้องกันระดับน้ำล้นความสามารถกักเก็บ (น้ำส่วนเกินจะไหลบ่าหนีไป - Drainage)
    drainage = 0.0
    if new_stock > max_soil_water_capacity:
        drainage = new_stock - max_soil_water_capacity
        new_stock = max_soil_water_capacity
    
    # ป้องกันค่าน้ำติดลบในดิน (กรณีดินแห้งขีดสุด)
    if new_stock < 0.0:
        new_stock = 0.0
        
    # อัปเดตตัวแปรกักเก็บน้ำสะสมเพื่อคำนวณต่อในวันถัดไป
    soil_water_stock = new_stock
    
    # รายงานผลประจำวัน
    print(f"วันที่ {day}: [เติมน้ำ: {added_water:4.1f} mm] | [คายน้ำ: {evapotranspiration:3.1f} mm] | [น้ำล้นทิ้ง: {drainage:4.1f} mm]")
    print(f"        -> ระดับน้ำคงเหลือในดิน: {soil_water_stock:5.1f} / {max_soil_water_capacity:.1f} mm")
    
    # แจ้งเตือนสถานะเชิงลึกเพื่อการเกษตรกรรม
    if soil_water_stock < 30.0:
        print("        ⚠️ เตือนภัย: ดินแห้งเกินไป! ต้นทุเรียนกำลังสะสมความเครียด")
    elif soil_water_stock > 90.0:
        print("        🌊 เตือนภัย: ดินเปียกชุ่มแฉะเกินไป! เสี่ยงเกิดโรครากเน่าโคนเน่า")
    print("-" * 70)

print("จบการจำลองระบบติดตามสมดุลน้ำรายสัปดาห์")
```

### 14.7.2 ระดับมัธยมศึกษาตอนปลาย: โปรแกรมคำนวณการคายระเหยน้ำอ้างอิงเชิงฟังก์ชัน (Functional Reference Evapotranspiration Calculator)

**แนวคิด:** ใช้การเขียนโปรแกรมเชิงฟังก์ชัน (Functional Programming) ในการสร้างชุดเครื่องมือคำนวณทางฟิสิกส์ แปลงความดันอากาศ พล็อตสมการความดันไออิ่มตัว ความชื้น และนำไปสู่การประมาณหาค่าสูญเสียน้ำรายวันอ้างอิงของระบบเกษตรในแต่ละเขตพิกัด

```python
# ==============================================================================
# ระดับมัธยมศึกษาตอนปลาย: โปรแกรมคำนวณการคายระเหยน้ำอ้างอิงเชิงฟังก์ชัน
# ==============================================================================
import math

def calculate_saturation_vapor_pressure(temp_celsius):
    """คำนวณความดันไออิ่มตัว (e_s) หน่วย kPa จากอุณหภูมิอากาศ"""
    return 0.6108 * math.exp((17.27 * temp_celsius) / (temp_celsius + 237.3))

def calculate_actual_vapor_pressure(saturation_vp, relative_humidity):
    """คำนวณความดันไอจริง (e_a) หน่วย kPa จากความดันไออิ่มตัวและความชื้นสัมพัทธ์"""
    return saturation_vp * (relative_humidity / 100.0)

def calculate_vapor_pressure_slope(temp_celsius, saturation_vp):
    """คำนวณความชันของเส้นโค้งความดันไอ (Delta) หน่วย kPa/C"""
    return (4098.0 * saturation_vp) / ((temp_celsius + 237.3) ** 2)

def estimate_penman_monteith_et0(temp, humidity, wind_speed, solar_rad, psychro_const=0.066):
    """
    คำนวณปริมาณการคายระเหยน้ำอ้างอิงรายวัน (ET_0) ด้วยสมการอย่างง่ายของ FAO-56 Penman-Monteith
    :param temp: อุณหภูมิเฉลี่ย (C)
    :param humidity: ความชื้นสัมพัทธ์เฉลี่ย (%)
    :param wind_speed: ความเร็วลม ณ 2 เมตร (m/s)
    :param solar_rad: รังสีสุทธิจากดวงอาทิตย์ (MJ/m2/day)
    :param psychro_const: ค่าคงที่ไซโครเมตริกสัมพัทธ์ (kPa/C)
    """
    # 1. อนุพัทธ์ค่าสัมประสิทธิ์ที่ได้จากความดันไอ
    e_s = calculate_saturation_vapor_pressure(temp)
    e_a = calculate_actual_vapor_pressure(e_s, humidity)
    vpd = e_s - e_a  # ความดันไอพร่อง
    
    delta = calculate_vapor_pressure_slope(temp, e_s)
    
    # 2. ตั้งสมการ Penman-Monteith (อิงค่า Soil Heat Flux G = 0 รายวัน)
    numerator_radiation = 0.408 * delta * solar_rad
    numerator_aerodynamic = psychro_const * (900.0 / (temp + 273.0)) * wind_speed * vpd
    denominator = delta + psychro_const * (1.0 + 0.34 * wind_speed)
    
    et0 = (numerator_radiation + numerator_aerodynamic) / denominator
    return round(et0, 3)

# ข้อมูลสภาพอากาศจำลองที่บันทึกโดยสถานีสนามจันทบุรี
weather_logs = [
    {"day": 1, "temp": 32.5, "humidity": 75.0, "wind": 1.8, "solar": 18.2},
    {"day": 2, "temp": 34.0, "humidity": 65.0, "wind": 2.2, "solar": 21.0},
    {"day": 3, "temp": 28.5, "humidity": 90.0, "wind": 1.2, "solar": 8.5},  # วันที่ครึ้มฟ้าครึ้มฝน
    {"day": 4, "temp": 31.0, "humidity": 80.0, "wind": 1.5, "solar": 16.0},
    {"day": 5, "temp": 33.2, "humidity": 70.0, "wind": 2.0, "solar": 19.5}
]

print("=== สถิติตรวจวัดอากาศประจำวันและวิเคราะห์ความต้องการน้ำอ้างอิง ===")
for log in weather_logs:
    et0 = estimate_penman_monteith_et0(
        temp=log["temp"],
        humidity=log["humidity"],
        wind_speed=log["wind"],
        solar_rad=log["solar"]
    )
    
    # คำนวณความต้องการน้ำของพืชประเภทต่างๆ โดยอิงค่าคงที่สัมประสิทธิ์พืช
    durian_kc_fruit = 1.15
    mangosteen_kc_growth = 0.75
    
    durian_water_need = et0 * durian_kc_fruit
    mangosteen_water_need = et0 * mangosteen_kc_growth
    
    print(f"วันที่ {log['day']}: [อุณหภูมิ: {log['temp']:.1f}°C] | [ความชื้น: {log['humidity']:.0f}%] | [ความเร็วลม: {log['wind']:.1f} m/s]")
    print(f"        -> อัตราการคายระเหยน้ำอ้างอิง (ET_0): {et0:.3f} มม./วัน")
    print(f"        -> ค่าน้ำชดเชย ทุเรียนหมอนทอง (ช่วงขยายขนาดผล): {durian_water_need:.3f} มม./วัน")
    print(f"        -> ค่าน้ำชดเชย มังคุด (ช่วงเตรียมสร้างกิ่งก้าน): {mangosteen_water_need:.3f} มม./วัน")
    print("-" * 80)
```

### 14.7.3 ระดับอุดมศึกษา: ระบบจำลองคู่แฝดดิจิทัลเพื่อการจัดการทรัพยากรน้ำพืชเชิงวัตถุ (Object-Oriented Crop Digital Twin Simulator)

**แนวคิด:** ออกแบบระบบจำลองชั้นสูงเชิงวัตถุ (Object-Oriented Programming - OOP) เต็มรูปแบบเพื่อรันระบบคู่แฝดดิจิทัลของแปลงทุเรียนและมังคุด โปรแกรมจะทำการคำนวณสมดุลน้ำพืช ดัชนีการเติบโตทางใบ (Leaf Area Index หรือ LAI) ขอบเขตระดับรากที่มีการเปลี่ยนพลศาสตร์ตามช่วงเวลา มีระบบบันทึกเหตุการณ์ (Logging) ที่พร้อมใช้งานจริงสำหรับการดีบั๊กข้อมูลสตรีมมิ่งเซ็นเซอร์ และส่งออกรายงานประจำวันเป็นไฟล์ JSON

```python
# ==============================================================================
# ระดับอุดมศึกษา: ระบบจำลองคู่แฝดดิจิทัลเพื่อการจัดการทรัพยากรน้ำพืชเชิงวัตถุ
# ==============================================================================
import math
import json
import logging
import os
from datetime import datetime, timedelta

# กำหนดสถาปัตยกรรมตัวจดบันทึกเหตุการณ์ระบบ (System Logger)
logging.basicConfig(
    level=logging.INFO,
    format='%(asctime)s [%(levelname)s] %(message)s'
)
logger = logging.getLogger("AgriDigitalTwin")

class CropProfile:
    """ข้อมูลโปรไฟล์ทางชีวภาพของพืชสำหรับป้อนเข้าเครื่องยนต์จำลองคู่แฝดดิจิทัล"""
    def __init__(self, name, base_kc, max_kc, initial_root_depth_m, max_root_depth_m):
        self.name = name
        self.base_kc = base_kc
        self.max_kc = max_kc
        self.root_depth_m = initial_root_depth_m
        self.max_root_depth_m = max_root_depth_m
        self.lai = 1.0  # ดัชนีพื้นที่ใบเริ่มต้น (Leaf Area Index)
        
    def grow_crop(self, days_passed):
        """จำลองพลศาสตร์การเจริญเติบโตทางสรีรวิทยาของต้นพืชตามคาบเวลา"""
        # รากชอนไชเติบโตขึ้นเรื่อยๆ จนเต็มประสิทธิภาพสูงสุด
        self.root_depth_m = min(self.root_depth_m + 0.005 * days_passed, self.max_root_depth_m)
        # ดัชนีพื้นที่ใบขยายกว้างตามอัตราความร้อนและอายุใบไม้
        self.lai = min(self.lai + 0.02 * days_passed, 5.5)
        
    def get_current_crop_coefficient(self):
        """คำนวณสัมประสิทธิ์พืช ณ วันนั้น โดยอ้างอิงดัชนีพื้นที่ใบปัจจุบัน"""
        # อัตรารวมของปากใบคายน้ำขยับตัวตามระดับความหนาแน่นทรงพุ่ม
        kc_factor = (self.lai / 5.5) * (self.max_kc - self.base_kc)
        return round(self.base_kc + kc_factor, 2)


class SoilProfile:
    """ระบบกายภาพจำลองพลศาสตร์กักเก็บน้ำของเนื้อดินแต่ละชนิด"""
    def __init__(self, soil_type, field_capacity, wilting_point):
        self.soil_type = soil_type
        self.field_capacity = field_capacity  # จุดชลประทานอิ่มตัวสูงสุดของดิน (%)
        self.wilting_point = wilting_point    # จุดเหี่ยวแห้งถาวรที่พืชไม่สามารถดึงน้ำออกได้ (%)
        self.current_moisture = field_capacity - 5.0 # เริ่มต้นที่ดินแห้งกว่าอิ่มน้ำเล็กน้อย
        
    def calculate_water_content_mm(self, root_depth_m):
        """คำนวณปริมาณน้ำเป็นมิลลิเมตรเทียบกับความลึกรากพืช ณ วันนั้น"""
        # สมการคำนวณปริมาตรน้ำในดินเชิงความลึก:
        # Water (mm) = Moisture_fraction * root_depth (m) * 1000 (mm/m)
        moisture_fraction = self.current_moisture / 100.0
        return moisture_fraction * root_depth_m * 1000.0

    def update_moisture(self, net_water_diff_mm, root_depth_m):
        """ปรับปรุงระดับเปอร์เซ็นต์ความชื้นดินตามสัดส่วนการใช้น้ำและการเติมน้ำ"""
        water_equivalent_depth = root_depth_m * 1000.0
        moisture_diff_pct = (net_water_diff_mm / water_equivalent_depth) * 100.0
        
        # ปรับความชื้นตัวแปลง
        self.current_moisture += moisture_diff_pct
        # จำกัดขอบเขตความจริงทางฟิสิกส์ในแปลงดิน
        self.current_moisture = max(self.wilting_point - 2.0, min(self.current_moisture, self.field_capacity))


class DurianDigitalTwin:
    """เครื่องยนต์หลักจำลองสภาวะคู่แฝดดิจิทัลอภิบาลน้ำทุเรียนอัจฉริยะ"""
    def __init__(self, crop: CropProfile, soil: SoilProfile, initial_latitude: float):
        self.crop = crop
        self.soil = soil
        self.latitude = initial_latitude
        self.days_simulated = 0
        self.history_log = []
        logger.info(f"เริ่มต้นประกอบโครงร่างคู่แฝดดิจิทัลสำหรับพืช: {crop.name} เรียบร้อยแล้ว")
        
    def solve_evapotranspiration_fao56(self, temp, rh, wind, solar_rad):
        """
        วิเคราะห์ประมวลผลสมการ Penman-Monteith ตามความสัมพันธ์ทางฟิสิกส์ชั้นสูง
        """
        try:
            # 1. ความดันไออิ่มตัวเฉลี่ยในอากาศ
            e_s = 0.6108 * math.exp((17.27 * temp) / (temp + 237.3))
            # 2. ความดันไอจริง
            e_a = e_s * (rh / 100.0)
            vpd = e_s - e_a
            
            # 3. ความชันความดันไอ
            delta = (4098.0 * e_s) / ((temp + 237.3) ** 2)
            
            # 4. ค่าคงที่ไซโครเมตริกมาตรฐาน
            gamma = 0.066
            
            # 5. วิเคราะห์เศษส่วน Penman-Monteith
            net_rad_net = solar_rad  # ถือว่าฟลักซ์ดินเป็นศูนย์
            num_rad = 0.408 * delta * net_rad_net
            num_aero = gamma * (900.0 / (temp + 273.0)) * wind * vpd
            den = delta + gamma * (1.0 + 0.34 * wind)
            
            et0 = (num_rad + num_aero) / den
            return max(0.1, round(et0, 4))
        except ZeroDivisionError as e:
            logger.error("เกิดข้อผิดพลาดคณิตศาสตร์หารด้วยศูนย์ในสมการ Penman-Monteith")
            raise e
        except Exception as ex:
            logger.error(f"ระบบไม่สามารถคำนวณสมการความลึกฟิสิกส์ได้: {str(ex)}")
            return 1.0

    def step_daily_simulation(self, weather_data: dict, irrigation_applied_mm: float):
        """จำลองการเปลี่ยนผ่านสถานะสะสมรายวันอัปเดตคู่แฝดดิจิทัล"""
        self.days_simulated += 1
        
        # 1. ปล่อยการเจริญเติบโตพืช
        self.crop.grow_crop(1)
        
        # 2. ประมวลหาปริมาณระเหยน้ำอ้างอิง
        et0 = self.solve_evapotranspiration_fao56(
            temp=weather_data["temp_c"],
            rh=weather_data["humidity_pct"],
            wind=weather_data["wind_speed_ms"],
            solar_rad=weather_data["solar_rad_mj_m2"]
        )
        
        # 3. หาน้ำคายระเหยจริงของพืชตามระยะสัมประสิทธิ์
        kc = self.crop.get_current_crop_coefficient()
        etc = et0 * kc
        
        # 4. วิเคราะห์สมดุลน้ำในระบบดินและพืช
        rain = weather_data.get("rain_mm", 0.0)
        water_added = rain + irrigation_applied_mm
        
        # เปรียบต่างน้ำสะสมเดิม
        initial_water_mm = self.soil.calculate_water_content_mm(self.crop.root_depth_m)
        net_diff_mm = water_added - etc
        
        # อัปเดตโครงสร้างความชื้นดิน
        self.soil.update_moisture(net_diff_mm, self.crop.root_depth_m)
        final_water_mm = self.soil.calculate_water_content_mm(self.crop.root_depth_m)
        
        # ตรวจวัดปริมาณน้ำล้นเดรนหนีลงใต้พิภพ
        drainage = max(0.0, (initial_water_mm + net_diff_mm) - final_water_mm)
        
        # คำนวณหาดัชนีความเครียดน้ำของพืช (Crop Water Stress Index - CWSI อย่างง่าย)
        # ถือว่าหากความชื้นต่ำกว่าค่าเฉลี่ยกึ่งกลาง ดินจะคายน้ำช้า พืชเริ่มเครียด
        moisture_midpoint = (self.soil.field_capacity + self.soil.wilting_point) / 2.0
        if self.soil.current_moisture < moisture_midpoint:
            cwsi = (moisture_midpoint - self.soil.current_moisture) / (moisture_midpoint - self.soil.wilting_point)
            cwsi = min(1.0, round(cwsi, 2))
        else:
            cwsi = 0.0

        daily_log = {
            "day": self.days_simulated,
            "crop_age_sim": self.days_simulated,
            "root_depth_m": round(self.crop.root_depth_m, 3),
            "lai": round(self.crop.lai, 2),
            "kc": kc,
            "et0_mm": et0,
            "etc_mm": round(etc, 2),
            "rain_mm": rain,
            "irrigation_mm": irrigation_applied_mm,
            "soil_moisture_pct": round(self.soil.current_moisture, 2),
            "water_content_mm": round(final_water_mm, 2),
            "drainage_loss_mm": round(drainage, 2),
            "crop_stress_index": cwsi
        }
        
        self.history_log.append(daily_log)
        logger.info(f"วันจำลองย่อยที่ {self.days_simulated} สำเร็จ: ดินชื้น {daily_log['soil_moisture_pct']}% | ความเครียดพืช: {cwsi}")
        return daily_log

    def export_twin_report_json(self, output_path):
        """ส่งออกรายงานผลลัพธ์การจัดการทรัพยากรน้ำของพืชเป็นไฟล์ JSON"""
        try:
            report_data = {
                "simulation_metadata": {
                    "crop_name": self.crop.name,
                    "soil_type": self.soil.soil_type,
                    "latitude": self.latitude,
                    "generated_at": datetime.now().isoformat(),
                    "total_days": self.days_simulated
                },
                "daily_timeseries": self.history_log
            }
            
            with open(output_path, 'w', encoding='utf-8') as f:
                json.dump(report_data, f, ensure_ascii=False, indent=4)
            logger.info(f"📝 บันทึกข้อมูลประวัติคู่แฝดดิจิทัลเสร็จสมบูรณ์ปลายทางที่: {output_path}")
        except IOError as e:
            logger.error(f"ไม่สามารถเขียนข้อมูลไฟล์ประวัติรายงานได้: {str(e)}")


# --- การรันการทำงานทดสอบเครื่องยนต์คู่แฝดดิจิทัลระบบอุดมศึกษา ---

    # 3. ข้อมูลภูมิอากาศเชิงสัปดาห์ในสนาม
    weekly_met_data = [

    # กำหนดแผนให้น้ำชดเชยของเกษตรกร (มิลลิเมตร)
```

## 🎯 แบบฝึกหัดทบทวนเชิงวิเคราะห์ (Analytical Review Exercises)

เพื่อให้ผู้เรียนเข้าใจอย่างถ่องแท้เกี่ยวกับการออกแบบเชิงวิศวกรรมข้อมูลคลาวด์และการประมวลพลศาสตร์แบบจำลองพืช ขอจงแสดงวิธีคิดและให้เหตุผลต่อคำถามต่อไปนี้:

### คำถามระดับประยุกต์และวิเคราะห์ (Applying and Analyzing)
1. **วิเคราะห์ความได้เปรียบเชิงมิติความหน่วง:** หากระบบเกษตรอัจฉริยะต้องการแสดงกราฟแนวโน้มความเปียกชื้นในรอบ 24 ชั่วโมงจากแปลงปลูกส้มโอที่ส่งสัญญาณเซ็นเซอร์ทุก 10 วินาที จงวิเคราะห์ความต่างของประสิทธิภาพในการประมวลผลและการใช้หน่วยจัดเก็บระหว่างการดึงข้อมูลจากตารางเชิงสัมพันธ์ (Relational SQL Database) ที่ทำการสืบค้นแบบผูกเงื่อนไขเวลา เทียบกับฐานข้อมูลอนุกรมเวลาที่มีฟังก์ชันลดขนาดข้อมูลแบบดาวน์แซมพลิงในตัว
2. **ประเมินผลสมการการคายน้ำ:** อุณหภูมิเฉลี่ย ณ แปลงทุเรียนสูงถึง $35.0^\circ\text{C}$ โดยเซ็นเซอร์ระบุดังนี้:
   * ความชื้นสัมพัทธ์อากาศ $RH = 60\%$
   * ความเร็วลม $u_2 = 2.0 \text{ m/s}$
   * รังสีดวงอาทิตย์สุทธิ $R_n = 20.0 \text{ MJ/m}^2/\text{day}$
   * กำหนดค่าคงที่ $\gamma = 0.066$
   * จงคำนวณหาค่าความดันไออิ่มตัว $e_s$, ความดันไอจริง $e_a$, ความดันไอพร่อง $(e_s - e_a)$ และความชันของสมการความดันไอ $\Delta$ เพื่อนำไปวิเคราะห์ศักยภาพการสูญเสียน้ำสูงสุดตามเกณฑ์สมการ Penman-Monteith

### คำถามระดับสังเคราะห์และประเมินค่า (Synthesizing and Evaluating)
3. **ออกแบบแดชบอร์ดสำหรับผู้สูงวัย:** จงร่างแผนงานและหลักการออกแบบส่วนติดต่อผู้ใช้งาน (User Interface หรือ UI) และประสบการณ์ของผู้ใช้สำหรับระบบแดชบอร์ดสั่งงานจ่ายน้ำในเขตแปลงปลูกมังคุดของวิสาหกิจชุมชน ที่มีสัดส่วนสมาชิกเกษตรกรเป็นผู้สูงวัยกว่าร้อยละ 60 โดยเน้นกลไกการรับรู้สภาวะวิกฤตของน้ำและความชัดเจนในการสั่งงานเปิดวาล์วระยะไกลแบบโต้ตอบได้อย่างแม่นยำ
4. **สังเคราะห์สถาปัตยกรรมคู่แฝดดิจิทัลทนทานต่อสภาพแวดล้อมขัดข้อง:** จงนำเสนอแผนภาพสถาปัตยกรรมและรายละเอียดคำอธิบายของกระบวนการส่งและบริหารจัดการข้อมูล (Data Pipeline) จากระบบแปลงทุเรียนแถบหุบเขาสูงพังงา ที่สัญญาณ 4G ขาดหายบ่อยครั้ง โดยเชื่อมข้อมูลเข้าสู่แบบจำลองคู่แฝดดิจิทัลบนระบบกระจายคลาวด์ พร้อมทั้งอธิบายรายละเอียดว่าระบบจะประสานเวลาและคัดแยกข้อมูลที่ซ้ำซ้อนหรือตกหล่นยามเครือข่ายฟื้นตัวกลับมาได้อย่างไร
