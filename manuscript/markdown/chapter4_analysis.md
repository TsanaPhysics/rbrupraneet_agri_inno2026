# บทที่ 4: การวิเคราะห์ข้อมูลอนุกรมเวลาและการตรวจจับสิ่งผิดปกติเชิงสถิติ
## (Time-series Analysis & Statistical Anomaly Detection)

---

## 🎯 วัตถุประสงค์การเรียนรู้ (Learning Objectives)
เมื่อศึกษาบทนี้จบแล้ว ผู้เรียนจะสามารถ:
1. **วิเคราะห์ (Analyze)** ลักษณะทางสถิติของข้อมูลอนุกรมเวลา (Time-series Data) ทางอุตุนิยมวิทยาเกษตร ทั้งมิติความเป็นฤดูกาล (Seasonality) และแนวโน้มหลัก (Trend)
2. **ประเมิน (Evaluate)** และคัดเลือกเทคนิคการทำความสะอาดข้อมูลร่วมกับการทดแทนค่าที่สูญหาย (Data Imputation) ที่เหมาะสมต่อคุณลักษณะทางฟิสิกส์ของตัวแปรภูมิอากาศแต่ละประเภท
3. **ประยุกต์ใช้ (Apply)** เทคนิคค่าเฉลี่ยเคลื่อนที่ (Moving Average) ทั้งแบบธรรมดาและแบบเอ็กซ์โพเนนเชียลในการกรองสัญญาณรบกวนความถี่สูงออกจากแนวโน้มข้อมูลสภาพอากาศ
4. **สังเคราะห์ (Synthesize)** และโปรแกรมอัลกอริทึมตรวจจับสิ่งผิดปกติ (Anomaly Detection) เชิงสถิติเพื่อระบุจุดวิกฤตภัยพิบัติในแปลงเพาะปลูก

## 4.1 ธรรมชาติของข้อมูลอนุกรมเวลาในงานภูมิอากาศเกษตร
ข้อมูลส่วนใหญ่ในทางเกษตรกรรมอัจฉริยะจัดเป็น **ข้อมูลอนุกรมเวลา (Time-series Data)** ซึ่งหมายถึงชุดข้อมูลของตัวแปรทางกายภาพที่เก็บรวบรวมอย่างต่อเนื่องตามลำดับเวลาที่แน่นอน คุณสมบัติเชิงสถิติของข้อมูลประเภทนี้มีความแตกต่างอย่างมากจากข้อมูลทั่วไป เนื่องจากมีการผูกมัดกับลำดับก่อนหลังและมีคุณลักษณะสำคัญ 3 ประการ:

```
  [ ข้อมูลอนุกรมเวลาภูมิอากาศ ] ───► [ แนวโน้มหลัก (Trend) ]
                                ───► [ ความเป็นฤดูกาล (Seasonality) ]
                                ───► [ สัญญาณรบกวนสุ่ม (Random Noise) ]
```

1. **แนวโน้มหลัก (Trend)**: ทิศทางการเปลี่ยนแปลงของข้อมูลในระยะยาว เช่น แนวโน้มการเพิ่มขึ้นของอุณหภูมิเฉลี่ยสะสมในจังหวัดจันทบุรีในรอบ 30 ปี เนื่องมาจากปัญหาสภาพภูมิอากาศเปลี่ยนแปลง
2. **ความเป็นฤดูกาล (Seasonality)**: รูปแบบความแปรปรวนที่เกิดขึ้นซ้ำ ๆ ในรอบเวลาที่แน่นอน เช่น อุณหภูมิอากาศพุ่งสูงขึ้นจุดสูงสุดในช่วงบ่ายและต่ำสุดในช่วงเช้ามืดของทุกวัน หรือปริมาณฝนสะสมที่สูงในฤดูมรสุม (พฤษภาคม-ตุลาคม) ของทุกปี
3. **ความผันผวนสุ่ม (Random Noise)**: ความแปรปรวนระยะสั้นที่ไม่มีรูปแบบแน่นอน เช่น ลมกระโชกแรงชั่วขณะ หรือฝนหลงฤดูตกกะทันหัน ซึ่งเกิดจากระบบสภาพอากาศที่มีความปั่นป่วน (Turbulent Systems)

การวิเคราะห์อนุกรมเวลาช่วยให้นักวิจัยสามารถถอดสัญญาณรบกวนสุ่มออกเพื่อสกัดเอาแนวโน้มความเป็นฤดูกาลที่แท้จริงมาใช้ในการพยากรณ์และบริหารจัดการฟาร์ม

## 4.2 เทคนิคการเตรียมข้อมูลและการจัดการค่าที่สูญหาย
ก่อนนำข้อมูลดิบจากภาคสนามไปประมวลผล จำเป็นต้องผ่านกระบวนการ **การเตรียมข้อมูลล่วงหน้า (Data Preprocessing)** เพื่อทำความสะอาดข้อมูลที่เสียหาย โดยเฉพาะปัญหาข้อมูลขาดหายเนื่องจากเซ็นเซอร์สูญเสียพลังงาน หรือการสื่อสารล่มชั่วคราว การเลือกกลยุทธ์ทดแทนค่าต้องสอดคล้องกับธรรมชาติทางฟิสิกส์ของพารามิเตอร์นั้น ๆ:

### 1. การทดแทนค่าด้วยค่าเฉลี่ย (Mean Imputation)
* **หลักการ**: การเติมช่องว่างที่ขาดหายด้วยค่าเฉลี่ยของข้อมูลทั้งหมดในชุดข้อมูล
* **การประยุกต์ใช้**: เหมาะสำหรับพารามิเตอร์ที่ไม่มีความผันแปรเชิงเวลาสูงและค่อนข้างคงที่ เช่น สภาพความเป็นกรด-ด่างของดิน (pH)
* **ข้อจำกัด**: ทำลายมิติเวลาและการแจกแจงทางสถิติของข้อมูลอนุกรมเวลา

### 2. การประมาณค่าในช่วงเชิงเส้น (Linear Interpolation)
* **หลักการ**: การคำนวณสัดส่วนค่าที่ขาดหายระหว่างจุดอ้างอิงก่อนหน้าและจุดถัดไปตามสมการเส้นตรง
* **การประยุกต์ใช้**: เหมาะอย่างยิ่งสำหรับพารามิเตอร์ทางสภาพอากาศที่มีการเปลี่ยนแปลงอย่างค่อยเป็นค่อยไปตามกฎฟิสิกส์การถ่ายเทความร้อน เช่น อุณหภูมิอากาศ หรือความชื้นสัมพัทธ์ในอากาศ
* **สมการคณิตศาสตร์**:
  $$y = y_0 + (x - x_0) \frac{y_1 - y_0}{x_1 - x_0}$$

### 3. การลบข้อมูลแถวนั้นออก (Data Removal)
* **หลักการ**: การลบช่วงเวลาที่ข้อมูลเสียหายทิ้งไป
* **การประยุกต์ใช้**: ใช้เมื่อเกิดความล้มเหลวเชิงระบบเป็นเวลานาน (เช่น เซ็นเซอร์พังติดต่อกันเกิน 1 สัปดาห์) การพยายามทดแทนค่าในช่วงที่กว้างเกินไปจะสร้างความบิดเบือนทางวิชาการสูง

## 4.3 การวิเคราะห์แนวโน้มด้วยค่าเฉลี่ยเคลื่อนที่
ข้อมูลเซ็นเซอร์ภาคสนามมักมีความผันผวนรายนาทีสูงเนื่องจากฝุ่นละออง เมฆบังแดดชั่วคราว หรือลมพัด เทคนิคการกรองสัญญาณรบกวนที่คลาสสิกและมีประสิทธิภาพสูงสุดคือการใช้ **ค่าเฉลี่ยเคลื่อนที่ (Moving Average - MA)** เพื่อเกลี่ยข้อมูลให้เรียบเนียน (Smoothing) ทำให้มองเห็นโครงสร้างแนวโน้มที่แท้จริงในการตัดสินใจควบคุม

### 1. ค่าเฉลี่ยเคลื่อนที่แบบง่าย (Simple Moving Average - SMA)
คำนวณโดยการหาค่าเฉลี่ยเลขคณิตธรรมดาของข้อมูลภายในกรอบเวลาหน้าต่าง (Window Size) ย้อนหลัง $k$ คาบ:
$$SMA_t = \frac{1}{k} \sum_{i=0}^{k-1} x_{t-i}$$

* *ข้อจำกัด*: ทุกจุดข้อมูลมีน้ำหนักเท่ากัน ทำให้เกิดอาการหน่วงช้า (Time Lag) ต่อการเปลี่ยนแปลงเฉลี่ยกะทันหัน

### 2. ค่าเฉลี่ยเคลื่อนที่แบบถ่วงน้ำหนักเอ็กซ์โพเนนเชียล (Exponential Moving Average - EMA)
ให้ความสำคัญและน้ำหนักสูงแก่ข้อมูล ณ เวลาปัจจุบัน และลดน้ำหนักลงแบบลดหลั่นตามฟังก์ชันชี้กำลังเมื่อย้อนเวลากลับไปในอดีต ช่วยลดอาการหน่วงช้าและตอบสนองต่อแนวโน้มใหม่ได้รวดเร็วกว่า:
$$EMA_t = \alpha \cdot x_t + (1 - \alpha) \cdot EMA_{t-1}$$

โดยที่ $\alpha$ คือ สัมประสิทธิ์การถ่วงน้ำหนัก (Smoothing Factor) มีค่าอยู่ระหว่าง $0$ ถึง $1$:
$$\alpha = \frac{2}{k + 1}$$

## 4.4 สถิติเชิงพรรณนาและการกระจายตัวของข้อมูลภูมิอากาศ
การทำความเข้าใจคุณลักษณะทางประชากรของข้อมูลภูมิอากาศผ่าน **สถิติเชิงพรรณนา (Descriptive Statistics)** ช่วยให้นวัตกรกำหนดขอบเขตความปกติของสิ่งแวดล้อมในสวนพืชแต่ละชนิด พารามิเตอร์สถิติที่สำคัญประกอบด้วย:

* **ค่าเฉลี่ยเลขคณิต (Mean - $\mu$)**: ตัวแทนศูนย์กลางของข้อมูล
* **ค่าความแปรปรวน (Variance - $\sigma^2$)** และ **ส่วนเบี่ยงเบนมาตรฐาน (Standard Deviation - $\sigma$)**: ตัวชี้วัดความแปรปรวนและการกระจายตัวของข้อมูลจากค่าเฉลี่ย
  $$\sigma = \sqrt{\frac{1}{N} \sum_{i=1}^{N} (x_i - \mu)^2}$$

การวิเคราะห์ข้อมูลสภาพอากาศมักตั้งอยู่บนสมมติฐานการแจกแจงแบบปกติ (Normal Distribution) ภายใต้เกณฑ์ **"กฎเชิงประจักษ์" (Empirical Rule หรือ กฎ $3\sigma$)**:
* $68\%$ ของข้อมูลทั้งหมดจะอยู่ห่างจากค่าเฉลี่ยไม่เกิน $\pm 1\sigma$
* $95\%$ ของข้อมูลทั้งหมดจะอยู่ห่างจากค่าเฉลี่ยไม่เกิน $\pm 2\sigma$
* $99.7\%$ ของข้อมูลทั้งหมดจะอยู่ห่างจากค่าเฉลี่ยไม่เกิน $\pm 3\sigma$

หากพบว่าค่าอุณหภูมิในวันหนึ่งพุ่งเกินขอบเขต $+2\sigma$ แสดงว่าแปลงปลูกกำลังเผชิญกับสภาวะร้อนจัดผิดปกติระดับวิกฤต (Heatwave Anomaly) ซึ่งพืช เช่น ทุเรียน หรือมังคุด จะเริ่มกระตุ้นปฏิกิริยาป้องกันตนเองโดยการสลัดดอกผลทันที

## 4.5 อัลกอริทึมการตรวจจับสิ่งผิดปกติเชิงสถิติ
สิ่งผิดปกติ (Anomaly) ในชุดข้อมูลภูมิอากาศ สามารถบ่งชี้ถึงสองประเด็นสำคัญ: ปัญหาความล้มเหลวของฮาร์ดแวร์ หรือ การเกิดภัยพิบัติธรรมชาติเฉียบพลัน การแยกแยะสามารถประยุกต์ใช้เทคนิค **คะแนนมาตรฐาน (Z-Score)** ในการตีกรอบข้อมูลเชิงสถิติ:

$$Z = \frac{x_i - \mu}{\sigma}$$

ค่า Z-score บ่งบอกว่าข้อมูลจุดนั้นอยู่ห่างจากค่าเฉลี่ยของประชากรเป็นกี่เท่าของส่วนเบี่ยงเบนมาตรฐาน:
* **Spike Anomaly (การตรวจจับจุดแหลมผิดปกติ)**: หาก $|Z| > 3.0$ แสดงว่าข้อมูลจุดนั้นหลุดออกจากขอบเขตความน่าจะเป็นปกติ (น้อยกว่า 0.3%) มักเกิดจากเซ็นเซอร์ช็อตชั่วคราว หรือกระแสไฟฟ้าขัดข้อง
* **Trend Shift Anomaly (การเปลี่ยนรูปแนวโน้ม)**: การที่ค่าเฉลี่ยเคลื่อนที่ของข้อมูลเบี่ยงเบนออกจากค่าเฉลี่ยระยะยาวในทิศทางเดียวอย่างต่อเนื่อง บ่งชี้ถึงภัยแล้งทิ้งช่วงยาว หรือการอุดตันของระบบท่อน้ำหยด

## 4.6 การสังเคราะห์กรณีศึกษา: การวิเคราะห์ภัยแล้งและการสะสมความเย็นของลำไยในภาคตะวันออก
*กรณีศึกษาวิจัยเชิงลึก:*
เพื่อวิเคราะห์การออกดอกของลำไยในพื้นที่จังหวัดจันทบุรีและสระแก้ว ซึ่งต้องการอุณหภูมิอากาศที่หนาวเย็นต่ำกว่า $15.0$ °C ต่อเนื่องกันไม่น้อยกว่า $100$ ชั่วโมงเพื่อกระตุ้นฮอร์โมนชักนำการออกดอก คณะวิจัย RBRU-Praneet ได้ทำการรวบรวมข้อมูลอุณหภูมิรายชั่วโมงย้อนหลัง 5 ปี (43,800 ข้อมูล)

```
  [ ข้อมูลอุณหภูมิรายชั่วโมง (43,800 ค่า) ] ──► [ ลบสัญญาณรบกวนรายชั่วโมงด้วย EMA ]
                                           ──► [ คำนวณหาค่าเฉลี่ยสะสมรายสัปดาห์ ]
                                           ──► [ ตรวจจับช่วงความหนาวเย็นวิกฤต (Low-temp Anomaly) ]
```

ผลการวิเคราะห์อนุกรมเวลาชี้ชัดว่าในปี 2568 เกิดสภาวะ **"ฤดูหนาวอบอุ่น"** อันเป็นสิ่งผิดปกติเชิงสถิติ (Warm Winter Anomaly) ค่าเฉลี่ยอุณหภูมิรายสัปดาห์ในช่วงเดือนธันวาคมเบี่ยงเบนไปจากค่าเฉลี่ย 30 ปีถึง $+2.4\sigma$ และเมื่อตรวจวัดสะสมชั่วโมงหนาวเย็น (Chill Hours) พบว่าสะสมได้เพียง 42 ชั่วโมงตลอดทั้งฤดูกาล ส่งผลให้ลำไยแตกใบอ่อนแทนการออกดอก เกิดความเสียหายต่อเศรษฐกิจชุมชน

การวิเคราะห์ข้อมูลนี้ช่วยให้สหกรณ์การเกษตรปรับตัวได้อย่างทันท่วงที โดยเปลี่ยนมาประยุกต์ใช้สารพ่นกระตุ้นสารอาหารทางใบร่วมกับการงดน้ำอย่างเข้มงวดเพื่อสร้างภาวะแล้งเทียมทดแทนความเย็น ช่วยให้รักษาผลผลิตลำไยได้สูงถึง 80% ท่ามกลางปีที่โลกรวนรุนแรง

## 4.7 โค้ดคอมพิวเตอร์และแบบฝึกหัดทบทวน 3 ระดับ

เพื่อให้สอดคล้องกับยุทธศาสตร์การพัฒนาเกษตรกรอัจฉริยะ (RBRU-Praneet) โค้ดคอมพิวเตอร์และแบบฝึกหัดท้ายบทนี้ได้รับการพัฒนาและแบ่งโครงสร้างออกเป็น 3 ระดับชั้นเรียนอย่างเป็นระบบ:

### 🟩 ระดับที่ 1: มัธยมศึกษาตอนต้น (Junior High Level) - เน้นความเข้าใจพื้นฐาน
*ระบบคัดกรองอุณหภูมิวิกฤตและหาค่าเฉลี่ยสถิติเบื้องต้น* โค้ดนี้สอนทักษะการหาค่าเฉลี่ยเลขคณิตจากรายการข้อมูลรายวันและการตรวจหาค่าความร้อนที่เสี่ยงต่อการหลุดร่วงของดอกทุเรียนโดยไม่ใช้ Library ซับซ้อน

#### 💻 โค้ดภาษา Python สำหรับมัธยมต้น:
```python
# โปรแกรมวิเคราะห์ค่าเฉลี่ยอุณหภูมิและแจ้งเตือนภัยร้อน
# จัดทำโดย: นวัตกรน้อย RBRU-Praneet (ระดับมัธยมต้น)

# 1. รายการข้อมูลอุณหภูมิสูงสุดรายวันสะสมในสวนทุเรียน 7 วัน (°C)
temperatures = [35.5, 36.8, 38.2, 39.5, 37.0, 36.5, 39.0]

# 2. คำนวณหาค่าเฉลี่ยอุณหภูมิโดยใช้คณิตศาสตร์พื้นฐาน
total_sum = sum(temperatures)
count = len(temperatures)
average_temp = total_sum / count

print("=== รายงานสถิติอุณหภูมิสวนทุเรียนสะสม 7 วัน ===")
print("ข้อมูลรายวัน:", temperatures)
print(f"จำนวนวันบันทึกข้อมูล: {count} วัน")
print(f"อุณหภูมิเฉลี่ยรอบสัปดาห์: {average_temp:.2f} °C")

# 3. ค้นหาและแจ้งเตือนวันที่มีอุณหภูมิวิกฤต (สูงกว่า 38 องศาเซลเซียส)
print("\n--- ผลการตรวจจับวันวิกฤต ---")
critical_limit = 38.0
warning_days = 0

for i in range(len(temperatures)):
    temp = temperatures[i]
    if temp > critical_limit:
        print(f"วันที่ {i + 1}: พบลักษณะ [อากาศร้อนจัดวิกฤต] อุณหภูมิสูงถึง {temp} °C (เสี่ยงไหลดอกร่วง!)")
        warning_days += 1

print(f"สรุป: พบบรรยากาศวิกฤตรวมทั้งสิ้น {warning_days} วันในสัปดาห์นี้")
print("=============================================")
```

#### ❓ แบบฝึกหัดทบทวนระดับมัธยมต้น:
1. จงอธิบายลอจิกของการหาอุณหภูมิเฉลี่ยข้างต้น และคำนวณด้วยตนเองว่าหากมีข้อมูลวันที่ 8 ที่มีค่าผิดปกติสุดขั้ว เช่น อุณหภูมิกระโดดไป `48.0` °C จะส่งผลให้ค่าเฉลี่ยเปลี่ยนแปลงไปอย่างไร
2. เพิ่มเงื่อนไขตรวจสอบว่า หากอุณหภูมิเฉลี่ยรอบสัปดาห์สูงกว่า 37 องศาเซลเซียส ให้ระบบพิมพ์แจ้งเตือนว่า "สภาพอากาศร้อนสะสมต่อเนื่อง ให้เพิ่มความถี่การรดน้ำ"

### 🟨 ระดับที่ 2: มัธยมศึกษาตอนปลาย (Senior High Level) - เน้นอัลกอริทึม Moving Average และการตรวจจับความเบี่ยงเบน
*โปรแกรมกรองข้อมูลอากาศด้วย Simple Moving Average และการตรวจจับความผิดปกติเชิงสถิติขั้นต้น* โค้ดนี้สอนทักษะการวนลูปแบบหน้าต่างข้อมูลเคลื่อนที่ (Sliding Window) และการใช้ฟังก์ชันวิเคราะห์สถิติท้องถิ่น

#### 💻 โค้ดภาษา Python สำหรับมัธยมปลาย:
```python
# โปรแกรมวิเคราะห์สัญญาณข้อมูลอนุกรมเวลาภูมิอากาศเกษตร
# จัดทำโดย: นวัตกรสายวิทย์ RBRU-Praneet (ระดับมัธยมปลาย)

import math

def calculate_mean(data_list):
    """ฟังก์ชันหาค่าเฉลี่ย"""
    return sum(data_list) / len(data_list)

def calculate_std_dev(data_list, mean):
    """ฟังก์ชันคำนวณส่วนเบี่ยงเบนมาตรฐาน (Standard Deviation)"""
    variance = sum((x - mean) ** 2 for x in data_list) / len(data_list)
    return math.sqrt(variance)

def calculate_sma(data, window_size=3):
    """ฟังก์ชันคำนวณค่าเฉลี่ยเคลื่อนที่แบบง่าย (Simple Moving Average)"""
    sma_result = []
    for i in range(len(data)):
        if i < window_size - 1:
            # ช่วงวันแรกๆ ที่ยังมีข้อมูลย้อนหลังไม่พอสำหรับกรอบเวลา ให้คงค่าเดิมไว้
            sma_result.append(data[i])
        else:
            window = data[i - window_size + 1 : i + 1]
            sma_result.append(calculate_mean(window))
    return sma_result

# ข้อมูลดิบอุณหภูมิรายชั่วโมงในรอบวันที่มีสัญญาณรบกวนขัดข้อง
raw_temperatures = [32.0, 32.5, 33.0, 33.2, 45.0, 34.0, 33.5, 32.8, 32.2, 31.8, 12.0, 32.5]
# ค่า 45.0 และ 12.0 คือข้อมูลสัญญาณรบกวนผิดปกติที่เกิดจากไฟกระชากและเซ็นเซอร์ช็อต

# 1. คำนวณค่าทางสถิติพื้นฐานของข้อมูล
mean_val = calculate_mean(raw_temperatures)
std_val = calculate_std_dev(raw_temperatures, mean_val)
sma_temperatures = calculate_sma(raw_temperatures, window_size=3)

print("=== สถิติวิเคราะห์และกรองอนุกรมเวลาในแปลง ===")
print(f"ค่าเฉลี่ยดิบทั้งหมด: {mean_val:.2f} °C | ส่วนเบี่ยงเบนมาตรฐาน: {std_val:.2f}")
print("--------------------------------------------")

# 2. ตรวจจับสิ่งผิดปกติเชิงสถิติโดยใช้หลักการ 1.5-Sigma Rule
threshold = 1.5
for idx in range(len(raw_temperatures)):
    raw = raw_temperatures[idx]
    sma = sma_temperatures[idx]
    
    # คำนวณระดับความเบี่ยงเบนจากค่าเฉลี่ย
    deviation = abs(raw - mean_val) / std_val if std_val > 0 else 0
    
    if deviation > threshold:
        print(f"ชั่วโมงที่ {idx + 1}: ข้อมูลดิบ = {raw}°C [ผิดปกติสถิติ] (Z-Score = {deviation:.2f}) -> แก้ไขกรองค่าใหม่เป็น SMA = {sma:.2f}°C")
    else:
        print(f"ชั่วโมงที่ {idx + 1}: ข้อมูลดิบ = {raw}°C [ปกติ] -> กรองได้ค่า SMA = {sma:.2f}°C")
print("============================================")
```

#### ❓ แบบฝึกหัดทบทวนระดับมัธยมปลาย:
1. จงวิเคราะห์ว่าทำไมการแทนค่าจุดผิดปกติ (เช่น 45.0 °C ในช่วงบ่าย) ด้วยค่าเฉลี่ยเคลื่อนที่ (SMA) จึงให้ผลลัพธ์ที่ดีและปลอดภัยต่อการประมวลผลเชิงควบคุมมากกว่าการปล่อยให้ค่าดิบนั้นส่งตรงไปยังปั๊มน้ำ
2. หากปรับกรอบหน้าต่างของค่าเฉลี่ยเคลื่อนที่ (`window_size`) จาก 3 วัน เป็น 5 วัน จะทำให้เส้นกราฟผลลัพธ์มีความหน่วงต่อข้อมูลจริงเพิ่มขึ้นหรือลดลงอย่างไร จงอธิบายหลักสถิติเบื้องหลัง

### 🟦 ระดับที่ 3: อุดมศึกษา (Higher Education Level) - เน้นการใช้ Pandas, Numpy และการวิเคราะห์อนุกรมเวลาขั้นสูง
*ระบบประมวลผลท่อส่งข้อมูลสถิติขั้นสูงและการตรวจจับสิ่งผิดปกติสถิติแบบกรอบเวลาเคลื่อนที่รายสัปดาห์* โค้ดนี้ใช้ Library มาตรฐานอุตสาหกรรมวิทยาศาสตร์ข้อมูลอย่าง Pandas และ Numpy ในการล้างข้อมูล การประมาณค่าในช่วง และตรวจจับ Outlier ของแปลงปลูกระดับสากล

#### 💻 โค้ดภาษา Python สำหรับอุดมศึกษา:
```python
# ระบบวิศวกรรมข้อมูลอนุกรมเวลาสภาพภูมิอากาศเกษตรขั้นสูง (Advanced Agro-Climate Time-Series Pipeline)
# พัฒนาขึ้นภายใต้ยุทธศาสตร์ RBRU-Praneet (ระดับอุดมศึกษา)

import numpy as np
import pandas as pd

class ClimateTimeSeriesEngine:
    def __init__(self, df_sensor_stream):
        """
        กำหนดค่าข้อมูลนำเข้าในรูปแบบ Pandas DataFrame
        โครงสร้างต้องประกอบด้วยหัวคอลัมน์: ['timestamp', 'temperature', 'humidity']
        """
        self.df = df_sensor_stream.copy()
        # แปลงคอลัมน์เวลาเป็นดัชนีอนุกรมเวลา (Datetime Index) เพื่อเสถียรภาพการประมวลผล
        self.df['timestamp'] = pd.to_datetime(self.df['timestamp'])
        self.df.set_index('timestamp', inplace=True)
        
    def execute_pipeline(self, ema_span=5, z_threshold=2.0):
        """
        รันกระบวนการล้างข้อมูล ประมาณค่าสูญหาย กรองสัญญาณรบกวนด้วย EMA และตรวจจับ Outliers เชิงลึก
        """
        print("[กระบวนการ] เริ่มรันการประมวลผลท่อส่งข้อมูลสถิติอนุกรมเวลา...")
        
        # 1. จัดการและแทนที่ข้อมูลสูญหาย (NaN Imputation) ด้วยการปะติดช่วงเชิงเส้น (Linear Interpolation)
        missing_count = self.df['temperature'].isna().sum()
        if missing_count > 0:
            print(f"[ล้างข้อมูล] ตรวจพบข้อมูลอุณหภูมิสูญหาย {missing_count} ค่า ดำเนินการ Linear Interpolation...")
            self.df['temperature'] = self.df['temperature'].interpolate(method='linear')
            self.df['humidity'] = self.df['humidity'].interpolate(method='linear')
            
        # 2. กรองสัญญาณรบกวนความถี่สูงด้วย Exponential Moving Average (EMA)
        self.df['temp_ema'] = self.df['temperature'].ewm(span=ema_span, adjust=False).mean()
        
        # 3. คำนวณคะแนนมาตรฐานเคลื่อนที่ (Rolling Z-Score) ของการกระจายตัวข้อมูลอุณหภูมิรอบหน้าต่าง 5 ช่วงเวลา
        rolling_mean = self.df['temperature'].rolling(window=5, min_periods=1).mean()
        rolling_std = self.df['temperature'].rolling(window=5, min_periods=1).std().fillna(1.0) # กันฐานหารเป็นศูนย์
        
        self.df['rolling_z'] = (self.df['temperature'] - rolling_mean) / rolling_std
        
        # 4. ระบุจุดผิดปกติเชิงสถิติ (Outliers/Anomaly Detection)
        self.df['is_anomaly'] = np.abs(self.df['rolling_z']) > z_threshold
        
        print("[เสร็จสิ้น] ท่อส่งประมวลผลข้อมูลเสร็จสมบูรณ์")
        return self.df
        
    def generate_anomaly_report(self):
        """
        สกัดแยกข้อมูลส่วนที่พบสิ่งผิดปกติออกมาทำรายงานเพื่อการตัดสินใจ
        """
        anomalies = self.df[self.df['is_anomaly']]
        report = []
        for time_idx, row in anomalies.iterrows():
            report.append({
                "time": str(time_idx),
                "recorded_temp": float(row['temperature']),
                "ema_smoothed": float(row['temp_ema']),
                "z_score": float(row['rolling_z']),
                "status_alert": "CRITICAL_SPIKE" if row['rolling_z'] > 0 else "CRITICAL_DROP"
            })
        return report

# --- ทดสอบระบบประมวลผลจำลองสถานการณ์แปลงวิจัยทุเรียนหมอนทอง จันทบุรี ---
# สร้างชุดข้อมูลอนุกรมเวลา 12 ช่วงเวลาจำลอง
raw_data_stream = {
    "timestamp": pd.date_range(start="2026-05-20 08:00", periods=12, freq="H"),
    "temperature": [32.5, 33.1, np.nan, 34.0, 34.5, 48.2, 35.0, 34.2, 33.8, 11.5, 33.0, 32.8], # พบ NaN, Spike 48.2, Drop 11.5
    "humidity": [75.0, 74.2, 73.0, np.nan, 71.5, 35.0, 70.0, 71.2, 72.0, 95.0, 73.5, 74.0]
}

df_raw = pd.DataFrame(raw_data_stream)
engine = ClimateTimeSeriesEngine(df_raw)

# รัน Pipeline คัดกรอง
df_processed = engine.execute_pipeline(ema_span=4, z_threshold=2.0)

print("\n=== รายงานผลลัพธ์การประมวลผล (Pandas DataFrame) ===")
print(df_processed[['temperature', 'temp_ema', 'rolling_z', 'is_anomaly']])

# สกัดคำสั่งและวิเคราะห์สิ่งผิดปกติ
anomaly_report = engine.generate_anomaly_report()
print("\n=== สรุปจุดวิกฤตที่ต้องส่งแจ้งเตือนภัยฟาร์ม ===")
print(json.dumps(anomaly_report, indent=2, ensure_ascii=False))
```

#### ❓ แบบฝึกหัดทบทวนระดับอุดมศึกษา:
1. จงวิเคราะห์เชิงเปรียบเทียบระหว่างเทคนิค **Exponential Moving Average (EMA)** และเทคนิค **Kalman Filter (ตัวกรองคัลมาน)** ในการประมาณค่าและลดระดับสัญญาณรบกวนของความชื้นในดินระดับลึก โดยเน้นเปรียบเทียบในแง่ของทรัพยากรการประมวลผล ความซับซ้อนของโปรแกรม และมิติความหน่วงเวลา
2. พัฒนาส่วนขยายฟังก์ชันการวิเคราะห์เชิงทำนาย โดยนำแนวคิดถดถอยเชิงเส้นแบบหน้าต่างข้อมูลเคลื่อนที่ (Rolling Linear Regression) ไปประยุกต์ใช้เพื่อประมาณการค่าความชื้นสัมพัทธ์ในแปลงล่วงหน้า 3 ชั่วโมง พร้อมวิเคราะห์ผลกระทบในกรณีที่ชุดข้อมูลเกิดสภาวะสูญเสียสัญญาณติดต่อกันเกิน 10 ชั่วโมง
