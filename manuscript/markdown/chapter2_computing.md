## 2.1 โครงสร้างและวากยสัมพันธ์ของภาษาไพทอนสำหรับการคำนวณ
ภาษาไพทอนได้รับความนิยมสูงสุดในฐานะภาษาหลักสำหรับงานวิทยาศาสตร์ข้อมูลและเทคโนโลยีเกษตรอัจฉริยะ โครงสร้างที่ออกแบบมาให้มีวากยสัมพันธ์ (Syntax) ที่สะอาด อ่านออกเขียนง่าย คล้ายคลึงกับภาษาอังกฤษ ทำให้โปรแกรมเมอร์สามารถมุ่งเน้นไปที่การแก้ปัญหาเชิงตรรกะของการเกษตร มากกว่าการจัดการหน่วยความจำคอมพิวเตอร์ในระดับล่าง

ในมิติของการคำนวณทางเกษตรศาสตร์ ภาษาไพทอนทำหน้าที่เป็น **"เครื่องมือแปลความหมายธรรมชาติ" (Natural-to-Digital Translator)** ซึ่งทำหน้าที่แปลงค่าพารามิเตอร์ทางกายภาพของสิ่งแวดล้อม เช่น อุณหภูมิ ความเร็วลม ความชื้นสัมพัทธ์ในแปลง ให้กลายเป็นตัวแปรและโครงสร้างที่คอมพิวเตอร์สามารถใช้คำนวณเชิงคณิตศาสตร์ได้

```
  [ โลกกายภาพ (Sensory) ] ───► [ อุณหภูมิ 38.5°C ] ───► [ ตัวแปลภาษา Python ] ───► temperature = 38.5 (Float)
```

## 2.2 การจัดการประเภทข้อมูลเชิงวิทยาศาสตร์
การพัฒนาซอฟต์แวร์เพื่อรองรับเกษตรอัจฉริยะจำเป็นต้องเข้าใจประเภทข้อมูลพื้นฐานและประเภทข้อมูลเชิงโครงสร้าง (Data Types & Structures) เพื่อการจัดเก็บข้อมูลสภาพแวดล้อมและพืชอย่างมีประสิทธิภาพ:

### 1. ประเภทข้อมูลพื้นฐาน (Primitive Data Types)
* **ข้อมูลจำนวนเต็ม (Integers - `int`)**: ใช้สำหรับพารามิเตอร์ที่ไม่มีค่าทศนิยม เช่น จำนวนผลทุเรียนบนต้น, จำนวนวันสะสมอาหาร, หรือจำนวนชั่วโมงที่แสงสว่าง
* **ข้อมูลจำนวนจริง (Floating-Point Numbers - `float`)**: ใช้สำหรับพารามิเตอร์ทางวิทยาศาสตร์ที่มีค่าทศนิยมละเอียด เช่น ค่าความเป็นกรด-ด่างในดิน (pH), อุณหภูมิอากาศ, หรือค่าความนำไฟฟ้าของสารละลายปุ๋ย (Electrical Conductivity - EC)
* **ข้อมูลอักขระ (Strings - `str`)**: ใช้สำหรับระบุคุณลักษณะเชิงบรรยาย เช่น ชื่อพืชเศรษฐกิจ (`"หมอนทอง"`, `"มังคุด"`), ระยะการเจริญเติบโต (`"ระยะไหลดอก"`, `"ระยะติดผล"`), หรือสถานะระบบเตือนภัย
* **ข้อมูลตรรกะ (Booleans - `bool`)**: มีค่าจริง (`True`) หรือเท็จ (`False`) เท่านั้น ใช้ในการระบุสถานะการทำงานทางกายภาพของเครื่องจักรกลการเกษตร เช่น สถานะปั๊มน้ำเปิด (`True`) หรือสถานะวาล์วไฟฟ้าปิด (`False`)

### 2. โครงสร้างข้อมูลเชิงวิทยาศาสตร์ (Scientific Data Structures)
* **รายการข้อมูล (Lists - `list`)**: ลำดับข้อมูลที่สามารถเปลี่ยนแปลงค่าได้ เหมาะสำหรับจัดเก็บข้อมูลอนุกรมเวลา (Time-series Data) รายชั่วโมง เช่น `hourly_temp = [28.5, 29.0, 31.2, 34.5, 36.0]`
* **ชุดข้อมูลคู่คีย์-ค่า (Dictionaries - `dict`)**: การจัดเก็บบนสถาปัตยกรรม "คู่คีย์และค่า" (Key-Value Pairs) เหมาะอย่างยิ่งสำหรับจัดกลุ่มข้อมูลที่มีโครงสร้างยืดหยุ่นของสถานีตรวจวัดภูมิอากาศ เช่น:
  ```python
  station_data = {
      "station_id": "RBRU-PRANEET-001",
      "crop_target": "ทุเรียนหมอนทอง",
      "sensors": {"temp": 32.4, "humidity": 72.5}
  }
  ```

## 2.3 การควบคุมทิศทางการประมวลผลสำหรับเงื่อนไขทางชีวภาพ
ความแม่นยำของการควบคุมอัตโนมัติในงานเกษตรอัจฉริยะขึ้นอยู่กับตรรกะเงื่อนไข (Conditional Logic) ในการตอบสนองต่อสรีรวิทยาและข้อจำกัดเชิงนิเวศน์ของพืชแต่ละชนิดผ่านคำสั่งโครงสร้างควบคุม `if-elif-else`:

```
                           ┌─────────────────────────┐
                           │   ตรวจวัดสภาพสิ่งแวดล้อม  │
                           └────────────┬────────────┘
                                        │
                                        ▼
                         /─────────────────────────────\
                        < อุณหภูมิ > 35°C และ ความชื้น < 45% >
                         \─────────────────────────────/
                                        │
                           ┌────────────┴────────────┐
                        ใช่ │                     ไม่ │
                            ▼                         ▼
                 [ เปิดปั๊มพ่นหมอกระบายความร้อน ]     [ รักษาสถานะปกติ ]
```

ตัวอย่างการกำหนดเกณฑ์การควบคุมระดับน้ำและปุ๋ยในพืชเศรษฐกิจต่างๆ:
* **ทุเรียน**: หากความชื้นดินระดับราก (30 ซม.) ต่ำกว่า 18% และอากาศร้อนจัด ให้สั่งรดน้ำทันทีเพื่อป้องกันใบร่วงสลัดผล
* **ยางพารา**: หากความชื้นสัมพัทธ์ในรอบเช้ามืดต่ำกว่า 70% และมีลมแรง ไม่ควรกรีดยางเนื่องจากอัตราการระเหยสูงทำให้ท่อน้ำยางแห้งเร็วเกินไป
* **มันสำปะหลัง**: หากตรวจพบฝนสะสมสัปดาห์สูงเกิน 200 มม. และดินอุ้มน้ำปนทรายแป้งหนาแน่น ให้ปิดระบบน้ำหยดถาวรเพื่อป้องกันการเกิดรากเน่า

## 2.4 สถาปัตยกรรมฟังก์ชันและการเขียนโปรแกรมเชิงโมดูล
เพื่อให้อัลกอริทึมทางเกษตรสามารถขยายผลและใช้งานซ้ำได้ในแปลงปลูกนับพันแปลง เราจึงต้องปฏิบัติตามหลักการเขียนโปรแกรมเชิงโมดูล (Modular Programming) โดยการห่อหุ้มตรรกะคอมพิวเตอร์ให้อยู่ในรูปของ **"ฟังก์ชัน" (Functions)** ผ่านคีย์เวิร์ด `def`

ประโยชน์ของการสร้างฟังก์ชันในงานเกษตรดิจิทัล:
1. **ลดความซ้ำซ้อนของรหัส**: เขียนสูตรคำนวณการคายน้ำของพืชหรือค่าดัชนีอุณหภูมิเพียงครั้งเดียว แต่นำไปใช้กับเซ็นเซอร์ทุกตัวในฟาร์มได้
2. **ง่ายต่อการปรับแต่งค่าสัมประสิทธิ์**: หากมีการเปลี่ยนพืชเป้าหมายจากทุเรียนเป็นมังคุด เกษตรกรเพียงแค่เปลี่ยนพารามิเตอร์ที่ส่งเข้าไปในฟังก์ชันเท่านั้น โดยไม่ต้องแก้ไขลอจิกโปรแกรมหลัก
3. **การทดสอบความถูกต้องขั้นสูง**: สามารถระบุข้อมูลทดสอบเพื่อยืนยันว่าฟังก์ชันคำนวณผลลัพธ์ได้อย่างแม่นยำก่อนนำไปรันบนระบบควบคุมฮาร์ดแวร์จริง

## 2.5 การจัดการข้อผิดพลาดในการรับส่งข้อมูลเซ็นเซอร์ภาคสนาม
ในสภาพแวดล้อมจริง สวนเกษตรเป็นพื้นที่ที่มีสัญญาณรบกวน (Noise) สูงมาก อุปกรณ์และสายสัญญาณต้องเผชิญกับสภาพอากาศร้อนจัด ความชื้นสูง และการรบกวนจากสัตว์ในแปลง ส่งผลให้ข้อมูลที่ส่งมาจากเซ็นเซอร์มักเกิดความสูญหาย (Data Loss) หรือเกิดค่าที่ผิดปกติเฉียบพลัน เช่น ค่าอุณหภูมิกระโดดไปที่ `999.0` หรือส่งค่าว่าง (`None`) กลับมา

หากซอฟต์แวร์ไม่มีกลไก **"การจัดการข้อผิดพลาด" (Exception Handling)** ด้วยคำสั่ง `try-except` โปรแกรมจะหยุดทำงานและระบบล่ม (Crash) ทันที ซึ่งในระบบเกษตรระดับอุตสาหกรรมจะนำไปสู่ความสูญเสียมหาศาล เช่น ระบบวาล์วน้ำไม่ทำงานจนต้นไม้แห้งตาย

```python
try:
    raw_reading = sensor.read_temperature()
    # ตรวจสอบค่าคลาดเคลื่อนทางฟิสิกส์ (Physical Boundary Check)
    if raw_reading < -10 or raw_reading > 60:
        raise ValueError("ค่าอุณหภูมินอกขอบเขตฟิสิกส์ที่เป็นไปได้ในธรรมชาติ")
    temperature = float(raw_reading)
except (TypeError, ValueError) as e:
    print(f"เกิดข้อผิดพลาดในการอ่านค่าเซ็นเซอร์: {e}")
    # ใช้กลยุทธ์ชดเชยค่าข้อมูลสูญหาย (Fallback Value Strategy)
    temperature = 25.0  # กำหนดค่ามาตรฐานเพื่อความปลอดภัยของระบบ
```

## 2.6 ทฤษฎีและการสังเคราะห์คำนวณค่าดัชนีความร้อนสะสม (Growing Degree Days)
ดัชนีความร้อนสะสม (Growing Degree Days - GDD) คือดัชนีทางชีวภูมิอากาศศาสตร์ที่ใช้วัดและสะสมปริมาณพลังงานความร้อนที่พืชได้รับในแต่ละวัน เพื่อพยากรณ์การเจริญเติบโตของพืชตลอดวัฏจักรชีวิต เช่น การแตกยอด การออกดอก และวันที่ผลผลิตจะสุกพร้อมเก็บเกี่ยว แทนการนับจำนวนวันปฏิทินแบบเดิมที่ไม่มีความแม่นยำเนื่องจากความร้อนในแต่ละปีผันแปรสูง

### สมการทางคณิตศาสตร์มาตรฐานในการคำนวณค่า GDD รายวัน:
$$GDD = \max\left(\frac{T_{\max} + T_{\min}}{2} - T_{\text{base}}, 0\right)$$

โดยที่:
* $T_{\max}$ คือ อุณหภูมิอากาศสูงสุดที่ตรวจวัดได้ในวันนั้น (°C)
* $T_{\min}$ คือ อุณหภูมิอากาศต่ำสุดที่ตรวจวัดได้ในวันนั้น (°C)
* $T_{\text{base}}$ คือ อุณหภูมิฐาน (Base Temperature) ของพืชเป้าหมาย ซึ่งเป็นอุณหภูมิต่ำสุดที่เริ่มมีการทำกิจกรรมทางชีวเคมีและการเจริญเติบโต หากอุณหภูมิต่ำกว่าค่านี้ พืชจะหยุดการเจริญเติบโตโดยสมบูรณ์

**เกณฑ์อุณหภูมิฐาน ($T_{\text{base}}$) ของพืชเศรษฐกิจสำคัญของไทย**:
* **ทุเรียนหมอนทอง**: $18.0$ °C (หากต่ำกว่านี้ระบบเอนไซม์ทำงานได้น้อยมาก)
* **ข้าว**: $10.0$ °C
* **อ้อย**: $12.0$ °C
* **มันสำปะหลัง**: $15.0$ °C
* **ยางพารา**: $15.0$ °C

หากความร้อนเฉลี่ยรายวันมีค่าน้อยกว่าหรือเท่ากับอุณหภูมิฐาน ค่า GDD ในวันนั้นจะถูกกำหนดให้เท่ากับ $0$ เพื่อไม่ให้เกิดค่าติดลบสะสมในสมการ

## 2.7 โค้ดคอมพิวเตอร์และแบบฝึกหัดทบทวน 3 ระดับ

เพื่อให้สอดคล้องกับยุทธศาสตร์การพัฒนาเกษตรกรอัจฉริยะ (RBRU-Praneet) โค้ดคอมพิวเตอร์และแบบฝึกหัดท้ายบทนี้ได้รับการพัฒนาและแบ่งโครงสร้างออกเป็น 3 ระดับชั้นเรียนอย่างเป็นระบบ:

### 🟩 ระดับที่ 1: มัธยมศึกษาตอนต้น (Junior High Level) - เน้นความเข้าใจพื้นฐาน
*ระบบตรวจสอบการขาดน้ำของต้นทุเรียนเบื้องต้น* โค้ดนี้ใช้สำหรับการศึกษาตรรกะการเปรียบเทียบตัวเลขและทักษะการใช้งานตัวแปรพื้นฐานในภาษาไพทอน

#### 💻 โค้ดภาษา Python สำหรับมัธยมต้น:
```python
# โปรแกรมระบบแจ้งเตือนการรดน้ำทุเรียนอย่างง่าย
# จัดทำโดย: นวัตกรน้อย RBRU-Praneet (ระดับมัธยมต้น)

# 1. กำหนดค่าตัวแปรระดับความชื้นในดิน (%)
soil_moisture = 16.5  
crop_name = "ทุเรียนหมอนทอง"

# 2. แสดงสถานะเริ่มต้น
print("--- ระบบวิเคราะห์สถานะฟาร์มเบื้องต้น ---")
print("พืชเป้าหมาย:", crop_name)
print("ความชื้นในดินปัจจุบัน:", soil_moisture, "%")

# 3. ใช้โครงสร้างควบคุมเงื่อนไขเพื่อตัดสินใจรดน้ำ
# เกณฑ์การรดน้ำ: หากความชื้นต่ำกว่า 18% ถือว่าวิกฤต
if soil_moisture < 18.0:
    print("แจ้งเตือน: [ดินแห้งเกินไป] สั่งการเปิดปั๊มน้ำรดต้นทุเรียนด่วน!")
    pump_status = True
else:
    print("แจ้งเตือน: [ความชื้นเหมาะสม] ดินมีความชื้นเพียงพอ ปิดปั๊มน้ำไว้")
    pump_status = False

print("สถานะปั๊มน้ำปัจจุบัน:", pump_status)
print("---------------------------------------")
```

#### ❓ แบบฝึกหัดทบทวนระดับมัธยมต้น:
1. จงดัดแปลงโค้ดข้างต้นเพื่อปรับเกณฑ์การตัดสินใจเป็นพืช **"มันสำปะหลัง"** โดยหากความชื้นในดินมีค่าสูงกว่า 80% ให้แจ้งเตือนว่า "เสี่ยงรากเน่า ปิดระบบน้ำด่วน!"
2. หากอุณหภูมิอากาศมีค่าสูงกว่า 35 องศาเซลเซียส ให้ระบบสั่งเปิดระบบพ่นหมอกเสริม จงแสดงวิธีการเพิ่มเงื่อนไข `if` ซ้อนลงไปในโปรแกรม

### 🟨 ระดับที่ 2: มัธยมศึกษาตอนปลาย (Senior High Level) - เน้นโครงสร้างฟังก์ชันและรายการข้อมูล
*โมดูลคำนวณค่าดัชนีความร้อนสะสม GDD รายวันและรายสัปดาห์* โค้ดนี้ใช้สำหรับการเรียนรู้การวนลูป การสร้างฟังก์ชัน การรับค่าจากรายการข้อมูล และการจัดการข้อผิดพลาดเบื้องต้น

#### 💻 โค้ดภาษา Python สำหรับมัธยมปลาย:
```python
# โปรแกรมวิเคราะห์ค่าดัชนีความร้อนสะสม (GDD) สำหรับการเกษตร
# จัดทำโดย: นวัตกรสายวิทย์ RBRU-Praneet (ระดับมัธยมปลาย)

def calculate_daily_gdd(t_max, t_min, t_base=18.0):
    """
    ฟังก์ชันคำนวณค่า GDD รายวัน สำหรับทุเรียนหมอนทอง
    """
    try:
        # 1. ป้องกันข้อผิดพลาดกรณีที่ส่งข้อมูลตัวเลขติดลบหรือผิดประเภท
        t_max = float(t_max)
        t_min = float(t_min)
        
        # 2. คำนวณอุณหภูมิเฉลี่ย
        t_avg = (t_max + t_min) / 2.0
        
        # 3. หักลบด้วยอุณหภูมิฐาน หากค่าเฉลี่ยต่ำกว่าฐาน ให้ GDD เป็น 0
        gdd = t_avg - t_base
        if gdd < 0.0:
            gdd = 0.0
            
        return gdd
    except ValueError:
        print("คำเตือน: ข้อมูลสภาพอากาศไม่อยู่ในรูปแบบตัวเลข กำหนดค่า GDD = 0")
        return 0.0

# 4. ทดลองคำนวณข้อมูลสภาพอากาศรายสัปดาห์ในสวนทุเรียนจันทบุรี
max_temperatures = [38.5, 39.0, 37.5, 34.0, 36.5, 38.0, 39.5]
min_temperatures = [24.5, 25.0, 26.0, 23.5, 24.0, 25.0, 26.5]
base_temp_durian = 18.0

print("--- รายงานดัชนีความร้อนสะสมทุเรียนรายสัปดาห์ ---")
weekly_total_gdd = 0.0

# 5. วนลูปผ่านรายการข้อมูลอุณหภูมิสูงสุด-ต่ำสุด เพื่อสะสมค่า GDD
for day in range(len(max_temperatures)):
    daily_gdd = calculate_daily_gdd(max_temperatures[day], min_temperatures[day], base_temp_durian)
    weekly_total_gdd += daily_gdd
    print(f"วันที่ {day + 1}: T_Max = {max_temperatures[day]}°C, T_Min = {min_temperatures[day]}°C | GDD วันนี้ = {daily_gdd:.2f}")

print(f"\nค่าความร้อนสะสมรวม 7 วัน (Weekly Total GDD): {weekly_total_gdd:.2f} GDD")
print("---------------------------------------------")
```

#### ❓ แบบฝึกหัดทบทวนระดับมัธยมปลาย:
1. หากเปลี่ยนพืชทดสอบจากทุเรียนเป็น **"ข้าวนาปี"** ที่มีอุณหภูมิฐาน ($T_{\text{base}}$) เท่ากับ $10.0$ °C จงคำนวณหาค่าสะสมความร้อนรวมรายสัปดาห์ และเปรียบเทียบความแตกต่างกับทุเรียนหมอนทอง
2. จงเพิ่มเติมกลไก `try-except` ในลูปหลักเพื่อป้องกันปัญหากรณีมีค่าแปลกปลอม เช่น ข้อมูลเป็นค่าว่างในบางวัน (`None`) โดยให้ข้ามการคำนวณวันนั้นไปโดยที่โปรแกรมไม่หยุดการทำงาน

### 🟦 ระดับที่ 3: อุดมศึกษา (Higher Education Level) - เน้นการเขียนโปรแกรมเชิงวัตถุและความแข็งแกร่งของระบบ
*กรอบระบบวิศวกรรมข้อมูลคลาสคำนวณและประมวลผลดัชนีสรีรวิทยาพืชระดับอุตสาหกรรม* โค้ดนี้ประยุกต์ใช้การเขียนโปรแกรมเชิงวัตถุ (Object-Oriented Programming - OOP) มีระบบชดเชยค่าข้อมูลสูญหาย (Data Imputation) และการจัดการกับเซ็นเซอร์ปริมาณมากที่ทนทานสูง

#### 💻 โค้ดภาษา Python สำหรับอุดมศึกษา:
```python
# กรอบระบบประมวลผลภูมิอากาศเกษตรอัจฉริยะขั้นสูง (Industrial-Grade Agri-Computing Framework)
# พัฒนาขึ้นภายใต้ยุทธศาสตร์ RBRU-Praneet (ระดับอุดมศึกษา)

class CropPhenologyCalculator:
    def __init__(self, crop_name, base_temp, target_gdd):
        """
        คอนสตรัคเตอร์กำหนดคุณลักษณะเฉพาะสรีรวิทยาพืชแต่ละชนิด
        """
        self.crop_name = crop_name
        self.base_temp = float(base_temp)
        self.target_gdd = float(target_gdd)
        self.accumulated_gdd = 0.0
        self.raw_history = []
        
    def process_daily_data(self, day_index, t_max, t_min):
        """
        ประมวลผลข้อมูลอุณหภูมิรายวัน พร้อมระบบตรวจสอบและชดเชยข้อมูลขาดหาย (Data Imputation)
        """
        try:
            # 1. การล้างข้อมูลและตรวจสอบค่าผิดพลาดทางกายภาพ
            if t_max is None or t_min is None:
                raise ValueError("ข้อมูลสภาพอากาศมีค่าว่าง (Missing Value Detected)")
                
            t_max = float(t_max)
            t_min = float(t_min)
            
            if t_max < t_min:
                raise ValueError("ความคลาดเคลื่อนทางฟิสิกส์: T_Max น้อยกว่า T_Min")
                
            if not (-10.0 <= t_max <= 65.0) or not (-10.0 <= t_min <= 60.0):
                raise ValueError("ข้อมูลเกินขีดจำกัดจริงทางอุตุนิยมวิทยา")
                
        except (TypeError, ValueError) as e:
            # 2. กลยุทธ์ชดเชยข้อมูลสูญหายโดยใช้ค่าเฉลี่ยย้อนหลัง 3 วันก่อนหน้า
            print(f"[ข้อผิดพลาดเชิงระบบ] วันที่ {day_index}: {e}. เริ่มใช้แผนกู้ข้อมูลสำรอง...")
            t_max, t_min = self._impute_missing_values()
            print(f"[การแก้ปัญหา] แทนที่ค่าด้วย T_Max_Imputed = {t_max:.1f}°C, T_Min_Imputed = {t_min:.1f}°C")
            
        # 3. ขั้นตอนการคำนวณและบันทึก
        t_avg = (t_max + t_min) / 2.0
        daily_gdd = max(t_avg - self.base_temp, 0.0)
        self.accumulated_gdd += daily_gdd
        self.raw_history.append({"day": day_index, "t_max": t_max, "t_min": t_min, "gdd": daily_gdd})
        return daily_gdd
        
    def _impute_missing_values(self):
        """
        อัลกอริทึมชดเชยข้อมูลด้วยค่าเฉลี่ยเคลื่อนที่แบบหน้าต่างยืดหยุ่น (Window Moving Average)
        """
        if len(self.raw_history) >= 3:
            recent_days = self.raw_history[-3:]
            avg_max = sum(day["t_max"] for day in recent_days) / 3.0
            avg_min = sum(day["t_min"] for day in recent_days) / 3.0
            return avg_max, avg_min
        else:
            # กรณีไม่มีข้อมูลย้อนหลังเพียงพอ ให้ใช้ค่าภูมิอากาศมาตรฐานภูมิภาคจันทบุรี
            return 35.0, 24.0
            
    def get_growth_progress(self):
        """
        ประเมินและสะท้อนร้อยละความพร้อมในการเก็บเกี่ยวผลผลิตตามเป้าหมาย
        """
        percentage = (self.accumulated_gdd / self.target_gdd) * 100.0
        return min(percentage, 100.0)

# --- การรันระบบจำลองระดับอุตสาหกรรมในแปลงทุเรียนหมอนทอง ---
# เป้าหมายสะสม GDD ทุเรียนหมอนทองระยะดอกถึงเก็บเกี่ยวประมาณ 1800 GDD
durian_processor = CropPhenologyCalculator(
    crop_name="ทุเรียนหมอนทอง",
    base_temp=18.0,
    target_gdd=1800.0
)

# ข้อมูลสภาพภูมิอากาศทดลองจริง 10 วันที่มีข้อมูลสูญหายและข้อมูลขัดข้องเชิงเครือข่าย
climate_stream = [
    {"day": 1, "t_max": 37.0, "t_min": 25.0},
    {"day": 2, "t_max": 38.5, "t_min": 26.0},
    {"day": 3, "t_max": 36.0, "t_min": 24.5},
    {"day": 4, "t_max": None, "t_min": 25.0},      # ข้อมูลสูญหาย
    {"day": 5, "t_max": 39.0, "t_min": 25.5},
    {"day": 6, "t_max": 20.0, "t_min": 27.0},      # อุณหภูมิผิดพลาดเชิงฟิสิกส์ (Max < Min)
    {"day": 7, "t_max": 38.0, "t_min": 26.0},
    {"day": 8, "t_max": 999.0, "t_min": 25.0},     # ข้อมูลเกิดสัญญาณรบกวนขีดจำกัด
    {"day": 9, "t_max": 37.5, "t_min": 24.0},
    {"day": 10, "t_max": 38.0, "t_min": 25.0}
]

print("=== เริ่มเปิดระบบวิศวกรรมการประมวลผลข้อมูล RBRU-Praneet ===")
for data in climate_stream:
    gdd = durian_processor.process_daily_data(data["day"], data["t_max"], data["t_min"])
    print(f"-> วันที่ {data['day']}: ผลลัพธ์ GDD สุทธิ = {gdd:.2f} | สะสมแล้ว = {durian_processor.accumulated_gdd:.2f} GDD")
    
progress = durian_processor.get_growth_progress()
print(f"===========================================================")
print(f"ความก้าวหน้าการสะสมความร้อนของทุเรียน: {progress:.2f}% ของเป้าหมาย {durian_processor.target_gdd} GDD")
print("===========================================================")
```

#### ❓ แบบฝึกหัดทบทวนระดับอุดมศึกษา:
1. จงขยายโครงสร้างคลาส `CropPhenologyCalculator` โดยให้รองรับการคำนวณค่า **สะสมดัชนีความเย็น (Chill Hours)** สำหรับพืชเมืองหนาวในภาคเหนือของไทย พร้อมทั้งระบุรูปแบบการบันทึกรายงานผลลัพธ์ลงเป็นไฟล์ JSON บนดิสก์อย่างเป็นระบบ
2. จงออกแบบและวิเคราะห์เชิงประสิทธิภาพ (Time Complexity) ของอัลกอริทึมการชดเชยข้อมูลแบบการปะติดช่วงเชิงเส้น (Linear Interpolation) เพื่อใช้แทนการใช้ค่าเฉลี่ยแบบเดิม ในกรณีที่เซ็นเซอร์ขัดข้องติดต่อกันเกินกว่า 5 วันขึ้นไป
