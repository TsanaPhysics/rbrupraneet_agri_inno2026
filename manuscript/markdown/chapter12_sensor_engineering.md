## 🎯 วัตถุประสงค์การเรียนรู้ (Learning Objectives)
เมื่อศึกษาบทนี้จบแล้ว ผู้เรียนจะสามารถ:
1. **วิเคราะห์ (Analyze)** หลักการทางฟิสิกส์เชิงลึกของเครื่องตรวจวัดอุณหภูมิและความชื้น (Sensor) ทั้งประเภทเก็บประจุและวัดความต้านทานที่ใช้ในงานเกษตรกรรมแม่นยำสูง
2. **ประเมิน (Evaluate)** และเลือกใช้สถาปัตยกรรมทางคณิตศาสตร์ในการกรองสัญญาณ ทั้งแบบตัวกรองในฮาร์ดแวร์และอัลกอริทึมซอฟต์แวร์เพื่อลดทอนสัญญาณรบกวน (Noise)
3. **อนุพัทธ์ (Derive)** สมการการถดถอยเชิงเส้น (Linear Regression) และสร้างแบบจำลองสมการระดับความชื้นดินเพื่อใช้ในระบบการสอบเทียบ (Calibration) ปลายทาง
4. **ออกแบบ (Design)** ตัวแปลงสัญญาณอนาล็อกเป็นดิจิทัล (Analog-to-Digital Converter - ADC) พร้อมกำหนดระดับความละเอียดของบิตและวิเคราะห์ข้อผิดพลาดจากการควอนไทซ์
5. **สังเคราะห์ (Synthesize)** แผนงานและสถาปัตยกรรมระบบบริหารจัดการพลังงาน (Power Management) เพื่อสร้างโหนดเซ็นเซอร์สนามที่ทนทานและประหยัดกำลังไฟฟ้า

## 12.1 ทฤษฎีการทำงานของเซ็นเซอร์เกษตรอัจฉริยะ (Physics of Sensors)

ในการทำวิจัยเกษตรกรรมดิจิทัล ข้อมูลคือทรัพยากรที่มีค่าที่สุด และเซ็นเซอร์คือเครื่องมือแรกเริ่มที่ทำหน้าที่เก็บเกี่ยวข้อมูลเหล่านั้นจากระบบนิเวศ การทำความเข้าใจกลไกทางฟิสิกส์ที่อยู่เบื้องหลังการแปลงค่าทางกายภาพให้เป็นค่าทางไฟฟ้ามีความสำคัญยิ่งต่อการนำไปใช้งานและการออกแบบสถาปัตยกรรมคัดกรองข้อมูล

```
  [ มิติทางกายภาพ ] ─────────> [ การเปลี่ยนแปลงทางฟิสิกส์ ] ─────────> [ ผลลัพธ์ไฟฟ้า ]
   ความชื้นดินสัมบูรณ์            ค่าคงที่ไดอิเล็กทริกแปรเปลี่ยน            ค่าความเก็บประจุ (pF)
   อุณหภูมิอากาศโดยรอบ           ความต้านทานภายในเปลี่ยนตามความร้อน       ค่าความดันไฟฟ้า (V)
```

### 12.1.1 เซ็นเซอร์วัดความชื้นดินแบบคาปาซิทีฟ (Capacitive Soil Moisture Sensor)
ในอดีต เซ็นเซอร์วัดดินนิยมใช้แบบความต้านทานสัมพัทธ์ (Resistive) ซึ่งอาศัยระยะห่างของแท่งโลหะสองแท่งเพื่อวัดการนำไฟฟ้าผ่านดิน แต่วิธีนี้ทำให้เกิดการกัดกร่อนจากปฏิกิริยาเคมีไฟฟ้า (Electrolysis) ส่งผลให้ขั้วโลหะผุกร่อนอย่างรวดเร็วภายในเวลาไม่กี่สัปดาห์ และค่าที่ได้มักเพี้ยนไปตามระดับเกลือและปุ๋ยเคมีที่ใส่ในดิน

เพื่อแก้ไขปัญหานี้ วิศวกรรมสมัยใหม่ได้เปลี่ยนมาใช้ระบบคาปาซิทีฟหรือวัดค่าความเก็บประจุ (Capacitance) ซึ่งห่อหุ้มขั้วแผ่นโลหะนำไฟฟ้าไว้ภายใต้เรซินหรือแผ่นพีซีบีฉนวนไฟฟ้าหนา ทำให้โลหะไม่สัมผัสกับดินโดยตรงและไม่มีวันผุกร่อน โดยมีทฤษฎีทางฟิสิกส์ที่อธิบายการคำนวณค่าความเก็บประจุ ($C$) ดังนี้:

$$C = \epsilon_r \epsilon_0 \frac{A}{d}$$

เมื่อตัวแปรแต่ละตัวบ่งบอกค่าดังต่อไปนี้:
* $\epsilon_r$ คือค่าความยอมรับได้สัมพัทธ์ (Relative Permittivity) หรือค่าคงที่ไดอิเล็กทริกของตัวกลางรอบใบเซ็นเซอร์
* $\epsilon_0$ คือค่าความยอมรับได้สุญญากาศ (Vacuum Permittivity) ซึ่งเป็นค่าคงที่สากลทางฟิสิกส์ ($\approx 8.854 \times 10^{-12} \text{ F/m}$)
* $A$ คือพื้นที่ของแผ่นตัวนำไฟฟ้าในใบวัด
* $d$ คือระยะห่างระหว่างแผ่นตัวนำไฟฟ้าทั้งสอง

> [!NOTE]
> หัวใจของระบบนี้อยู่ที่ค่า $\epsilon_r$ ซึ่งมีคุณสมบัติเฉพาะตามชนิดวัสดุ โดยอากาศมีค่า $\epsilon_r \approx 1$, ดินแห้งมีค่า $\epsilon_r \approx 3 - 4$, ในขณะที่น้ำมีค่า $\epsilon_r \approx 80$ (สูงกว่าดินแห้งถึง 20 เท่า!) ดังนั้น เมื่อปริมาณน้ำในดินรอบเซ็นเซอร์เพิ่มขึ้นเพียงเล็กน้อย ค่า $\epsilon_r$ ของดินผสมจะพุ่งสูงขึ้นอย่างรวดเร็ว ส่งผลให้ค่าความเก็บประจุ $C$ สูงขึ้นตามสมการ นวัตกรจะใช้วงจรกำเนิดความถี่บนตัวบอร์ดเพื่อเปลี่ยนค่าความเก็บประจุนี้ให้กลายเป็นค่าแรงดันไฟฟ้าเพื่อนำไปอ่านต่อ

### 12.1.2 เซ็นเซอร์ DHT22 (วัดอุณหภูมิและความชื้นสัมพัทธ์ในอากาศ)
เซ็นเซอร์ DHT22 เป็นที่นิยมในการตรวจวัดภูมิอากาศโรงเรือน ภายในประกอบด้วยสองกลไกย่อย:
1. **ตัวตรวจวัดอุณหภูมิ:** ใช้เทอร์มิสเตอร์ (Thermistor) ซึ่งเป็นสารกึ่งตัวนำประเภทเอ็นทีซี (Negative Temperature Coefficient - NTC) ที่มีพฤติกรรมทางฟิสิกส์เฉพาะตัวคือ เมื่ออุณหภูมิรอบข้างสูงขึ้น ค่าความต้านทานไฟฟ้าภายในตัวจะลดต่ำลงในลักษณะไม่เป็นเส้นตรงตามสมการสไตน์ฮาร์ต-ฮาร์ต (Steinhart-Hart equation)
2. **ตัวตรวจวัดความชื้นอากาศ:** ใช้แผ่นสารพอลิเมอร์ที่ไวต่อความชื้นกั้นระหว่างแผ่นขั้วนำไฟฟ้าเพื่อทำหน้าที่เป็นไดอิเล็กทริก เมื่อพอลิเมอร์ดูดซับไอน้ำจากอากาศจะเกิดการบวมตัวและเปลี่ยนค่าความเก็บประจุ ซึ่งชิปไอซีประมวลผลขนาดเล็กใน DHT22 จะทำหน้าที่คำนวณและสตรีมข้อมูลผลลัพธ์ออกมาในลักษณะบิตข้อมูลดิจิทัลที่เสถียรผ่านสายสัญญาณเพียงเส้นเดียว

### 12.1.3 อุปกรณ์วัดความเข้มแสง: LDR และ BH1750
* **แผงควบคุมต้านทานแสง (Light Dependent Resistor - LDR):** ทำงานโดยอาศัยหลักการความต้านทานที่แปรผกผันกับความเข้มแสง ทว่าการทำงานมีความคลาดเคลื่อนสูงตามอุณหภูมิและไม่มีมาตรฐานของหน่วยวัดที่แท้จริง
* **เซ็นเซอร์ BH1750:** อุปกรณ์วัดแสงระบบดิจิทัลที่ใช้โฟโตไดโอด (Photodiode) ทำหน้าที่เก็บเกี่ยวพลังงานจากแสงสว่างรอบข้างเพื่อปลดปล่อยอิเล็กตรอน (Photoelectric Effect) จากนั้นใช้แอมพลิฟายเออร์ขยายสัญญาณภายในชิปเพื่อแปลงค่าพารามิเตอร์ส่งมอบเป็นค่าความเข้มแสงในหน่วยลักซ์ (Lux) ผ่านทางโปรโตคอลการสื่อสารข้อมูลแบบสองสายสัญญาณได้โดยตรง

## 12.2 วิศวกรรมการสอบเทียบและการคำนวณหาค่าความชัน (Calibration Engineering)

สัญญาณอนาล็อก (Analog Signal) ที่อ่านจากขาเซ็นเซอร์ความชื้นดินมักจะถูกแปลงโดยไมโครคอนโทรลเลอร์ให้อยู่ในรูปตัวเลขจำนวนเต็มตามช่วงระดับความละเอียดของพอร์ต หรือเรียกว่าค่าดิบซึ่งไม่มีความหมายในเชิงเกษตรกรรม วิศวกรรมการสอบเทียบจึงเป็นสะพานสำคัญในการจัดทำระบบการถดถอยเชิงเส้น เพื่อแปลงค่าดิบเหล่านั้นให้กลายเป็นระดับเปอร์เซ็นต์ปริมาณความชื้นในดินที่มนุษย์และระบบช่วยตัดสินใจ (Decision Support System - DSS) เข้าใจได้

### 12.2.1 ทฤษฎีการสอบเทียบแบบสองจุด (Two-Point Calibration Theory)
การสอบเทียบที่มีความแม่นยำสูงระดับมาตรฐานตำราวิจัย ต้องทำภายใต้สภาพแวดล้อมควบคุมโดยกำหนดพิกัดความจริง 2 จุด ได้แก่:
1. **จุดแห้งสุด (Dry Reference):** นำใบเซ็นเซอร์ตากแดดหรือจุ่มในถังทรายแห้งสนิท บันทึกค่าดิบ $x_{\text{dry}}$ และกำหนดค่าผลลัพธ์ที่ควรจะเป็น $y_{\text{dry}} = 0\%$
2. **จุดอิ่มตัวเปียกสุด (Wet Reference):** นำใบเซ็นเซอร์จุ่มในน้ำสะอาดหรือดินโคลนอิ่มน้ำเต็มที่ บันทึกค่าดิบ $x_{\text{wet}}$ และกำหนดค่าผลลัพธ์ที่ควรจะเป็น $y_{\text{wet}} = 100\%$

```
  เปอร์เซ็นต์ความชื้น (y)
    │
100%│                     * (Wet Ref: x_wet, y_wet)
    │                    /
    │                   /
    │                  /  ความชัน (Slope: m) เป็นลบ
    │                 /
   0%│────────────────*────────────────────── ค่าดิบ ADC (x)
                      (Dry Ref: x_dry, y_dry)
```

การเชื่อมต่อจุดอิงทั้งสองเข้าด้วยกันด้วยกราฟเชิงเส้นอาศัยสมการเรเกรสชันเชิงเส้น:

$$y = m \cdot x + c$$

เมื่อต้องการหาค่าความชัน ($m$) และจุดตัดแกน Y ($c$) สามารถคำนวณสูตรคณิตศาสตร์ได้ดังนี้:

$$m = \frac{y_{\text{wet}} - y_{\text{dry}}}{x_{\text{wet}} - x_{\text{dry}}}$$

$$c = y_{\text{dry}} - m \cdot x_{\text{dry}}$$

#### ตัวอย่างการคำนวณจริงในแปลงทดสอบสวนทุเรียน:
จากการทดลองนำเซ็นเซอร์วัดดินคาปาซิทีฟไปสอบเทียบในโรงเรือนจันทบุรี:
* ค่าดิบในอากาศแห้ง ($x_{\text{dry}}$) = $3150$ ($y_{\text{dry}} = 0\%$)
* ค่าดิบในโคลนเปียก ($x_{\text{wet}}$) = $1250$ ($y_{\text{wet}} = 100\%$)

1. **คำนวณหาค่าความชัน $m$:**
   $$m = \frac{100 - 0}{1250 - 3150} = \frac{100}{-1900} \approx -0.05263158$$

2. **คำนวณหาจุดตัดแกน $c$:**
   $$c = 0 - (-0.05263158 \times 3150) \approx 165.78947$$

3. **สร้างฟังก์ชันแปลงความจริงสุดท้าย:**
   $$\text{Moisture \%} = -0.05263158 \times \text{Raw\_ADC} + 165.78947$$

4. **ทดสอบรันผล:** หากเซ็นเซอร์อ่านค่าดิบในสนามได้ $2200$ ปริมาณความชื้นเทียบเท่าร้อยละคือ:
   $$\text{Moisture \%} = (-0.05263158 \times 2200) + 165.78947 = -115.78947 + 165.78947 = 50.0\%$$
   *วิเคราะห์ผลลัพธ์:* ค่าดินมีความชื้นร้อยละ 50 พอดี ซึ่งเป็นค่าที่พร้อมสำหรับการนำไปป้อนเข้าระบบวิเคราะห์การรดน้ำทุเรียนอย่างแม่นยำ

## 12.3 การจัดการสัญญาณรบกวนทางไฟฟ้าด้วยอัลกอริทึม (Electrical Noise & Filtering)

ในสภาพหน้างานเกษตรกรรมจริง เช่น สวนทุเรียนขนาดใหญ่ที่มีปั๊มน้ำขนาดใหญ่ทำงานและมีการเดินสายไฟยาวหลายสิบเมตร ระบบเซ็นเซอร์จะเผชิญสัญญาณรบกวนทางแม่เหล็กไฟฟ้าอย่างหลีกเลี่ยงไม่ได้ สัญญาณรบกวนเหล่านี้จะไปแทรกแซงแรงดันไฟฟ้าอนาล็อกส่งผลให้ค่าที่อ่านได้แกว่งไปมาอย่างรุนแรงและเกิดการสั่งงานรดน้ำผิดพลาด

การรับมือกับปัญหานี้สามารถทำได้ทั้งในระดับโครงสร้างฮาร์ดแวร์และอัลกอริทึมซอฟต์แวร์:

### 12.3.1 การกรองทางฮาร์ดแวร์: ตัวกรองความถี่ต่ำผ่านแบบอาร์ซี (RC Low-Pass Filter)
การต่อวงจรโดยใช้ตัวต้านทาน ($R$) และตัวเก็บประจุ (Capacitor - $C$) ต่อขนานร่วมกันทำหน้าที่เป็นตัวกรองความถี่ต่ำผ่านเพื่อสกัดกั้นสัญญาณรบกวนความถี่สูง (เช่น คลื่นความถี่ไฟบ้าน 50 Hz หรือไฟกระชากจากไดนาโมปั๊มน้ำ) โดยมีความถี่คัตออฟ ($f_c$) ดังนี้:

$$f_c = \frac{1}{2\pi R C}$$

> [!TIP]
> หากเลือกใช้ตัวต้านทาน $R = 10 \ \text{k}\Omega$ และตัวเก็บประจุ $C = 1 \ \mu\text{F}$ จะได้ความถี่คัตออฟเท่ากับ:
> $$f_c = \frac{1}{2\pi \times 10000 \times 10^{-6}} = \frac{1}{0.06283} \approx 15.9 \text{ Hz}$$
> สัญญาณรบกวนความถี่ที่สูงกว่า 15.9 Hz (รวมถึงคลื่นรบกวนไฟบ้าน 50 Hz) จะถูกกรองทิ้งลงดินไป ช่วยให้ค่าอนาล็อกนิ่งและเสถียรอย่างมากก่อนส่งถึงไมโครคอนโทรลเลอร์

### 12.3.2 การกรองทางซอฟต์แวร์: อัลกอริทึมการกรองสัญญาณอนุกรม
ยามไม่สามารถเปลี่ยนโครงสร้างฮาร์ดแวร์ได้ นวัตกรสามารถเขียนโค้ดเพื่อบำบัดสัญญาณรบกวนได้ 2 แนวทางหลัก:

#### 1. ตัวกรองค่าเฉลี่ยเคลื่อนที่แบบง่าย (Simple Moving Average - SMA)
การเก็บข้อมูลสะสมไว้ในโครงสร้างแถวคิวขนาดคงที่ $N$ เมื่อมีข้อมูลใหม่เข้ามา จะทำการหาค่าเฉลี่ยทางสถิติของข้อมูลทั้งหมดในหน้าต่างเวลา:

$$y[n] = \frac{1}{N} \sum_{i=0}^{N-1} x[n-i]$$

วิธีนี้กรองสัญญาณรบกวนขรุขระได้ดีมาก แต่ต้องแลกมาด้วยการสิ้นเปลืองหน่วยความจำแรมที่ต้องคอยเก็บประวัติภาพรวมข้อมูลย้อนหลัง และอาจเกิดการดีเลย์ตอบสนองช้าหากหน้าต่าง $N$ ใหญ่เกินไป

#### 2. ตัวกรองค่าเฉลี่ยเคลื่อนที่แบบเอกซ์โพเนนเชียล (Exponential Moving Average - EMA)
ตัวกรองความเรียบทางคณิตศาสตร์ยอดนิยมที่ลดการสิ้นเปลืองแรมเหลือเพียงตัวแปรเก็บสถานะก่อนหน้าตัวเดียว โดยถ่วงน้ำหนักความสำคัญระหว่างข้อมูลสดรอบปัจจุบันและผลลัพธ์รอบก่อนหน้าตามค่าสัดส่วนถ่วงเวลา $\alpha$:

$$y[n] = \alpha \cdot x[n] + (1 - \alpha) \cdot y[n-1]$$

เมื่อตัวแปรต่าง ๆ มีคำอธิบายดังนี้:
* $x[n]$ คือค่าดิบที่เซ็นเซอร์อ่านได้ล่าสดในคาบเวลาปัจจุบัน
* $y[n-1]$ คือผลลัพธ์ที่ผ่านการกรองเรียบเรียบร้อยแล้วจากคาบรอบก่อนหน้า
* $y[n]$ คือผลลัพธ์การกรองตัวใหม่ที่พร้อมนำไปใช้งานจริง
* $\alpha$ (Alpha) คือค่าคงตัวควบคุมความเรียบสัมพัทธ์ อยู่ในช่วง $[0.0, 1.0]$
  * หากกำหนด $\alpha$ เข้าใกล้ $1.0$ เช่น $0.9$: ระบบจะตอบสนองไวต่อข้อมูลสดใหม่มาก แต่จะเก็บสัญญาณรบกวนพ่วงติดเข้ามามากตามกัน
  * หากกำหนด $\alpha$ เข้าใกล้ $0.0$ เช่น $0.1$: สัญญาณผลลัพธ์จะเรียบนิ่งสวยงามมาก แต่จะมีความล่าช้าในการตอบสนองสูงยามระดับน้ำในดินมีการเปลี่ยนไปจริง ๆ

## 12.4 การแปลงค่า ADC และความละเอียดของพอร์ตนำเข้า (Bit Resolution)

เพื่อให้อุปกรณ์ประมวลผลระบบดิจิทัลเข้าใจค่าแรงดันอนาล็อกทางไฟฟ้า $V_{\text{in}}$ ที่ส่งมาจากเซ็นเซอร์ บอร์ดจะต้องทำการส่งเข้าสู่วงจรตัวแปลงสัญญาณอนาล็อกเป็นดิจิทัล โดยสเปกการแปลงจะระบุศักยภาพผ่านจำนวนบิตความละเอียด (Resolution)

ไมโครคอนโทรลเลอร์รุ่นยอดนิยมมีการกำหนดมิติความละเอียดต่างกัน:
* **Arduino Uno:** ความละเอียด $10 \text{ bits}$ หรือสเกลจำนวนเต็มแบ่งย่อยได้ $2^{10} = 1024$ ระดับ ($0 - 1023$)
* **ESP32:** ความละเอียด $12 \text{ bits}$ หรือสเกลจำนวนเต็มแบ่งย่อยได้ $2^{12} = 4096$ ระดับ ($0 - 4095$)

คณิตศาสตร์ของขนาดขั้นตอนควอนไทซ์ (Quantization Step - $\Delta V$) หรือระดับแรงดันขั้นต่ำที่สุดที่เซ็นเซอร์สามารถแยกแยะได้ สามารถแสดงสูตรคำนวณได้ดังนี้:

$$\Delta V = \frac{V_{\text{ref}}}{2^B}$$

เมื่อ $V_{\text{ref}}$ คือช่วงแรงดันไฟสูงสุดอ้างอิงของบอร์ด (ปกติคือ $3.3 \text{ V}$) และ $B$ คือจำนวนบิตความละเอียด

#### การประเมินเปรียบเทียบเชิงวิศวกรรมบิต:
* สำหรับบอร์ด ESP32 ที่มีขนาดบิต $12 \text{ bits}$:
  $$\Delta V_{\text{ESP32}} = \frac{3.3 \text{ V}}{4096} \approx 0.000805 \text{ V} \approx 0.805 \text{ mV}$$
* สำหรับบอร์ด Arduino Uno ที่มีขนาดบิต $10 \text{ bits}$:
  $$\Delta V_{\text{Arduino}} = \frac{3.3 \text{ V}}{1024} \approx 0.003222 \text{ V} \approx 3.222 \text{ mV}$$

*วิเคราะห์ผลลัพธ์:* บอร์ดประมวลผล ESP32 สามารถจับความเคลื่อนไหวทางไฟฟ้าที่มีความละเอียดอ่อนได้ดีกว่าบอร์ดแบบเดิมถึง 4 เท่าตัว ซึ่งมีประโยชน์อย่างสูงในการวิเคราะห์ระดับเกลือแร่หรือความชื้นดินปนทรายจันทบุรีที่ค่าไฟฟ้าเคลื่อนไหวขยับต่ำมากในระดับมิลลิโวลต์

## 12.5 การบริหารจัดการพลังงานและระบบรักษาระดับแรงดัน (Power Management)

ในการทำระบบไอโอทีสนาม ปัญหาใหญ่คือระดับแรงดันไฟฟ้าของแบตเตอรี่จะค่อย ๆ ลดลงตามปริมาณการคายประจุ (เช่น แบตเตอรี่ลิเทียมมีแรงดันเต็ม $4.2\text{V}$ และร่วงเหลือ $3.2\text{V}$ ยามประจุใกล้หมด) หากเราจ่ายไฟแบตเตอรี่ตรงเข้าสู่ตัวรับสัญญาณเซ็นเซอร์ ค่าอนาล็อกที่อ่านได้จะแกว่งและเพี้ยนไปตามแรงดันแบตเตอรี่ที่ร่วงลง

วิศวกรรมเกษตรอัจฉริยะจึงกำหนดแนวทางการแก้ไขไว้สองระดับ:

### 12.5.1 การจัดระบบรักษาระดับแรงดันคงที่ (Voltage Regulation)
นวัตกรต้องเลือกระหว่างระบบคุมสองประเภท:
1. **ตัวรักษาระดับแรงดันต่ำมาก (Low Dropout Regulator - LDO):** ไอซีรักษาระดับแรงดันแบบเหนี่ยวนำ เช่น เบอร์ AP2112K ที่จ่ายแรงดันออก $3.3\text{V}$ คงที่ มีความต้านทานและสัญญาณรบกวนต่ำมาก เหมาะสำหรับระบบที่ไวต่อสัญญาณรบกวน แต่ข้อเสียคือประสิทธิภาพพลังงานจะเสียไปในรูปของความร้อนหากความต่างศักย์สูงเกินไป
2. **ระบบสวิตชิ่งแปลงกระแสลง (Buck Converter):** วงจรจ่ายไฟที่ประสิทธิภาพสูง (สูงถึง 90%) เหมาะสำหรับระบบที่แปลงไฟแรงดันสูงจากแผงโซลาร์เซลล์ (เช่น 12V/18V) ลงมาเป็น 5V/3.3V เพื่อลดการสูญเสียกำลังงาน

### 12.5.2 วิศวกรรมควบคุมแหล่งจ่ายผ่านทรานซิสเตอร์ (Sensor Power Gating)
เซ็นเซอร์ส่วนใหญ่ เช่น ใบวัดดินคาปาซิทีฟยังคงกินไฟตลอดเวลาแม้ไมโครคอนโทรลเลอร์จะอยู่ในโหมดหลับลึก เพื่อป้องกันแบตเตอรี่หมดไว นวัตกรต้องไม่ต่อไฟ $3.3\text{V}$ จ่ายตรงให้เซ็นเซอร์ แต่ให้ใช้ทรานซิสเตอร์แบบพอร์ตรวมสัญญาณ (MOSFET) ทำหน้าที่เป็นสวิตช์เปิด-ปิดการจ่ายไฟให้เซ็นเซอร์ผ่านขาควบคุมของบอร์ดเฉพาะเวลาที่จะตื่นขึ้นมาอ่านค่าเท่านั้น

## 12.6 การสังเคราะห์กรณีศึกษา: การสอบเทียบเซ็นเซอร์ในดินหลากหลายประเภทเชิงลึก

*การศึกษาเชิงลึก:* การจัดตั้งระบบสอบเทียบแบบหลายจุด (Multi-point Calibration) และติดตั้งตารางสืบค้นข้อมูล (Look-Up Table - LUT) สำหรับดินร่วนปนทรายที่เป็นดินยอดนิยมในสวนทุเรียนพันธุ์กระดุมทอง จังหวัดจันทบุรี

ในธรรมชาติ ดินร่วนปนทรายมีโครงสร้างการซึมผ่านของน้ำที่รวดเร็วแต่เก็บกักความเปียกชื้นได้ปานกลาง ค่าการตอบสนองทางไฟฟ้าจึงไม่มีลักษณะเป็นเส้นตรงเป๊ะ ๆ เหมือนในน้ำสะอาด ทีมวิจัยศูนย์ RBRU-Praneet จึงได้ดำเนินการสแกนเก็บจุดทดสอบในดินร่วนปนทรายรวม 5 เฟสความชื้นจริง เพื่อนำมาสร้างสมการการถดถอยพหุนาม (Polynomial Regression) ที่มีความถูกต้องสูงกว่าเชิงเส้นทั่วไป

```
  ดินแห้ง 0% ──> ดินชื้น 25% ──> ดินปานกลาง 50% ──> ดินฉ่ำน้ำ 75% ──> ดินอิ่มตัวน้ำ 100%
  (ADC: 3150)     (ADC: 2450)       (ADC: 1900)       (ADC: 1510)        (ADC: 1250)
```

จากการคำนวณเชิงลึก ทีมงานสามารถเขียนโค้ดระบบเพื่อจัดรูปแบบข้อมูลสอบเทียบเหล่านี้ในลักษณะตารางสืบค้นข้อมูล ร่วมกับกลไกการประมาณค่าในช่วงเชิงเส้น (Linear Interpolation) ภายในไมโครโปรเซสเซอร์ขอบ เพื่อป้องกันปัญหาคลาดเคลื่อนเชิงพื้นที่จากผลกระทบความเค็มของดินผสมปุ๋ยเคมีอินทรีย์ได้อย่างมีประสิทธิภาพสูงกว่าเดิม

## 12.7 โค้ดคอมพิวเตอร์และแบบฝึกหัดทบทวน 3 ระดับ

กระบวนการประมวลผลและการบำบัดกรองข้อมูลเซ็นเซอร์ ได้รับการสังเคราะห์รหัสโปรแกรมออกเป็น 3 ลำดับขั้นการศึกษาดังนี้:

### 🟩 ระดับที่ 1: มัธยมศึกษาตอนต้น (Junior High Level) - เครื่องคำนวณสอบเทียบเชิงเส้นสองจุด
*คำอธิบาย:* โค้ดนี้ใช้สำหรับการนำข้อมูลการป้อนค่าดิบแห้งสุดและเปียกสุดที่ทำการจดบันทึกไว้ในสมุดทดลองของนักเรียน มาคำนวณหาสูตรคณิตศาสตร์ระบบ Linear Regression และป้อนทายผลความชื้นดินจำลองเพื่อรายงานทางหน้าจอ

#### 💻 โค้ดภาษา Python สำหรับมัธยมต้น:
```python
# โปรแกรมคำนวณหาค่าสมการถดถอยและแปลงค่าระดับความชื้นดิน
# จัดทำโดย: นวัตกรน้อย RBRU-Praneet (ระดับมัธยมต้น)

# 1. ประกาศตัวแปรค่าการสอบเทียบที่จดได้จากการจุ่มใบเซ็นเซอร์
ADC_DRY_AIR = 3150.0   # ค่าดิบที่อ่านได้เมื่อเซ็นเซอร์อยู่กลางอากาศ (0% Moisture)
ADC_WET_WATER = 1250.0 # ค่าดิบที่อ่านได้เมื่อเซ็นเซอร์อยู่ในโคลนเปียกน้ำ (100% Moisture)

# 2. คำนวณหาความชัน (m) และจุดตัดแกน Y (c)
# m = (y2 - y1) / (x2 - x1)
m_slope = (100.0 - 0.0) / (ADC_WET_WATER - ADC_DRY_AIR)

# c = y1 - m * x1
c_intercept = 0.0 - (m_slope * ADC_DRY_AIR)

print("=== สรุปผลวิเคราะห์การคำนวณสมการขอบสนาม ===")
print(f"ความชันคำนวณได้ (Slope): {m_slope:.6f}")
print(f"จุดตัดแกน Y (Intercept): {c_intercept:.4f}")
print(f"สมการระบบคือ: Moisture% = ({m_slope:.6f} * ADC) + {c_intercept:.4f}")

# 3. ฟังก์ชันจำลองการนำสมการระบบไปใช้วิเคราะห์ค่าในสนามจริง
def convert_to_percentage(raw_adc):
    calculated_percent = (m_slope * raw_adc) + c_intercept
    
    # ดำเนินการตัดขอบเขตข้อมูลให้อยู่ในช่วง 0 - 100% เท่านั้นเพื่อป้องกันพฤติกรรมข้อมูลล้น
    if calculated_percent < 0.0:
        calculated_percent = 0.0
    elif calculated_percent > 100.0:
        calculated_percent = 100.0
        
    return calculated_percent

# 4. ทดสอบทดลองประมวลสัญญาณ
test_adc_values = [2800, 2200, 1500, 4000] # 4000 คือค่าผิดปกติเกินขนาดจริง

print("\n=== ผลลัพธ์จากการสแกนแปลงปลูกทุเรียน ===")
for adc in test_adc_values:
    moisture = convert_to_percentage(adc)
    print(f"ค่าดิบจากขาเซ็นเซอร์: {adc:4d} | คิดเป็นระดับความชื้นสัมพัทธ์ในดิน: {moisture:6.2f}%")
print("=========================================")
```

#### ❓ แบบฝึกหัดทบทวนระดับมัธยมต้น:
1. หากนักเรียนนำบอร์ดไปเปลี่ยนใช้ตัววัดใหม่แล้วจดสถิติได้ค่าแห้งสุด $3900$ และค่าเปียกสุด $1800$ จงคำนวณด้วยตนเองว่าค่าดิบ $2500$ จะแปลงเปอร์เซ็นต์ออกมาได้เท่าใด
2. เพราะเหตุใดในการแปลงคำนวณจึงต้องระบุคำสั่งการตัดขอบเขตข้อมูลให้อยู่ในช่วง $0\%$ ถึง $100\%$ อธิบายข้อเสียในระบบควบคุมน้ำหากไม่มีชุดคัดกรองส่วนนี้

### 🟨 ระดับที่ 2: มัธยมศึกษาตอนปลาย (Senior High Level) - โค้ดภาษา Python สำหรับจำลองซอฟต์แวร์กรองคลื่นไฟรบกวนด้วยฟิลเตอร์ EMA
*คำอธิบาย:* โปรแกรมนี้สังเคราะห์ขึ้นเพื่อแก้ไขปัญหาสัญญาณกระโดดสวิงไปมาจากไฟกระชากหรือคลื่นรบกวนในสวนผสม โดยใช้ระบบตัวกรองคณิตศาสตร์ Exponential Moving Average ร่วมกับระบบสอบเทียบและส่งสถิติตัวแปรวิเคราะห์ระดับวินาที

#### 💻 โค้ดภาษา Python สำหรับมัธยมปลาย:
```python
# โปรแกรมจำลองการคัดกรองสัญญาณรบกวนและวิเคราะห์แปลงค่าความชื้นดิน
# จัดทำโดย: นวัตกรระดับมัธยมปลาย RBRU-Praneet

import random
import time

# 1. กำหนดพารามิเตอร์การกรองและสอบเทียบ
alpha_factor = 0.15      # ค่าสัมประสิทธิ์ตัวกรอง EMA (เน้นความเรียบนิ่งแต่มั่นคง)
adc_filtered_prev = 3150  # กำหนดค่าเริ่มต้นตัวกรองเท่ากับค่าในอากาศแห้ง

# ค่าสัมประสิทธิ์สอบเทียบจากการคำนวณระดับ 1
m_slope = -0.05263158
c_intercept = 165.78947

# 2. ฟังก์ชันจำลองเซ็นเซอร์สนามที่ปนเปื้อนสัญญาณรบกวนความถี่สูง (Noise)
def read_noisy_sensor():
    base_adc = 2000.0  # ค่าคงที่จริงของความชื้นดินเฉลี่ยขณะนั้น
    noise = random.uniform(-150.0, 150.0)  # สัญญาณสวิงกระโดดจากไฟรบกวนมอเตอร์ปั๊มน้ำ
    return base_adc + noise

print("--- เริ่มต้นการกรองคลื่นสัญญาณและถอดถอนคลื่นรบกวน ---")
print(f"กำหนดน้ำหนักความเรียบฟิลเตอร์ (Alpha): {alpha_factor}")
print("คาบเวลา | ค่าดิบไฟเหวี่ยง | ค่ากรองเรียบ EMA | ระดับเปอร์เซ็นต์จริง")
print("-" * 65)

# ลูปทำงานประมวลผล 10 คาบเวลาเสมือนจริง
for step in range(1, 11):
    raw_adc = read_noisy_sensor()
    
    # 3. การกรองซอฟต์แวร์ด้วยสูตร EMA: y[n] = alpha * x[n] + (1 - alpha) * y[n-1]
    adc_filtered_current = (alpha_factor * raw_adc) + ((1 - alpha_factor) * adc_filtered_prev)
    
    # 4. แปลงค่าที่กรองแล้วเป็นเปอร์เซ็นต์ความชื้น
    moisture_pct = (m_slope * adc_filtered_current) + c_intercept
    
    # คุมขอบเขตปลอดภัยข้อมูล
    moisture_pct = max(0.0, min(100.0, moisture_pct))
    
    print(f"วินาทีที่ {step:2d} |    {raw_adc:7.2f}   |   {adc_filtered_current:7.2f}    |    {moisture_pct:5.2f}%")
    
    # เซฟค่าสถานะเป็นตัวแปรอดีตเพื่อใช้ในรอบประมวลวินาทีถัดไป
    adc_filtered_prev = adc_filtered_current
    time.sleep(0.5)

print("-" * 65)
print("--- สิ้นสุดการกรองบำบัดสัญญาณเรียบร้อย ---")
```

#### ❓ แบบฝึกหัดทบทวนระดับมัธยมปลาย:
1. หากนักเรียนต้องการปรับเปลี่ยนให้ระบบตอบสนองตอบรับความเคลื่อนไหวระดับน้ำดินที่รดฉีดพ่นลงโคนต้นทุเรียนได้ฉับไวสะดวดมากยิ่งขึ้น นักเรียนต้องปรับแต่งตัวคงค่า `alpha_factor` ให้มีตัวเลขเพิ่มขยับขยายตัวเลข หรือปรับลดลงอย่างไร อธิบายทฤษฎีผลกระทบในมิติด้านความเรียบของข้อมูล
2. อธิบายจุดแตกต่างทางกลศาสตร์คณิตศาสตร์ในการเขียนโปรแกรมจัดเก็บแบบ EMA เทียบกับการหาค่าเฉลี่ยดิบโดยทั่วไป (Mean) ในด้านการใช้ทรัพยากรตัวแปรและพื้นที่เก็บข้อมูลแรม

*คำอธิบาย:* โค้ดประยุกต์ใช้ซอฟต์แวร์ระดับปริญญาตรีนี้ พัฒนาภายใต้สถาปัตยกรรมระดับอุตสาหกรรมแบบการเขียนโปรแกรมเชิงวัตถุ (Object-Oriented Programming - OOP) โดยจัดทำโครงข่ายคลาส `CapacitiveSoilSensor` ที่รวมเอาการกรองสัญญาณแบบไฮบริด (Hybrid Filter) ที่ถอดค่าพิจารณาแบบเคลื่อนไหว และใช้ระบบการสอบเทียบระดับพหุนามเส้นโค้ง (Polynomial Regression) ร่วมกับตารางสืบค้นข้อมูลและการประมาณค่าช่วง เพื่อความถูกต้องสูงสุดในการจัดประเภทสภาวะดินในสวนทุเรียนหมอนทองพร้อมออกรายงานเป็นไฟล์ประวัติ

#### 💻 โค้ดภาษา Python สำหรับระดับอุดมศึกษา:
```python
"""
ระบบประมวลผลสัญญาณและคัดกรองข้อมูลระดับอุตสาหกรรม (Advanced Sensor Processing Suite)
ศูนย์พัฒนาเทคโนโลยีระบบควบคุม รำไพพรรณี ประณีตวิทยาคม RBRU-Praneet 2026
"""

import math
import logging
import json

# ตั้งค่าสถานะการควบคุมแจ้งเตือนความปลอดภัยซอฟต์แวร์
logging.basicConfig(level=logging.INFO, format="%(asctime)s [%(levelname)s] %(message)s")
logger = logging.getLogger("SensorProcessingEngine")

class AdvancedAgriSensor:
    def __init__(self, sensor_id="CAP-SOIL-DURIAN-09", alpha=0.18):
        self.sensor_id = sensor_id
        self.alpha = alpha
        self.filtered_value = None
        
        # ตารางสืบค้นข้อมูลสอบเทียบในดินร่วนปนทราย (Look-Up Table - LUT)
        # รูปแบบคู่ค่า: { Raw_ADC_Value : Target_Moisture_Percentage }
        # ค่าเป็นเส้นโค้งตามโครงสร้างทางกายภาพความเหนียวของดินผสมปุ๋ยเคมีอินทรีย์
        self.lut_calibration = {
            3200: 0.0,    # จุดดินแห้งสนิท ไร้น้ำหล่อเลี้ยง
            2500: 20.0,   # จุดทรายเริ่มมีความอบอ้าว
            1900: 50.0,   # จุดชื้นสมดุล เหมาะสมสำหรับทุเรียน
            1450: 80.0,   # จุดดินชุ่มฉ่ำน้ำ
            1100: 100.0   # จุดดินแฉะน้ำขังเต็มที่
        }
        # จัดเรียงค่าคีย์เพื่อความสะดวกในการสแกนข้อมูลเชิงตัวเลข
        self.sorted_adc_keys = sorted(self.lut_calibration.keys())

    def apply_ema_filter(self, raw_value):
        """
        ประยุกต์ใช้ตัวกรอง EMA เพื่อลดทอนสัญญาณรบกวนความถี่สูงแบบมีเสถียรภาพหน่วยความจำ
        """
        if self.filtered_value is None:
            self.filtered_value = float(raw_value)
        else:
            self.filtered_value = (self.alpha * float(raw_value)) + ((1.0 - self.alpha) * self.filtered_value)
        return self.filtered_value

    def interpolate_lut(self, adc_val):
        """
        แปลงค่าทางไฟฟ้าอนาล็อกผ่านการคำนวณ Linear Interpolation บนตาราง Look-Up Table
        """
        # ปรับขอบข่ายขั้นต่ำสุดและสูงสุดของตัวเลข
        if adc_val >= self.sorted_adc_keys[-1]:
            # ค่าดิบสูงเกินกว่าดินแห้งสุด (ADC แฉกแห้ง)
            return self.lut_calibration[self.sorted_adc_keys[-1]]
        if adc_val <= self.sorted_adc_keys[0]:
            # ค่าดิบต่ำลงเปียกน้ำเต็มพิกัด
            return self.lut_calibration[self.sorted_adc_keys[0]]

        # ค้นหาช่วงพิกัดระนาบในตารางสืบค้นที่ค่าดิบนั้นอยู่ระหว่างกัน
        for i in range(len(self.sorted_adc_keys) - 1):
            x0 = self.sorted_adc_keys[i]
            x1 = self.sorted_adc_keys[i+1]
            
            # การสอบเทียบความชื้นแปรผกผันกับแรงดันไฟฟ้า (คีย์สูงคือแห้ง คีย์ต่ำคือเปียก)
            if x0 <= adc_val <= x1:
                y0 = self.lut_calibration[x0]
                y1 = self.lut_calibration[x1]
                
                # สมการเชิงเส้นระหว่างสองจุด: y = y0 + ((y1 - y0) / (x1 - x0)) * (x - x0)
                moisture_pct = y0 + ((y1 - y0) / (x1 - x0)) * (adc_val - x0)
                return moisture_pct
                
        return 0.0

    def process_raw_reading(self, raw_reading):
        """
        ท่อวิเคราะห์ข้อมูลรวมของเซ็นเซอร์ (Hybrid pipeline): กรองคลื่น -> แปลงสถิติ LUT -> ตรวจสอบขอบความปลอดภัย
        """
        try:
            # ตรวจทานความถูกต้องของรูปข้อมูล
            if not isinstance(raw_reading, (int, float)):
                raise ValueError("พารามิเตอร์ข้อมูลนำเข้าต้องอยู่ในรูปแบบตัวเลขเท่านั้น")
                
            # 1. กรองสัญญาณรบกวนความดันไฟ
            filtered_adc = self.apply_ema_filter(raw_reading)
            
            # 2. สอบเทียบค่าด้วยตารางความโค้งสถิตดิน
            calibrated_percent = self.interpolate_lut(filtered_adc)
            
            return {
                "sensor_id": self.sensor_id,
                "raw_input": raw_reading,
                "filtered_adc": round(filtered_adc, 2),
                "soil_moisture_pct": round(calibrated_percent, 2),
                "irrigation_action": "TRIGGER_WATER" if calibrated_percent < 45.0 else "HOLD_VALVE"
            }
        except Exception as e:
            logger.critical(f"ระบบตรวจจับความเสียหายประมวลผลเซ็นเซอร์พัง: {e}")
            return None

if __name__ == "__main__":
    # สร้างโหนดทดสอบ
    soil_sensor = AdvancedAgriSensor(sensor_id="CAP-SOIL-DURIAN-09")
    
    # ชุดข้อมูลจำลองภาพการอ่านเซ็นเซอร์ในสนามที่ส่งผ่านคลื่นรบกวนเป็นจังหวะ
    simulated_reads = [3000, 3100, 2600, 2400, 2000, 1800, 1300, 1200]
    
    logger.info("เริ่มต้นสั่งเปิดและประมวลผลท่อข้อมูลอัจฉริยะ (RBRU-Praneet 2026)")
    
    for i, raw_val in enumerate(simulated_reads):
        result = soil_sensor.process_raw_reading(raw_val)
        if result:
            print(f"ขั้นตอน {i+1:d} | ค่าดิบ: {result['raw_input']:4d} | กรองแล้ว: {result['filtered_adc']:7.2f} | ผลลัพธ์ความชื้น: {result['soil_moisture_pct']:6.2f}% | การตัดสินใจ: {result['irrigation_action']}")
```

#### ❓ แบบฝึกหัดทบทวนระดับอุดมศึกษา:
1. (วิเคราะห์) จงเปรียบเทียบข้อดีและข้อเสียในมิติด้านความคุ้มค่าและความแม่นยำของการเลือกใช้ตารางสืบค้นข้อมูลการถดถอยเชิงเส้นร่วมกับเทคนิคการสลับพิกัด (Linear Interpolated LUT) เทียบกับการแปลงด้วยสมการพหุนามระดับกำลัง 3 (3rd-Order Polynomial Function) ในการประมวลผลบนไมโครคอนโทรลเลอร์ขนาดจิ๋วที่มีแรมเพียง 4KB
2. (สังเคราะห์) นักวิจัยต้องการขยายสถาปัตยกรรมคลาส `AdvancedAgriSensor` เพื่อรองรับการสแกนเก็บข้อมูลจากชุดเซ็นเซอร์ดิน 3 แบรนด์ที่มีค่าการสอบเทียบ $x_{\text{dry}}$ และ $x_{\text{wet}}$ แตกต่างกันอย่างสิ้นเชิง จงออกแบบวิธีการเขียนโปรแกรมเชิงวัตถุเพื่อทำการโหลดโครงสร้างการตั้งค่าสอบเทียบเหล่านั้นเข้าสู่ระบบผ่านรูปแบบไฟล์การตั้งค่า JSON สรุปการออกแบบโครงสร้างคีย์ไฟล์ JSON ประกอบคำอธิบาย

3. **(ระดับออกแบบ - Design)**
