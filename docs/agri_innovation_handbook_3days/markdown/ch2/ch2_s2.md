## 2.2 โครงสร้างคณิตศาสตร์ของ CNN (Convolutional Neural Networks: Convolution, ReLU, Max Pooling, Softmax)

# 2.2 โครงสร้างคณิตศาสตร์ของโครงข่ายประสาทเทียมแบบคอนโวลูชัน (Mathematical Structure of Convolutional Neural Networks: CNN)

ในฐานะผู้เชี่ยวชาญด้านฟิสิกส์ประยุกต์และเกษตรดิจิทัล ข้าพเจ้าขออธิบายถึงโครงสร้างคณิตศาสตร์ที่อยู่เบื้องหลังโครงข่ายประสาทเทียมแบบคอนโวลูชัน (Convolutional Neural Networks: CNN) ซึ่งถือเป็นสถาปัตยกรรมหลักที่ปฏิวัติวงการการเรียนรู้เชิงลึก (Deep Learning) โดยเฉพาะอย่างยิ่งในการประมวลผลข้อมูลภาพ (Image Processing) และการวิเคราะห์ภาพทางการแพทย์หรือภาพพืชผลทางการเกษตร (Plant Phenotyping)

CNN ถูกออกแบบมาเพื่อเลียนแบบการทำงานของระบบการมองเห็นของสิ่งมีชีวิต โดยการสกัดคุณลักษณะ (Feature Extraction) ที่มีความสำคัญจากข้อมูลภาพอย่างเป็นลำดับชั้น (Hierarchical) ตั้งแต่คุณลักษณะพื้นฐาน เช่น ขอบ (Edges) และมุม (Corners) ไปจนถึงคุณลักษณะที่ซับซ้อน เช่น ลักษณะของโรคหรือสายพันธุ์พืช

---

## 2.2.1 ภาพรวมสถาปัตยกรรม CNN

โครงข่าย CNN โดยทั่วไปประกอบด้วยการเรียงตัวของชั้น (Layers) ต่างๆ ที่ทำงานร่วมกันอย่างเป็นระบบ โดยมีองค์ประกอบหลักที่สำคัญ ได้แก่:

1.  **ชั้นคอนโวลูชัน (Convolutional Layer):** ทำหน้าที่สกัดคุณลักษณะเฉพาะจากอินพุตภาพ
2.  **ฟังก์ชันกระตุ้นค่า (Activation Function):** เช่น ReLU เพื่อเพิ่มความเป็นไม่เชิงเส้น (Non-linearity) ให้กับแบบจำลอง
3.  **ชั้นลดขนาดพื้นที่สูงสุด (Pooling Layer):** ทำหน้าที่ลดมิติข้อมูล (Dimensionality Reduction) เพื่อลดความซับซ้อนในการคำนวณและป้องกันปัญหา Overfitting
4.  **ชั้นเชื่อมต่อสมบูรณ์ (Fully Connected Layer):** ทำหน้าที่รับคุณลักษณะที่สกัดมาทั้งหมดเพื่อทำการจำแนกประเภท (Classification)

เมื่อเรานำ CNN มาประยุกต์ใช้ในการจำแนกภาพใบพืช (Leaf Image Classification) ข้อมูลภาพใบพืชจะถูกป้อนเข้าสู่ชั้นคอนโวลูชันเป็นอินพุต (Input) จากนั้นผ่านการสกัดคุณลักษณะหลายชั้น จนกระทั่งคุณลักษณะที่ถูกสกัดออกมาถูกป้อนเข้าสู่ชั้นเชื่อมต่อสมบูรณ์เพื่อคำนวณความน่าจะเป็น (Probability) ของการเป็นโรคหรือสายพันธุ์ต่างๆ

## 2.2.2 ชั้นคอนโวลูชัน (Convolutional Layer)

ชั้นคอนโวลูชันเป็นหัวใจสำคัญของ CNN โดยอาศัยกลไกทางคณิตศาสตร์ที่เรียกว่า **การสังวัตนาการ (Convolution)** ซึ่งเป็นการดำเนินการทางคณิตศาสตร์ที่ใช้ตัวกรอง (Filter) หรือเคอร์เนล (Kernel) เล็กๆ เคลื่อนที่ไปทั่วทั้งภาพอินพุต เพื่อคำนวณค่าคุณลักษณะ (Feature Map) ออกมา

### 2.2.2.1 หลักการทางคณิตศาสตร์ของการสังวัตนาการ

สมมติให้ $I$ คือภาพอินพุต (Input Image) ที่มีขนาด $H \times W$ (ความสูง $\times$ ความกว้าง) และ $K$ คือตัวกรอง (Kernel) หรือฟิลเตอร์ (Filter) ที่มีขนาด $F \times F$ (โดยที่ $F$ คือขนาดของฟิลเตอร์)

การสังวัตนาการคือกระบวนการที่ค่าพิกเซลของ $K$ ถูกนำไปคูณกับค่าพิกเซลของ $I$ ในบริเวณที่ทับซ้อนกัน (Overlap Region) จากนั้นนำผลคูณทั้งหมดมาบวกกันเพื่อได้ค่าเดียวในแผนที่คุณลักษณะ (Feature Map) ณ ตำแหน่งนั้นๆ

**สมการการสังวัตนาการ (Convolution Equation):**

หากเรากำหนดให้ $G$ คือแผนที่คุณลักษณะ (Feature Map) ที่ได้จากการสังวัตนาการระหว่างภาพอินพุต $I$ และตัวกรอง $K$ ณ ตำแหน่ง $(i, j)$ จะสามารถเขียนได้ดังนี้:

$$G[i, j] = (I * K)[i, j] = \sum_{m=0}^{F-1} \sum_{n=0}^{F-1} I[i-m, j-n] K[m, n]$$

**คำอธิบายองค์ประกอบทางคณิตศาสตร์:**

*   $G[i, j]$: คือค่าพิกเซลที่ตำแหน่ง $(i, j)$ ของแผนที่คุณลักษณะที่ได้
*   $I[i-m, j-n]$: คือค่าพิกเซลของภาพอินพุต $I$ ที่ตำแหน่ง $(i-m, j-n)$
*   $K[m, n]$: คือค่าพิกเซลของตัวกรอง $K$ ที่ตำแหน่ง $(m, n)$
*   $\sum_{m=0}^{F-1} \sum_{n=0}^{F-1}$: คือการดำเนินการผลรวม (Summation) ที่ครอบคลุมทุกองค์ประกอบของตัวกรอง $K$

**การดำเนินการในทางปฏิบัติ:**

ในการคำนวณค่า $G[i, j]$ ตัวกรอง $K$ จะถูกวางทับบนบริเวณ $F \times F$ ของภาพ $I$ ที่ตำแหน่ง $(i, j)$ จากนั้นจะมีการคูณค่าพิกเซลที่สอดคล้องกันทั้งหมด (Element-wise Multiplication) และนำผลลัพธ์ที่ได้ทั้งหมดมาบวกกัน (Summation)

**พารามิเตอร์ที่เกี่ยวข้อง:**

1.  **ขนาดของตัวกรอง (Filter Size, $F$):** กำหนดขนาดของ $K$ เช่น $3 \times 3$ หรือ $5 \times 5$
2.  **จำนวนตัวกรอง (Number of Filters, $N$):** กำหนดจำนวนคุณลักษณะที่ต้องการสกัด (เท่ากับจำนวนช่องสัญญาณในชั้นถัดไป)
3.  **ขนาดสเต็ป (Stride, $S$):** คือจำนวนพิกเซลที่ตัวกรองขยับไปในแต่ละครั้ง (เช่น $S=1$ คือขยับทีละ 1 พิกเซล, $S=2$ คือขยับทีละ 2 พิกเซล)
4.  **การเติมขอบ (Padding, $P$):** คือการเพิ่มค่าศูนย์ (Zero Padding) รอบขอบของภาพอินพุต เพื่อให้ขนาดของแผนที่คุณลักษณะไม่ลดลงมากเกินไป

## 2.2.3 ฟังก์ชันกระตุ้นค่าเชิงเส้นแบบปรับแก้ (Rectified Linear Unit: ReLU)

หลังจากที่ชั้นคอนโวลูชันได้ทำการสกัดคุณลักษณะออกมาแล้ว ค่าที่ได้มักจะเป็นค่าที่ต่อเนื่อง (Continuous) และมีทั้งค่าบวกและค่าลบ การนำค่าเหล่านี้ไปใช้โดยตรงในชั้นถัดไปอาจทำให้แบบจำลองมีปัญหาเรื่องการหาค่าความชัน (Gradient) ในระหว่างการฝึกฝน (Training)

ดังนั้น จึงจำเป็นต้องใช้ **ฟังก์ชันกระตุ้นค่า (Activation Function)** เพื่อเพิ่มความเป็นไม่เชิงเส้น (Non-linearity) ให้กับแบบจำลอง ซึ่งทำให้ CNN สามารถเรียนรู้ความสัมพันธ์ที่ซับซ้อนในข้อมูลได้

**ReLU Function:**

ReLU เป็นฟังก์ชันกระตุ้นค่าที่ได้รับความนิยมสูงสุดเนื่องจากความเรียบง่ายและประสิทธิภาพในการคำนวณ

$$f(x) = \max(0, x)$$

**คำอธิบาย:**

ฟังก์ชันนี้จะทำการตรวจสอบค่าอินพุต $x$ หาก $x$ มีค่าเป็นบวก (Positive) ค่าเอาต์พุตจะเท่ากับ $x$ แต่ถ้า $x$ มีค่าเป็นลบ (Negative) ค่าเอาต์พุตจะถูกกำหนดให้เป็นศูนย์ (Zero)

**ความสำคัญทางฟิสิกส์และคณิตศาสตร์:**

การใช้ ReLU ช่วยแก้ปัญหาที่เรียกว่า "Vanishing Gradient Problem" ในโครงข่ายประสาทเทียมแบบดั้งเดิม (เช่น Sigmoid หรือ Tanh) เนื่องจากค่าอนุพันธ์ (Derivative) ของ ReLU สำหรับ $x > 0$ คือ 1 ซึ่งช่วยให้การส่งผ่านค่าความชัน (Gradient) ในระหว่างการย้อนกลับ (Backpropagation) มีประสิทธิภาพและไม่ลดลงจนเข้าใกล้ศูนย์เร็วเกินไป

## 2.2.4 ชั้นลดขนาดพื้นที่สูงสุด (Max Pooling Layer)

หลังจากผ่านชั้นคอนโวลูชันหลายครั้ง แผนที่คุณลักษณะที่ได้จะมีขนาดใหญ่ขึ้นและมีจำนวนพิกเซลมากเกินความจำเป็น (High Dimensionality) ซึ่งจะทำให้แบบจำลองมีความซับซ้อนในการคำนวณสูงและเสี่ยงต่อการเกิด Overfitting

**Max Pooling** คือชั้นที่ทำหน้าที่ลดขนาดพื้นที่ (Downsampling) ของแผนที่คุณลักษณะ โดยการเลือกค่าสูงสุด (Maximum Value) จากบริเวณย่อยๆ ของแผนที่คุณลักษณะ

**หลักการทำงาน:**

1.  กำหนดขนาดของหน้าต่าง (Pool Size) เช่น $2 \times 2$
2.  กำหนดสเต็ป (Stride) เช่น $2$
3.  ตัวกรองจะเลื่อนไปทั่วแผนที่คุณลักษณะ และ ณ แต่ละตำแหน่ง จะทำการเลือกค่าสูงสุด (Maximum) จากบริเวณที่ทับซ้อนกันนั้น

**สมการเชิงแนวคิด:**

หาก $F_{in}$ คือแผนที่คุณลักษณะอินพุต และ $P$ คือขนาดของ Pool Size และ $S$ คือ Stride แผนที่คุณลักษณะเอาต์พุต $F_{out}$ ณ ตำแหน่ง $(i, j)$ คือ:

$$F_{out}[i, j] = \max_{(m, n) \in \text{Window}} \{ F_{in}[i \cdot S + m, j \cdot S + n] \}$$

**ประโยชน์:**

1.  **ลดมิติข้อมูล:** ลดจำนวนพารามิเตอร์และค่าที่ต้องคำนวณในชั้นถัดไปอย่างมาก
2.  **เพิ่มความทนทาน (Robustness):** ทำให้แบบจำลองมีความทนทานต่อการเปลี่ยนแปลงเล็กน้อยของตำแหน่งของคุณลักษณะ (Translation Invariance) เช่น หากใบพืชเอียงไปเล็กน้อย การจำแนกประเภทก็ยังคงแม่นยำ

## 2.2.5 ชั้นเชื่อมต่อสมบูรณ์และฟังก์ชันซอฟต์แมกซ์ (Fully Connected Layer and Softmax Function)

หลังจากที่ผ่านการสกัดคุณลักษณะด้วยชั้นคอนโวลูชันและลดขนาดด้วย Max Pooling มาหลายชั้นแล้ว คุณลักษณะที่ได้จะถูกจัดเรียงและแผ่ขยาย (Flatten) ให้กลายเป็นเวกเตอร์ (Vector) ขนาดใหญ่ ซึ่งจะถูกป้อนเข้าสู่ **ชั้นเชื่อมต่อสมบูรณ์ (Fully Connected Layer: FC)**

ชั้น FC คือชั้นที่ทุกพิกเซล (หรือคุณลักษณะ) ที่ถูกสกัดมาในชั้นก่อนหน้า จะเชื่อมต่อกับทุกโหนด (Node) ในชั้นนี้ ซึ่งทำหน้าที่เป็นตัวจำแนกประเภทขั้นสุดท้าย

### 2.2.5.1 การคำนวณในชั้นเชื่อมต่อสมบูรณ์

ในชั้น FC การคำนวณจะใช้สมการพื้นฐานของโครงข่ายประสาทเทียม:

$$\mathbf{z} = \mathbf{W} \mathbf{x} + \mathbf{b}$$

*   $\mathbf{x}$: คือเวกเตอร์คุณลักษณะที่ถูกแผ่ขยายมาจากชั้นก่อนหน้า (Input Feature Vector)
*   $\mathbf{W}$: คือเมทริกซ์น้ำหนัก (Weight Matrix) ที่เรียนรู้ได้
*   $\mathbf{b}$: คือเวกเตอร์ไบแอส (Bias Vector)
*   $\mathbf{z}$: คือเวกเตอร์ผลลัพธ์ก่อนการกระตุ้นค่า (Logits)

### 2.2.5.2 ฟังก์ชันซอฟต์แมกซ์ (Softmax Function)

เนื่องจากเป้าหมายสุดท้ายคือการจำแนกประเภท (Classification) เราต้องการให้ผลลัพธ์ที่ได้เป็นชุดของความน่าจะเป็น (Probability Distribution) ซึ่งผลรวมของความน่าจะเป็นทั้งหมดต้องเท่ากับ 1

**Softmax Function** คือฟังก์ชันที่แปลงค่า Logits ($\mathbf{z}$) ให้เป็นความน่าจะเป็นที่ตีความได้

สมมติให้ $\mathbf{z}$ เป็นเวกเตอร์ของค่า Logits สำหรับ $C$ คลาส (Classes)

$$P(y = c \mid \mathbf{x}) = \frac{e^{z_c}}{\sum_{j=1}^{C} e^{z_j}}$$

**คำอธิบาย:**

*   $P(y = c \mid \mathbf{x})$: คือความน่าจะเป็นที่อินพุต $\mathbf{x}$ เป็นของคลาส $c$
*   $e^{z_c}$: คือค่า Exponential ของ Logit สำหรับคลาส $c$
*   $\sum_{j=1}^{C} e^{z_j}$: คือผลรวมของค่า Exponential ของ Logits สำหรับทุกคลาสทั้งหมด

ฟังก์ชันนี้จะรับประกันว่าค่าเอาต์พุตทั้งหมดจะมีค่าอยู่ระหว่าง 0 ถึง 1 และผลรวมของค่าเอาต์พุตทั้งหมดจะเท่ากับ 1 เสมอ

## 2.2.6 การประยุกต์ใช้ในโค้ด (Python Implementation Example)

เพื่อความเข้าใจที่สมบูรณ์ ข้าพเจ้าขอแสดงตัวอย่างการจำลองการทำงานของชั้นคอนโวลูชัน (Convolutional Layer) และการใช้ ReLU ในภาษา Python โดยใช้ไลบรารี NumPy ซึ่งเป็นพื้นฐานของการคำนวณเชิงตัวเลขในงานวิศวกรรมและฟิสิกส์

```python
import numpy as np

# ----------------------------------------------------------------------
# 1. การจำลองชั้นคอนโวลูชัน (Convolutional Layer Simulation)
# ----------------------------------------------------------------------

def convolution_2d(input_image, kernel, stride=1, padding=0):
    """
    ฟังก์ชันจำลองการดำเนินการสังวัตนาการ (Convolution) 2 มิติ
    
    Args:
        input_image (np.array): ภาพอินพุต (H_in, W_in)
        kernel (np.array): ตัวกรอง (F_h, F_w)
        stride (int): ขนาดสเต็ป (S)
        padding (int): ค่า Padding (P)
        
    Returns:
        np.array: แผนที่คุณลักษณะ (Feature Map)
    """
    
    # คำนวณขนาดของอินพุตที่ปรับด้วย Padding
    H_in = input_image.shape[0]
    W_in = input_image.shape[1]
    
    # การเพิ่ม Padding ให้กับภาพอินพุต
    if padding > 0:
        padded_image = np.pad(input_image, 
                               ((padding, padding), (padding, padding)), 
                               mode='constant', constant_values=0)
    else:
        padded_image = input_image
        
    H_padded, W_padded = padded_image.shape
    F_h, F_w = kernel.shape
    
    # คำนวณขนาดของเอาต์พุต (Output Dimension Calculation)
    H_out = int((H_padded - F_h) / stride + 1)
    W_out = int((W_padded - F_w) / stride + 1)
    
    feature_map = np.zeros((H_out, W_out))
    
    print(f"\n--- [Convolution Step] ---")
    print(f"Input Size: {H_in}x{W_in}, Kernel Size: {F_h}x{F_w}, Stride: {stride}, Padding: {padding}")
    print(f"Calculated Output Size: {H_out}x{W_out}")
    
    # วนลูปเพื่อทำการสังวัตนาการ (Convolution Loop)
    for i in range(H_out):
        for j in range(W_out):
            # กำหนดขอบเขตของบริเวณที่ตัวกรองจะทับ
            h_start = i * stride
            w_start = j * stride
            
            # ดึงบริเวณย่อย (Receptive Field) จากภาพที่ถูก Padding แล้ว
            receptive_field = padded_image[
                h_start : h_start + F_h, 
                w_start : w_start + F_w
            ]
            
            # คำนวณผลคูณแบบองค์ประกอบต่อองค์ประกอบ (Element-wise Multiplication)
            # และรวมผลลัพธ์ทั้งหมด (Summation)
            convolution_sum = np.sum(receptive_field * kernel)
            
            # กำหนดค่าในแผนที่คุณลักษณะ
            feature_map[i, j] = convolution_sum
            
    return feature_map

# ----------------------------------------------------------------------
# 2. การจำลอง ReLU Activation Function
# ----------------------------------------------------------------------

def relu_activation(input_data):
    """
    ฟังก์ชัน ReLU: f(x) = max(0, x)
    """
    print(f"\n--- [ReLU Activation Step] ---")
    # NumPy สามารถใช้ฟังก์ชัน max(0, x) ได้โดยตรง
    output_data = np.maximum(0, input_data)
    return output_data

# ----------------------------------------------------------------------
# 3. การจำลอง Softmax Function (สำหรับการจำแนกประเภท)
# ----------------------------------------------------------------
def softmax(z):
    """
    ฟังก์ชัน Softmax เพื่อคำนวณความน่าจะเป็นของคลาส
    """
    # หักลบค่าสูงสุดออกเพื่อความเสถียรเชิงตัวเลข (Prevent Overflow)
    exp_z = np.exp(z - np.max(z))
    return exp_z / np.sum(exp_z)
```
