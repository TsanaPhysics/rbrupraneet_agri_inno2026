ก่อนที่เราจะเริ่มเขียนโค้ดใดๆ สิ่งแรกที่นักนวัตกรต้องทำความเข้าใจคือ "สภาพแวดล้อม" (Environment) ที่เรากำลังทำงานอยู่

**1. การติดตั้งและการทำงานของ Python:**
ภาษาไพธอนเป็นภาษาโปรแกรมมิ่งระดับสูง (High-level Programming Language) ที่ถูกออกแบบมาให้มีไวยากรณ์ที่ใกล้เคียงกับภาษาอังกฤษ ทำให้มนุษย์สามารถอ่านและทำความเข้าใจโครงสร้างของโค้ดได้ง่าย

**2. Python Sandbox (สภาพแวดล้อมจำลอง):**
สำหรับผู้เริ่มต้น เราจะใช้สิ่งที่เรียกว่า **Sandbox** ซึ่งคือพื้นที่จำลอง (Simulated Area) ที่ช่วยให้เราสามารถรันโค้ด (Execute Code) และทดสอบผลลัพธ์ (Test Output) ได้ทันทีโดยไม่จำเป็นต้องติดตั้งระบบปฏิบัติการที่ซับซ้อน การทำงานใน Sandbox นี้จะช่วยให้เราสามารถมุ่งเน้นไปที่ตรรกะ (Logic) ของโปรแกรมได้อย่างเต็มที่

**หลักการทำงาน:** เมื่อเราพิมพ์โค้ด Python ลงใน Sandbox แล้วกดปุ่มรัน (Run) ระบบจะทำการแปลภาษา (Interpret) โค้ดเหล่านั้นทีละบรรทัด และส่งผลลัพธ์ (Output) ออกมาให้เราเห็นทันที

### 1.2.2 ชนิดข้อมูลพื้นฐานและโครงสร้างข้อมูล (Data Types and Data Structures)

ในทางฟิสิกส์หรือเกษตรกรรม ทุกค่าที่เราวัดได้ ไม่ว่าจะเป็นอุณหภูมิ ความชื้น หรือปริมาณแสง ล้วนเป็นข้อมูล (Data) ที่ต้องถูกจัดเก็บในรูปแบบที่คอมพิวเตอร์เข้าใจ ซึ่งเราเรียกสิ่งเหล่านี้ว่า **ชนิดข้อมูล (Data Types)**

#### 2.2.1 ชนิดข้อมูลพื้นฐาน (Primitive Data Types)

| ชนิดข้อมูล (Type) | คำอธิบาย (Description) | ตัวอย่าง (Example) | การใช้งานทางฟิสิกส์/เกษตรกรรม |
| :--- | :--- | :--- | :--- |
| **Integer (`int`)** | จำนวนเต็มที่ไม่มีส่วนทศนิยม (Whole numbers: $\dots, -2, -1, 0, 1, 2, \dots$) | `10`, `-50` | จำนวนครั้งของการวัด, จำนวนวัน |
| **Floating Point (`float`)** | จำนวนจริงที่มีส่วนทศนิยม (Real numbers) | `25.5`, `-0.8` | ค่าความชื้นดิน (Soil Moisture), อุณหภูมิ ($\text{°C}$), ความเข้มแสง ($\text{W}/\text{m}^2$) |
| **String (`str`)** | ข้อความตัวอักษร (Sequence of characters) | `"ทุเรียน"`, `"Warning!"` | ชื่อพืช, คำเตือน, คำอธิบายผลลัพธ์ |
| **Boolean (`bool`)** | ค่าความจริง มีเพียงสองสถานะเท่านั้น (True หรือ False) | `True`, `False` | การตรวจสอบเงื่อนไข (เช่น: *ความชื้นต่ำกว่าเกณฑ์หรือไม่?*) |

**ตัวอย่างการกำหนดค่า (Variable Assignment):**
การกำหนดค่าใน Python ใช้เครื่องหมายเท่ากับ (`=`)

```python
# การกำหนดตัวแปร (Variable Assignment)
soil_moisture_level = 0.35  # float: ระดับความชื้นดิน (หน่วย: fraction)
critical_threshold = 0.20   # float: เกณฑ์วิกฤต
is_warning_active = True    # bool: สถานะเตือนภัย
orchard_name = "ทุเรียนทองคำ" # str: ชื่อแปลง
```

#### 2.2.2 โครงสร้างข้อมูลแบบลำดับ (Collection Data Structures)

เมื่อเราต้องการเก็บข้อมูลหลายค่าที่เกี่ยวข้องกัน เราจะไม่ใช้ตัวแปรเดี่ยวๆ แต่จะใช้โครงสร้างข้อมูลเหล่านี้:

**1. List (รายการ):**
*   **คำอธิบาย:** คือกลุ่มของข้อมูลที่สามารถเก็บชนิดข้อมูลที่แตกต่างกันได้หลายค่า และสามารถแก้ไขลำดับ (Ordered) ได้
*   **การใช้งาน:** เหมาะสำหรับการเก็บชุดข้อมูลการวัดที่เกิดขึ้นตามลำดับเวลา (Time Series Data) เช่น ค่าความชื้นดิน 5 ค่าที่วัดได้ใน 5 ชั่วโมง

```python
# ตัวอย่าง List: เก็บค่าความชื้นดิน 5 จุด
soil_moisture_readings = [0.45, 0.38, 0.25, 0.19, 0.15]
```

**2. Dictionary (พจนานุกรม):**
*   **คำอธิบาย:** คือการจัดเก็บข้อมูลในรูปแบบของคู่ **คีย์:ค่า (Key:Value)** ซึ่งทำให้การเรียกใช้ข้อมูลมีความรวดเร็วและเข้าใจง่ายกว่าการใช้ Index (ลำดับ)
*   **การใช้งาน:** เหมาะสำหรับการเก็บข้อมูลที่ระบุคุณสมบัติเฉพาะของระบบ เช่น การระบุค่าความชื้นของเซนเซอร์แต่ละตัว

```python
# ตัวอย่าง Dictionary: เก็บข้อมูลเซนเซอร์หลายตัว
sensor_data = {
    "moisture_sensor_A": 0.35,  # Key: ชื่อเซนเซอร์, Value: ค่าความชื้น
    "temperature_sensor_B": 28.5, # Key: ชื่อเซนเซอร์, Value: อุณหภูมิ
    "humidity_sensor_C": 75.0   # Key: ชื่อเซนเซอร์, Value: ความชื้นสัมพัทธ์
}
```

### 1.2.3 โครงสร้างควบคุมทิศทางโปรแกรม (Control Flow Structures)

หากตัวแปรคือ "ข้อมูล" (Data) โครงสร้างควบคุมทิศทางโปรแกรมก็คือ "ตรรกะ" (Logic) ที่กำหนดว่าโปรแกรมจะตัดสินใจทำอะไรต่อไป

#### 3.2.1 โครงสร้างเงื่อนไข (Conditional Statements): `if-elif-else`

โครงสร้างนี้ใช้ในการตัดสินใจ (Decision Making) ว่าจะดำเนินการชุดคำสั่งใดชุดคำสั่งหนึ่งหรือไม่ โดยอ้างอิงจากเงื่อนไขทางตรรกะ (Boolean Condition)

**หลักการ:**
1.  **`if`:** ตรวจสอบเงื่อนไขแรก หากเป็นจริง (True) ให้ทำชุดคำสั่งนี้
2.  **`elif` (Else If):** หากเงื่อนไขแรกเป็นเท็จ (False) ให้ตรวจสอบเงื่อนไขที่สอง หากเป็นจริง ให้ทำชุดคำสั่งนี้
3.  **`else`:** หากทุกเงื่อนไขข้างต้นเป็นเท็จทั้งหมด ให้ทำชุดคำสั่งสุดท้ายนี้

**ตัวอย่างทางคณิตศาสตร์ (Mathematical Analogy):**
สมมติให้ $M$ คือระดับความชื้นดิน และ $T_{crit}$ คือเกณฑ์วิกฤต
$$\text{IF } M < T_{crit} \text{ THEN } \text{Warning} = \text{High}$$
$$\text{ELSE IF } M < T_{crit} + 0.1 \text{ THEN } \text{Warning} = \text{Medium}$$
$$\text{ELSE } \text{Warning} = \text{Normal}$$

#### 3.2.2 การวนลูป (Loops): `for` และ `while`

การวนลูปใช้เพื่อทำซ้ำชุดคำสั่งเดิมๆ หลายครั้ง โดยไม่ต้องเขียนโค้ดซ้ำๆ ซึ่งช่วยให้โปรแกรมมีประสิทธิภาพสูง

**1. `for` Loop (การวนซ้ำตามจำนวนที่กำหนด):**
*   **หลักการ:** ใช้เมื่อเรารู้จำนวนรอบที่ต้องการทำซ้ำ หรือต้องการวนผ่านสมาชิกทั้งหมดในโครงสร้างข้อมูล (เช่น List หรือ Dictionary)
*   **ไวยากรณ์:** `for item in sequence:`

**2. `while` Loop (การวนซ้ำตามเงื่อนไข):**
*   **หลักการ:** ใช้เมื่อเราต้องการให้โปรแกรมทำงานซ้ำไปเรื่อยๆ ตราบใดที่เงื่อนไขที่กำหนดยังคงเป็นจริง (True)
*   **ข้อควรระวัง:** ต้องแน่ใจว่ามีกลไกบางอย่างที่ทำให้เงื่อนไขกลายเป็นเท็จในที่สุด มิฉะนั้นโปรแกรมจะเกิด **Infinite Loop** (การวนลูปไม่สิ้นสุด)

### 1.2.4 การประยุกต์ใช้: การจำลองระบบตรวจจับสภาวะวิกฤตความชื้นดิน (Soil Moisture Crisis Detection Simulation)

ในภาคปฏิบัติ เราจะจำลองสถานการณ์ที่ระบบเกษตรอัจฉริยะ (Smart Farming System) ทำการอ่านค่าความชื้นดินของแปลงทุเรียน และตัดสินใจส่งคำเตือนภัย (Warning Alert) หากค่าความชื้นต่ำกว่าเกณฑ์ที่กำหนด

**สมมติฐานทางฟิสิกส์และเกษตรกรรม:**
1.  **เกณฑ์วิกฤต (Critical Threshold):** ความชื้นดินที่ต่ำกว่า $0.20$ (หน่วยเป็นเศษส่วนของปริมาตร) ถือเป็นสภาวะวิกฤต (Critical State)
2.  **การวัดค่า:** เราจะจำลองการวัดค่าความชื้นดิน 5 ครั้งติดต่อกัน (5 Readings)
3.  **การตอบสนอง:** ระบบต้องตรวจสอบค่าที่วัดได้แต่ละค่า และแจ้งเตือนสถานะที่เหมาะสม

#### 💻 โค้ด Python สำหรับการจำลองระบบ (Python Source Code)

```python
# -*- coding: utf-8 -*-
# ----------------------------------------------------------------------
# ระบบจำลองการตรวจจับสภาวะวิกฤตความชื้นดินสำหรับแปลงทุเรียน
# โดย รศ.ดร. [ชื่อสมมติ]
# ----------------------------------------------------------------------

# 1. การกำหนดค่าคงที่ทางวิศวกรรม (Defining Constants)
CRITICAL_MOISTURE_THRESHOLD = 0.20  # เกณฑ์วิกฤต (float)
MIN_MOISTURE_THRESHOLD = 0.10      # เกณฑ์ต่ำสุดที่ยอมรับได้ (float)
ORCHARD_NAME = "ทุเรียนพันธุ์หมอนทอง" # ชื่อแปลง (str)

# 2. ข้อมูลจำลองการวัดค่าความชื้นดิน (Simulated Data)
# List: เก็บค่าความชื้นดินที่วัดได้ 5 ครั้ง (หน่วย: fraction)
soil_moisture_readings = [0.45, 0.38, 0.25, 0.19, 0.15]

def analyze_soil_moisture(readings: list) -> str:
    """
    ฟังก์ชันสำหรับวิเคราะห์ระดับความชื้นดินและกำหนดระดับความเสี่ยง
    :param readings: รายการค่าความชื้นดินที่วัดได้
    :return: ข้อความสรุปสถานะความเสี่ยง
    """
    print("========================================================")
    print(f"🔬 [ระบบวิเคราะห์] การตรวจสอบความชื้นดิน ณ แปลง: {ORCHARD_NAME}")
    print("========================================================")
    
    # 3. การวนลูปเพื่อประมวลผลข้อมูลแต่ละค่า (Using 'for' loop)
    print("\n--- 📊 ผลการวัดค่าความชื้นดินแต่ละรอบ ---")
    for i, moisture_value in enumerate(readings):
        # การแสดงผลค่าที่วัดได้ พร้อมลำดับรอบที่ (i+1)
        print(f"รอบที่ {i+1}: ค่าความชื้นดินที่วัดได้ = {moisture_value:.2f} (fraction)")
        
        # 4. โครงสร้างเงื่อนไขเพื่อตัดสินใจ (Using 'if-elif-else')
        if moisture_value < CRITICAL_MOISTURE_THRESHOLD:
            # กรณีวิกฤต: ต่ำกว่าเกณฑ์วิกฤต
            print("🚨 [คำเตือนวิกฤต] ระดับความชื้นต่ำกว่าเกณฑ์วิกฤต! ต้องทำการรดน้ำทันที!")
            print("   (Action: ปั๊มน้ำอัตโนมัติ, แจ้งเตือนเจ้าหน้าที่)")
            
        elif moisture_value < MIN_MOISTURE_THRESHOLD:
            # กรณีเตือน: ต่ำกว่าเกณฑ์ต่ำสุดที่ยอมรับได้ แต่ยังไม่วิกฤต
            print("⚠️ [คำเตือนระดับกลาง] ระดับความชื้นต่ำกว่าปกติ ควรเฝ้าระวังและเตรียมการรดน้ำ")
            
        else:
            # กรณีปกติ: อยู่ในเกณฑ์ที่ยอมรับได้
            print("✅ [สถานะปกติ] ระดับความชื้นอยู่ในเกณฑ์ที่เหมาะสมต่อการเจริญเติบโต")
```
