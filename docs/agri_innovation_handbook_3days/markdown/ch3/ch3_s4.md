## 3.4 ตรรกะระบบควบคุมป้อนกลับ (Feedback Control Loop) สั่งงานวงจรไฟฟ้าปั๊มน้ำด้วยรีเลย์

# 3.4 ตรรกะระบบควบคุมป้อนกลับ (Feedback Control Loop) สำหรับการสั่งงานวงจรไฟฟ้าปั๊มน้ำด้วยรีเลย์

**(ระดับความลึก: การวิเคราะห์ระบบควบคุมเชิงวิศวกรรม, การประยุกต์ใช้ไฟฟ้ากำลัง, การเขียนโปรแกรมไมโครคอนโทรลเลอร์)**

---

## บทนำเชิงทฤษฎี: หลักการควบคุมระบบอัตโนมัติ

ในสาขาวิชาวิศวกรรมเกษตรดิจิทัล (Digital Agriculture Engineering) การควบคุมสภาพแวดล้อมของพืชผลให้เหมาะสมที่สุดเป็นหัวใจสำคัญของการเพิ่มผลผลิตอย่างยั่งยืน (Sustainable Yield Enhancement) หนึ่งในตัวแปรทางกายภาพที่สำคัญที่สุดที่ต้องได้รับการควบคุมอย่างแม่นยำคือความชื้นในดิน (Soil Moisture)

ระบบควบคุมแบบป้อนกลับ (Feedback Control Loop) คือรากฐานทางทฤษฎีที่ใช้ในการรักษาสภาวะของตัวแปรทางกายภาพ (Process Variable, PV) ให้คงที่ใกล้เคียงกับค่าที่กำหนดไว้ล่วงหน้า (Set Point, SP) โดยอาศัยการวัดค่าที่เกิดขึ้นจริงและนำความแตกต่างนั้นมาปรับปรุงการทำงานของอุปกรณ์ขับเคลื่อน (Actuator) อย่างต่อเนื่อง

### 3.4.1 หลักการควบคุมป้อนกลับ (Feedback Control Loop) และระบบ On-Off Controller

#### 3.4.1.1 แนวคิดพื้นฐานของวงจรควบคุมป้อนกลับ

ระบบควบคุมป้อนกลับเป็นวงจรที่ประกอบด้วยองค์ประกอบหลัก 3 ส่วน ได้แก่:
1. **ตัวแปรกระบวนการ (Process Variable, PV):** คือค่าที่วัดได้จริง ณ จุดปฏิบัติการ เช่น ความชื้นในดิน (Soil Moisture Content) ที่วัดได้จากเซนเซอร์
2. **ค่าตั้งต้น (Set Point, SP):** คือค่าเป้าหมายที่ต้องการให้ระบบรักษาไว้ เช่น ค่าความชื้นที่เหมาะสมที่สุด (Optimal VWC)
3. **ตัวควบคุม (Controller):** คือหน่วยประมวลผล (เช่น ไมโครคอนโทรลคอนโทรลเลอร์) ที่คำนวณหาความผิดพลาด (Error Signal, $e(t)$) และส่งสัญญาณคำสั่งไปยังตัวขับเคลื่อน

ความผิดพลาด (Error Signal) ถูกนิยามทางคณิตศาสตร์ดังนี้:
$$e(t) = SP - PV$$

เมื่อ $e(t)$ มีค่าเป็นบวก หมายความว่า $PV < SP$ (เช่น ความชื้นต่ำกว่าที่กำหนด) และเมื่อ $e(t)$ มีค่าเป็นลบ หมายความว่า $PV > SP$ (เช่น ความชื้นสูงกว่าที่กำหนด)

#### 3.4.1.2 การควบคุมแบบสองตำแหน่ง (On-Off Control) และปรากฏการณ์ฮิสเทรีซิส (Hysteresis)

ในการควบคุมความชื้นในดินด้วยปั๊มน้ำ เรามักใช้กลไกควบคุมแบบ On-Off Controller เนื่องจากมีความเรียบง่ายในการประยุกต์ใช้และมีประสิทธิภาพเพียงพอสำหรับงานเกษตรกรรมส่วนใหญ่

**On-Off Controller** คือตัวควบคุมที่ให้เอาต์พุตเพียงสองสถานะ คือ "เปิด (ON)" หรือ "ปิด (OFF)" โดยไม่มีการปรับค่าเอาต์พุตอย่างต่อเนื่อง (เช่น PID Controller ที่จะปรับกำลังปั๊มตามความผิดพลาด)

อย่างไรก็ตาม การใช้ On-Off Controller โดยไม่มีการกำหนดขอบเขตที่ชัดเจนจะทำให้เกิดปรากฏการณ์ที่เรียกว่า **การสวิง (Cycling)** อย่างรวดเร็ว ซึ่งอาจส่งผลเสียต่ออายุการใช้งานของปั๊มและรีเลย์

เพื่อแก้ไขปัญหานี้ เราจึงต้องใช้หลักการ **ฮิสเทรีซิส (Hysteresis)** ซึ่งคือการกำหนดช่วงค่า (Bandwidth) ของความผิดพลาดที่ยอมให้ระบบทำงานก่อนที่จะสลับสถานะเอาต์พุต

เราจะกำหนดขีดจำกัดสองค่า:
1. **ขีดเริ่มต่ำสุด (Lower Threshold, $T_{low}$):** เมื่อ $PV < T_{low}$ ระบบต้องสั่งเปิดปั๊ม (Pump ON)
2. **ขีดเริ่มสูงสุด (Upper Threshold, $T_{high}$):** เมื่อ $PV > T_{high}$ ระบบต้องสั่งปิดปั๊ม (Pump OFF)

โดยที่ $T_{high} > T_{low}$ เพื่อให้เกิดช่วงความคลาดเคลื่อน (Deadband) ที่เหมาะสม

**สมการตรรกะการควบคุม (Control Logic):**
$$\text{IF } (PV < T_{low}) \text{ THEN } \text{Output} = \text{ON}$$
$$\text{ELSE IF } (PV > T_{high}) \text{ THEN } \text{Output} = \text{OFF}$$
$$\text{ELSE } \text{Output} = \text{Maintain Current State}$$

### 3.4.2 การเชื่อมต่อทางไฟฟ้า: รีเลย์และหลักการแยกสัญญาณ (Isolation)

ในทางปฏิบัติ ระบบควบคุมที่ใช้ไมโครคอนโทรลเลอร์ (Microcontroller) ซึ่งทำงานด้วยแรงดันไฟฟ้าต่ำ (Low Voltage, เช่น 3.3 V หรือ 5 V) ไม่สามารถจ่ายกำลังไฟฟ้าที่สูงพอที่จะขับเคลื่อนปั๊มน้ำ (ซึ่งอาจใช้แรงดันไฟฟ้ากระแสสลับ AC 220 V หรือ DC 12 V) ได้โดยตรง

ดังนั้น เราจึงต้องใช้ **โมดูลรีเลย์ (Relay Module)** ซึ่งทำหน้าที่เป็นสวิตช์ทางไฟฟ้า (Electromechanical Switch) ที่แยกวงจรควบคุมออกจากวงจรโหลด (Load Circuit) อย่างสมบูรณ์

#### 3.4.2.1 หลักการทำงานของรีเลย์ (Relay Operation)

รีเลย์ประกอบด้วยขดลวดแม่เหล็กไฟฟ้า (Electromagnet Coil) และหน้าสัมผัส (Contacts) เมื่อจ่ายกระแสไฟฟ้ากระแสตรง (DC) ไปยังขดลวด จะเกิดสนามแม่เหล็ก (Magnetic Field) ดึงดูดหน้าสัมผัสให้ปิดวงจร ทำให้กระแสไฟฟ้าแรงสูงสามารถไหลผ่านหน้าสัมผัสไปยังโหลดได้

#### 3.4.2.2 การแยกสัญญาณด้วยออปโตคัปเปลอร์ (Optocoupler Isolation)

ในระบบควบคุมที่ซับซ้อน การเชื่อมต่อวงจรควบคุม (Microcontroller) เข้ากับวงจรขับเคลื่อน (Relay Coil) โดยตรงอาจทำให้เกิดปัญหาทางไฟฟ้าหลายประการ เช่น:
1. **Ground Loop:** ความต่างศักย์ของกราวด์ที่อาจทำให้เกิดสัญญาณรบกวน (Noise)
2. **Voltage Spike:** ไฟกระชากกลับ (Back EMF) เมื่อขดลวดแม่เหล็กไฟฟ้าถูกตัดกระแส

เพื่อป้องกันปัญหานี้ เราจึงต้องใช้ **ออปโตคัปเปลอร์ (Optocoupler)** หรือ **ออปโตไอโซเลเตอร์ (Optoisolator)** ซึ่งเป็นอุปกรณ์ที่ใช้หลักการส่งสัญญาณด้วยแสง (Optical Signal Transmission)

**หลักการ:** ออปโตคัปเปลอร์จะแยกวงจรไฟฟ้าควบคุม (Input Side) ออกจากวงจรขับเคลื่อน (Output Side) อย่างสมบูรณ์ด้วยการใช้แสงเป็นตัวกลางในการส่งสัญญาณ ทำให้เกิด **การแยกทางไฟฟ้า (Galvanic Isolation)** ซึ่งเป็นมาตรการด้านความปลอดภัยทางไฟฟ้าที่สำคัญยิ่ง

#### 3.4.2.3 ปรากฏการณ์ Active Low

โมดูลรีเลย์ส่วนใหญ่ที่ใช้กับไมโครคอนโทรลเลอร์ถูกออกแบบมาให้ทำงานในโหมด **Active Low** หมายความว่า:
*   **สถานะ OFF (ปกติ):** ขาควบคุม (Control Pin) ของรีเลย์จะถูกดึงให้เป็นระดับแรงดันไฟฟ้าสูง (High Voltage, $V_{CC}$) หรือถูกปล่อยลอย (Floating)
*   **สถานะ ON (ทำงาน):** ต้องจ่ายสัญญาณแรงดันไฟฟ้าต่ำ (Low Voltage, $GND$) หรือระดับแรงดันไฟฟ้าที่ต่ำกว่า $V_{CC}$ เข้าไปที่ขาควบคุม เพื่อให้เกิดการทำงานของรีเลย์

ดังนั้น ในการเขียนโค้ด เราจะต้องกำหนดให้สถานะ "เปิด" (Activate) คือการส่งสัญญาณ LOW ไปยังขาควบคุมของรีเลย์

### 3.4.3 การออกแบบตรรกะระบบควบคุมเชิงปฏิบัติ (Algorithmic Design)

การเขียนโปรแกรมควบคุมต้องคำนึงถึงประสิทธิภาพและความเสถียรของระบบเป็นหลัก โดยเฉพาะอย่างยิ่งการหลีกเลี่ยงการหยุดการทำงานของโปรแกรม (Blocking) ซึ่งจะทำให้ระบบไม่สามารถอ่านค่าเซนเซอร์อื่น ๆ หรือตอบสนองต่อเหตุการณ์อื่น ๆ ได้

เราจะใช้เทคนิค **Non-blocking delay** โดยอาศัยฟังก์ชัน `millis()` เพื่อติดตามเวลาที่ผ่านไปแทนการใช้ฟังก์ชัน `delay()` แบบดั้งเดิม

#### 3.4.3.1 โครงสร้างการทำงานของโปรแกรม (Program Flow)

1. **Initialization:** กำหนดค่าขีดจำกัด ($T_{low}, T_{high}$) และกำหนดสถานะเริ่มต้นของรีเลย์ (OFF)
2. **Loop Cycle:** ทำซ้ำอย่างต่อเนื่อง
    a. **Read Sensor:** อ่านค่าความชื้นในดินเชิงปริมาตร (Volumetric Water Content, VWC) จากเซนเซอร์
    b. **Check Threshold:** เปรียบเทียบค่า VWC ที่อ่านได้กับ $T_{low}$ และ $T_{high}$
    c. **Control Logic Execution:**
        *   ถ้า $VWC < T_{low}$: สั่งเปิดรีเลย์ (Activate Pump)
        *   ถ้า $VWC > T_{high}$: สั่งปิดรีเลย์ (Deactivate Pump)
        *   ถ้าอยู่ในช่วง $T_{low} \le VWC \le T_{high}$: คงสถานะเดิม (Maintain State)
    d. **Non-blocking Delay:** หน่วงเวลาการวนรอบ (Cycle Time) โดยไม่บล็อกการทำงานของไมโครคอนโทรลคอนโทรลเลอร์

### 3.4.4 การประยุกต์ใช้โค้ด (C++ Arduino Implementation)

เราจะใช้ภาษา C++ สำหรับแพลตฟอร์ม Arduino ซึ่งเป็นมาตรฐานในการพัฒนาโปรเจกต์ IoT และระบบควบคุมขนาดเล็ก

**สมมติฐานของฮาร์ดแวร์:**
*   **Sensor Pin:** ขาที่เชื่อมต่อกับเซนเซอร์วัดความชื้น (Analog Input)
*   **Relay Pin:** ขาควบคุมที่เชื่อมต่อกับโมดูลรีเลย์ (Digital Output)

```cpp
// ========================================================================================
// [รศ.ดร. Academic Code Listing]
// หัวข้อ: 3.4 ตรรกะระบบควบคุมป้อนกลับ (Feedback Control Loop)
// ภาษา: C++ (Arduino Sketch)
// วัตถุประสงค์: ควบคุมปั๊มน้ำโดยใช้หลักการฮิสเทรีซิส (Hysteresis) และ Non-blocking delay
// ========================================================================================

// --- 1. การกำหนดค่าคงที่ (Constants Definition) ---
// กำหนดขา GPIO ที่ใช้ในการเชื่อมต่อกับอุปกรณ์
const int SENSOR_PIN = A0;    // ขา Analog สำหรับอ่านค่าความชื้นในดิน (Soil Moisture Sensor)
const int RELAY_PIN = 7;      // ขา Digital สำหรับควบคุมโมดูลรีเลย์ (Relay Module)

// กำหนดค่าขีดจำกัดความชื้น (Thresholds) ในหน่วย % VWC (Volumetric Water Content)
// ค่าเหล่านี้ควรปรับให้เหมาะสมกับชนิดของดินและพืชที่ปลูก
const float LOWER_THRESHOLD = 35.0; // T_low: ขีดเริ่มต่ำสุด (เมื่อต่ำกว่านี้ ต้องเปิดปั๊ม)
const float UPPER_THRESHOLD = 65.0; // T_high: ขีดเริ่มสูงสุด (เมื่อสูงกว่านี้ ต้องปิดปั๊ม)

// กำหนดช่วงเวลาหน่วงรอบการทำงานของระบบควบคุม (Cycle Time) เป็นมิลลิวินาที (ms)
// การหน่วงเวลานี้ช่วยลดการสวิงของรีเลย์และลดการใช้พลังงาน
const long CONTROL_CYCLE_INTERVAL = 5000; // 5 วินาทีต่อรอบการอ่านค่า

// --- 2. ตัวแปรสถานะ (State Variables) ---
// ตัวแปรสำหรับติดตามเวลาการทำงานรอบล่าสุด (Non-blocking delay mechanism)
unsigned long previousMillis = 0; 

// ตัวแปรสำหรับติดตามสถานะปัจจุบันของปั๊ม (Pump State)
// เพื่อป้องกันการสลับสถานะที่ไม่จำเป็น (State Machine Logic)
bool isPumpActive = false; 

// ========================================================================================
// ฟังก์ชัน Setup: การเริ่มต้นระบบ (Initialization)
// ========================================================================================
void setup() {
  // เริ่มต้นการสื่อสาร Serial Monitor เพื่อแสดงผลการทำงานของระบบ (Debugging/Monitoring)
  Serial.begin(9600);
  Serial.println("=========================================================");
  Serial.println("ระบบควบคุมความชื้นดินด้วยตรรกะป้อนกลับ (Feedback Control System)");
  Serial.println("=========================================================");

  // กำหนดขาควบคุมรีเลย์เป็น OUTPUT
  pinMode(RELAY_PIN, OUTPUT);
  
  // ตั้งค่าสถานะเริ่มต้นของรีเลย์ให้เป็น OFF (Deactivate)
  // เนื่องจากโมดูลรีเลย์ส่วนใหญ่เป็น Active Low, การตั้งค่า HIGH คือสถานะ OFF
  digitalWrite(RELAY_PIN, HIGH); 
  
  Serial.println("System Initialized. Pump Status: OFF.");
}

// ========================================================================================
// ฟังก์ชัน Loop: วงจรการทำงานหลัก (Main Control Loop)
// ========================================================================================
void loop() {
  // 1. การตรวจสอบเวลา (Timing Check)
  // ใช้ millis() เพื่อตรวจสอบว่าถึงเวลาที่ต้องทำการอ่านค่าและประมวลผลรอบใหม่หรือไม่
  unsigned long currentMillis = millis();

  if (currentMillis - previousMillis >= CONTROL_CYCLE_INTERVAL) {
    // บันทึกเวลาปัจจุบันเป็นเวลาที่ทำการควบคุมรอบล่าสุด
    previousMillis = currentMillis; 
    
    // 2. การอ่านค่าเซนเซอร์ (Sensor Reading)
    // อ่านค่าความชื้นในดินเชิงปริมาตร (VWC) จากเซนเซอร์
    // (สมมติว่ามีฟังก์ชัน readSoilMoistureVWC() ที่แปลงค่า ADC ดิบเป็นเปอร์เซ็นต์ VWC เรียบร้อยแล้ว)
    float currentVWC = readSoilMoistureVWC();
    
    // 3. ตรรกะการควบคุมป้อนกลับเชิงระดับ (Feedback Control Logic with Hysteresis)
    if (currentVWC < LOWER_THRESHOLD) {
      // ความชื้นในดินต่ำกว่าขีดเริ่มล่าง -> เปิดปั๊มน้ำ
      isPumpActive = true;
      // สั่งงานรีเลย์ (Active Low: สั่ง LOW เพื่อให้รีเลย์นำกระแส)
      digitalWrite(RELAY_PIN, LOW); 
      Serial.print("[PUMP ALERT] Soil is too dry! VWC: ");
      Serial.print(currentVWC);
      Serial.println("% | Pump Status: ON");
    } 
    else if (currentVWC > UPPER_THRESHOLD) {
      // ความชื้นในดินสูงกว่าขีดเริ่มบน -> ปิดปั๊มน้ำเพื่อป้องกันน้ำล้น/รากเน่า
      isPumpActive = false;
      // สั่งงานรีเลย์ (Active Low: สั่ง HIGH เพื่อตัดกระแส)
      digitalWrite(RELAY_PIN, HIGH);
      Serial.print("[PUMP ALERT] Soil is saturated. VWC: ");
      Serial.print(currentVWC);
      Serial.println("% | Pump Status: OFF");
    }
    else {
      // ความชื้นอยู่ในช่วงปลอดภัย (LOWER_THRESHOLD <= VWC <= UPPER_THRESHOLD)
      // คงสถานะเดิมของปั๊มไว้ (Hysteresis Deadband)
      Serial.print("[SYSTEM OK] Soil moisture is stable. VWC: ");
      Serial.print(currentVWC);
      Serial.print("% | Pump Status: ");
      Serial.println(isPumpActive ? "ON" : "OFF");
    }
  }
}

/**
 * @brief ฟังก์ชันจำลองการอ่านค่าความชื้นเชิงปริมาตร (% VWC)
 * @return เปอร์เซ็นต์ความชื้นดินจำลอง
 */
float readSoilMoistureVWC() {
  int rawADC = analogRead(SENSOR_PIN);
  // สมมติช่วง ADC ของระบบ 10-bit (0 - 1023)
  // 300 คือจุดอิ่มน้ำ (Wet), 800 คือจุดแห้ง (Dry)
  if (rawADC <= 300) return 100.0;
  if (rawADC >= 800) return 0.0;
  return ((800.0 - (float)rawADC) / 500.0) * 100.0;
}
```
