## 3.2 บอร์ดฝังตัว ESP32, ไดรเวอร์ CH340 และสิ่งแวดล้อมโปรแกรม Arduino IDE

# 3.2 บอร์ดฝังตัว ESP32, ไดรเวอร์ CH340 และสิ่งแวดล้อมโปรแกรม Arduino IDE

ในฐานะผู้เชี่ยวชาญด้านฟิสิกส์ประยุกต์และระบบเกษตรดิจิทัล (Digital Agriculture) การทำความเข้าใจกับสถาปัตยกรรมของบอร์ดฝังตัว (Embedded Board) เป็นรากฐานสำคัญในการพัฒนาระบบ Internet of Things (IoT) และระบบอัตโนมัติ (Automation System) บทที่ 3.2 นี้จะเจาะลึกถึงองค์ประกอบหลักของบอร์ด ESP32, ความสำคัญของตัวแปลงสัญญาณ (Converter Chip) เช่น CH340, และการตั้งค่าสภาพแวดล้อมการพัฒนา (Development Environment) ด้วย Arduino IDE อย่างละเอียดถี่ถ้วน

---

## 3.2.1 สถาปัตยกรรมของไมโครคอนโทรลเลอร์ ESP32

ESP32 เป็นไมโครคอนโทรลเลอร์ (Microcontroller Unit: MCU) ที่พัฒนาโดย Espressif Systems ซึ่งถูกออกแบบมาเพื่อตอบโจทย์การประมวลผลในระบบ IoT โดยเฉพาะ จุดเด่นที่สำคัญที่สุดของ ESP32 คือการรวมเอาคุณสมบัติการเชื่อมต่อไร้สาย (Wireless Connectivity) เข้ากับหน่วยประมวลผลหลัก (Central Processing Unit: CPU) ในชิปเดียว

### 3.2.1.1 โครงสร้างทางสถาปัตยกรรม (Architectural Structure)

ESP32 ถูกสร้างขึ้นบนสถาปัตยกรรมแบบ **Tensilica Xtensa LX6** ซึ่งเป็นสถาปัตยกรรมแบบ 32 บิต (32-bit architecture) การใช้สถาปัตยกรรม 32 บิตหมายความว่า MCU สามารถจัดการกับข้อมูล (Data) ได้ทีละ 32 บิต ซึ่งช่วยเพิ่มประสิทธิภาพในการประมวลผล (Processing Efficiency) อย่างมาก เมื่อเทียบกับสถาปัตยกรรม 8 บิต หรือ 16 บิต

องค์ประกอบหลักทางสถาปัตยกรรมของ ESP32 ประกอบด้วย:

1.  **หน่วยประมวลผลกลาง (CPU Core):** ESP32 มักจะมีแกนประมวลผลหลัก (Core) อย่างน้อยหนึ่งแกน (Core) และสามารถมีแกนที่สองได้ (Dual-core) โดยแต่ละแกนสามารถทำงานแบบอิสระ (Independent Operation) ซึ่งช่วยให้สามารถรันงานหลายอย่างพร้อมกัน (Multitasking) ได้ เช่น แกนหนึ่งจัดการการเชื่อมต่อ Wi-Fi ขณะที่อีกแกนจัดการการอ่านค่าเซนเซอร์ (Sensor Reading)
2.  **โมดูลการสื่อสารไร้สาย (Wireless Modules):** ESP32 มีโมดูล Wi-Fi (IEEE 802.11 b/g/n) และ Bluetooth (Classic Bluetooth และ Bluetooth Low Energy: BLE) ในตัว ซึ่งช่วยลดความซับซ้อนในการออกแบบวงจรภายนอก (External Circuitry) และทำให้บอร์ดมีขนาดเล็กลง
3.  **หน่วยความจำ (Memory):**
    *   **RAM (Random Access Memory):** ใช้สำหรับเก็บข้อมูลที่กำลังถูกประมวลผลชั่วคราว (Temporary Data)
    *   **Flash Memory:** ใช้สำหรับเก็บโค้ดโปรแกรม (Firmware) และข้อมูลที่ถาวร (Persistent Data)
4.  **ส่วนต่อประสาน (Peripherals):** ประกอบด้วย GPIO, ADC, DAC, I²C, SPI, UART และ Timer ซึ่งเป็นส่วนที่ใช้ในการติดต่อสื่อสารกับอุปกรณ์ภายนอก (External Devices)

### 3.2.1.2 หลักการทำงานของ Wi-Fi และ Bluetooth

การรวมโมดูลไร้สายเข้ากับ MCU ทำให้ ESP32 สามารถทำหน้าที่เป็นโหนด (Node) ในเครือข่าย IoT ได้ทันที

*   **Wi-Fi:** ทำงานตามมาตรฐาน IEEE 802.11 โดยใช้คลื่นความถี่ 2.4 GHz มีอัตราการส่งข้อมูล (Data Rate) ที่เหมาะสมสำหรับการส่งข้อมูลขนาดเล็กถึงปานกลาง เช่น การส่งค่าเซนเซอร์ไปยังคลาวด์ (Cloud)
*   **Bluetooth:** ใช้สำหรับการสื่อสารระยะใกล้ (Short-range communication) เหมาะสำหรับการเชื่อมต่อกับอุปกรณ์ปลายทาง (End Devices) เช่น โทรศัพท์มือถือ หรือเซนเซอร์เฉพาะจุด

## 3.2.2 การทำความเข้าใจแผนผังขาพิน (Pinout Map) ของ ESP32

การทำความเข้าใจว่าขาพินใดทำหน้าที่อะไรเป็นสิ่งสำคัญอย่างยิ่งในการเชื่อมต่อวงจรภายนอก (External Circuit Interfacing) ESP32 มีขาพินที่หลากหลายประเภท ซึ่งแต่ละประเภทมีข้อจำกัดในการใช้งานที่แตกต่างกัน

### 3.2.2.1 ประเภทของขาพิน (Pin Types)

| ประเภทขาพิน | คำย่อ | หน้าที่หลัก | คำอธิบายทางเทคนิค |
| :--- | :--- | :--- | :--- |
| **General Purpose Input/Output** | GPIO | ขาอินพุต/เอาต์พุตดิจิทัล | สามารถตั้งค่าให้เป็นเอาต์พุต (Output) เพื่อขับสัญญาณดิจิทัล (Digital Signal) หรือเป็นอินพุต (Input) เพื่อรับสัญญาณดิจิทัล |
| **Analog-to-Digital Converter** | ADC | การแปลงสัญญาณอนาล็อกเป็นดิจิทัล | ใช้สำหรับวัดค่าแรงดันไฟฟ้า (Voltage) หรือสัญญาณต่อเนื่อง (Continuous Signal) จากเซนเซอร์ เช่น อุณหภูมิ ความชื้น |
| **Power Supply** | $V_{CC}$, $GND$ | การจ่ายกำลังไฟฟ้า | ขาสำหรับจ่ายแรงดันไฟฟ้าหลัก (Power Rail) และกราวด์ (Ground Reference) |

### 3.2.2.2 การวิเคราะห์ขา GPIO และ ADC

**1. ขา GPIO (General Purpose Input/Output):**
ขาเหล่านี้สามารถกำหนดค่าได้ทั้งเป็นอินพุตและเอาต์พุต โดยทั่วไปแล้วสัญญาณดิจิทัลที่ส่งออกจะมีระดับแรงดันไฟฟ้า (Voltage Level) ที่ 3.3 V (โวลต์) ซึ่งเป็นแรงดันไฟฟ้ามาตรฐานของ ESP32 การขับโหลด (Load) ที่ใช้แรงดันไฟฟ้า 5 V จำเป็นต้องใช้ระดับตัวแปลงสัญญาณ (Level Shifter) เพื่อป้องกันความเสียหายต่อ MCU

**2. ขา ADC (Analog-to-Digital Converter):**
ESP32 มี ADC ที่มีความละเอียด (Resolution) มักอยู่ที่ 12 บิต (bits) ซึ่งหมายความว่าสามารถวัดค่าแรงดันไฟฟ้าในช่วงที่กำหนด (เช่น 0 V ถึง 3.3 V) และแปลงเป็นค่าดิจิทัลจำนวนเต็ม (Integer Value) ได้ $2^{12} = 4096$ ระดับ

**หลักการคำนวณค่าดิจิทัลจาก ADC:**
หากกำหนดให้ $V_{ref}$ คือแรงดันอ้างอิง (Reference Voltage) และ $V_{in}$ คือแรงดันไฟฟ้าที่วัดได้ สูตรการแปลงค่าดิจิทัล $D$ คือ:
$$D = \text{round} \left( \frac{V_{in}}{V_{ref}} \times 2^N - 1 \right)$$
โดยที่ $N$ คือความละเอียดของ ADC (ในที่นี้คือ 12 บิต)

**ข้อควรระวัง:** การใช้งาน ADC ต้องคำนึงถึงข้อจำกัดของแรงดันไฟฟ้าที่ป้อนเข้า (Input Voltage Range) และความแม่นยำ (Accuracy) ของ ADC ในแต่ละรุ่นของบอร์ด

## 3.2.3 ความสำคัญของชิปแปลงสัญญาณ USB-to-Serial (CH340 Driver)

แม้ว่า ESP32 จะเป็นไมโครคอนโทรลเลอร์ที่ทรงพลัง แต่การสื่อสารกับคอมพิวเตอร์ (Host Computer) เพื่อการเขียนโปรแกรม (Programming) และการดีบัก (Debugging) จำเป็นต้องมีส่วนประกอบที่ทำหน้าที่เป็น **ตัวแปลงสัญญาณ USB-to-Serial**

### 3.2.3.1 บทบาทของ USB-to-Serial Converter

คอมพิวเตอร์ส่วนใหญ่สื่อสารกับอุปกรณ์ภายนอกผ่านพอร์ต USB ซึ่งใช้โปรโตคอล (Protocol) ที่ซับซ้อนกว่าการสื่อสารแบบ Serial (เช่น UART) โดยตรง

*   **Serial Communication (UART):** เป็นการส่งข้อมูลแบบลำดับ (Sequential Data) ผ่านสายสัญญาณสองเส้น (TX/RX) โดยมีอัตราการส่งข้อมูล (Baud Rate) ที่กำหนด
*   **USB-to-Serial Chip (เช่น CH340, FTDI, CP2102):** ชิปเหล่านี้ทำหน้าที่เป็นสะพาน (Bridge) โดยรับสัญญาณดิจิทัลความเร็วสูงจาก USB (USB Protocol) และแปลงให้เป็นสัญญาณระดับแรงดันไฟฟ้า (Voltage Level) และรูปแบบสัญญาณ (Signal Format) ที่ MCU สามารถเข้าใจได้ในรูปแบบ Serial (UART)

**CH340:** เป็นชิปที่ได้รับความนิยมอย่างสูงเนื่องจากมีราคาถูกและมีประสิทธิภาพในการแปลงสัญญาณ USB เป็น UART ทำให้บอร์ดพัฒนาจำนวนมากใช้ชิปนี้เป็นตัวเชื่อมต่อระหว่าง ESP32 กับคอมพิวเตอร์

### 3.2.3.2 ขั้นตอนการติดตั้งไดรเวอร์ (Driver Installation)

เพื่อให้ระบบปฏิบัติการ (Operating System: OS) เช่น Windows หรือ macOS สามารถ "มองเห็น" และสื่อสารกับบอร์ด ESP32 ได้อย่างถูกต้อง จำเป็นต้องติดตั้งไดรเวอร์ (Driver) ของชิปแปลงสัญญาณ CH340 ก่อน

**ขั้นตอนโดยสังเขป (สำหรับ Windows OS):**

1.  **การเชื่อมต่อ:** เสียบสาย USB เข้ากับบอร์ด ESP32 และเสียบปลายอีกด้านเข้ากับพอร์ต USB ของคอมพิวเตอร์
2.  **การตรวจจับ:** ระบบปฏิบัติการจะตรวจพบอุปกรณ์ใหม่ แต่เนื่องจากไม่มีไดรเวอร์ที่เหมาะสม จึงอาจแสดงข้อความแจ้งเตือนว่า "Unknown Device"
3.  **การติดตั้งไดรเวอร์:** ผู้ใช้จะต้องดาวน์โหลดไฟล์ไดรเวอร์ CH340 (มักเป็นไฟล์ `.zip` หรือ `.exe`) จากเว็บไซต์ผู้ผลิตที่เชื่อถือได้
4.  **การดำเนินการ:** ดับเบิลคลิกที่ไฟล์ติดตั้งและทำตามขั้นตอนบนหน้าจอ (Next $\rightarrow$ Finish)
5.  **การตรวจสอบ:** หลังจากติดตั้งเสร็จสิ้น ให้ตรวจสอบใน **Device Manager** (ตัวจัดการอุปกรณ์) ของ Windows ภายใต้หมวดหมู่ "Ports (COM & LPT)" ควรปรากฏชื่ออุปกรณ์ที่ระบุว่า "USB-SERIAL CH340" หรือคล้ายกัน พร้อมหมายเลขพอร์ต (เช่น COM3) ซึ่งหมายความว่าการสื่อสารทางกายภาพ (Physical Communication) พร้อมใช้งานแล้ว

## 3.2.4 การตั้งค่าสภาพแวดล้อมการพัฒนา Arduino IDE

Arduino IDE (Integrated Development Environment) เป็นเครื่องมือที่ช่วยให้ผู้ใช้สามารถเขียนโค้ด (Sketch Code) อัปโหลด (Upload) และตรวจสอบ (Monitor) การทำงานของไมโครคอนโทรลเลอร์ได้อย่างง่ายดาย

### 3.2.4.1 การเพิ่มการรองรับบอร์ด ESP32 ผ่าน Boards Manager

โดยค่าเริ่มต้น Arduino IDE อาจรองรับเฉพาะบอร์ด Arduino รุ่นเก่าเท่านั้น การจะให้ IDE สามารถคอมไพล์ (Compile) และอัปโหลดโค้ดไปยัง ESP32 ได้ ผู้ใช้ต้องเพิ่มการสนับสนุนบอร์ด (Board Support Package: BSP) ของ ESP32 เข้าไปใน IDE

**ขั้นตอนการเพิ่ม BSP:**

1.  **การเพิ่ม URL Repository:** เข้าไปที่เมนู `File` $\rightarrow$ `Preferences` (หรือ `Arduino` $\rightarrow$ `Preferences` ใน macOS) และเพิ่ม URL ของ Board Manager สำหรับ ESP32 (เช่น `https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json`) ในช่อง "Additional Boards Manager URLs"
2.  **การติดตั้ง Board Package:** เข้าไปที่เมนู `Tools` $\rightarrow$ `Board` $\rightarrow$ `Boards Manager`
3.  **การค้นหาและติดตั้ง:** ในช่องค้นหา ให้พิมพ์ "esp32" และคลิกที่ปุ่ม "Install" เพื่อดาวน์โหลดและติดตั้งไลบรารีและคอมไพเลอร์ที่จำเป็นสำหรับ ESP32

### 3.2.4.2 โครงสร้างหลักของโค้ด (The Sketch Structure)

โค้ดโปรแกรมที่เขียนสำหรับไมโครคอนโทรลลอร์ส่วนใหญ่จะถูกจัดโครงสร้างตามหลักการทำงานที่กำหนดไว้ในฟังก์ชันหลักสองฟังก์ชัน คือ `setup()` และ `loop()`

**1. ฟังก์ชัน `setup()`:**
*   **วัตถุประสงค์:** ฟังก์ชันนี้จะถูกเรียกใช้เพียง **ครั้งเดียว** (Once) เมื่อบอร์ดได้รับพลังงาน (Power Up) หรือถูกรีเซ็ต (Reset)
*   **การใช้งาน:** ใช้สำหรับกำหนดค่าเริ่มต้น (Initialization) ของระบบทั้งหมด เช่น การกำหนดโหมดของขาพิน (Pin Mode: `INPUT` หรือ `OUTPUT`), การเริ่มต้นการสื่อสาร (Serial Communication), และการเชื่อมต่อเครือข่าย (Wi-Fi Connection)

**2. ฟังก์ชัน `loop()`:**
*   **วัตถุประสงค์:** ฟังก์ชันนี้คือหัวใจของการทำงานของโปรแกรม มันจะถูกเรียกใช้ซ้ำๆ (Repeatedly) อย่างต่อเนื่องด้วยความถี่สูงสุดเท่าที่ MCU จะทำได้ (High Frequency) ตราบใดที่บอร์ดยังมีพลังงาน
*   **การใช้งาน:** ใช้สำหรับใส่ตรรกะการทำงานหลัก (Main Logic) ของระบบ เช่น การอ่านค่าเซนเซอร์, การประมวลผลข้อมูล, และการส่งข้อมูลผ่านเครือข่าย

### 3.2.4.3 ตัวอย่างโค้ด C++ สำหรับการทดสอบการทำงานพื้นฐาน (Blink Test)

ตัวอย่างโค้ดนี้แสดงการกระพริบไฟ LED ที่ต่อกับขา GPIO 2 (ซึ่งเป็นขาที่มักจะเชื่อมต่อกับ LED บนบอร์ด ESP32 DevKit) เพื่อยืนยันว่าการตั้งค่าบอร์ดและการอัปโหลดโค้ดทำงานได้อย่างสมบูรณ์

```cpp
// =================================================================
// ไฟล์: esp32_blink_test.ino
// คำอธิบาย: โค้ดตัวอย่างการกระพริบ LED เพื่อทดสอบการทำงานของบอร์ด ESP32
// ภาษา: C++ (Arduino Framework)
// =================================================================

// กำหนดขา GPIO ที่เชื่อมต่อกับ LED (สำหรับ ESP32 DevKit ส่วนใหญ่คือ GPIO 2)
const int ledPin = 2; 

// ฟังก์ชัน setup(): ถูกเรียกใช้เพียงครั้งเดียวเมื่อเริ่มทำงาน (Initialization)
void setup() {
  // 1. การเริ่มต้นการสื่อสารแบบอนุกรม (Serial Communication)
  // Baud Rate 115200 bps: กำหนดอัตราการส่งข้อมูล 115,200 บิตต่อวินาที
  // ใช้สำหรับการดีบักและดูค่าที่ส่งจากบอร์ดไปยังคอมพิวเตอร์
  Serial.begin(115200);
  Serial.println("--- ESP32 Initializing System ---");

  // 2. การกำหนดโหมดของขาพิน (Pin Mode Configuration)
  // กำหนดให้ขา ledPin ทำหน้าที่เป็นเอาต์พุต (Output) เพื่อจ่ายสัญญาณไฟฟ้า
  pinMode(ledPin, OUTPUT);
  Serial.println("GPIO Pin " + String(ledPin) + " configured as OUTPUT.");
}

// ฟังก์ชัน loop(): ถูกเรียกใช้ซ้ำๆ อย่างต่อเนื่อง (Main Program Loop)
void loop() {
  // 1. การจ่ายสัญญาณ HIGH (High Voltage State)
  // การตั้งค่าแรงดันไฟฟ้าที่ขา ledPin ให้เป็นระดับสูง (3.3 V) เพื่อให้ LED สว่าง
  digitalWrite(ledPin, HIGH); 
  Serial.println("LED ON. Time: " + String(millis() / 1000) + " s");

  // 2. การหน่วงเวลา (Delay Function)
  // หยุดการทำงานชั่วคราวเป็นเวลา 500 มิลลิวินาที (ms) หรือ 0.5 วินาที
  delay(500); 

  // 3. การจ่ายสัญญาณ LOW (Low Voltage State)
  // การตั้งค่าแรงดันไฟฟ้าที่ขา ledPin ให้เป็นระดับต่ำ (0 V) เพื่อให้ LED ดับ
  digitalWrite(ledPin, LOW);
  Serial.println("LED OFF. Time: " + String(millis() / 1000) + " s");

  // 4. การหน่วงเวลาอีกครั้ง
  delay(500); 
}
```

## สรุปเชิงวิชาการ (Academic Summary)

การทำงานของระบบ IoT ที่ใช้ ESP32 นั้นอาศัยการบูรณาการองค์ประกอบทางอิเล็กทรอนิกส์และซอฟต์แวร์เข้าด้วยกันอย่างแม่นยำ:

1.  **ฮาร์ดแวร์ (Hardware):** ESP32 ทำหน้าที่เป็นสมองกล (MCU) ที่มีขีดความสามารถในการประมวลผล 32 บิต และการสื่อสารไร้สายในตัว
2.  **การเชื่อมต่อ (Interfacing):** ชิป CH340 ทำหน้าที่เป็นตัวแปลงสัญญาณที่จำเป็น
